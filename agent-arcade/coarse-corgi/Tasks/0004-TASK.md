# Task 0004 — Arcade foundation (machines + bus + render)

## Goal
Stand up the Arcade's vanilla-JS + XState + mitt core and fix the headline draft-loss bug, with
the UI reused **verbatim**.

## Steps
1. Reuse `arcade/renderer/index.html` + CSS from the reference **verbatim**; bundle the new JS
   via `build:js` → `arcade/dist/**` (keep loading the pre-vendored `xterm.js`).
2. Build the **root Arcade machine** + a **child actor spawned per agent** holding durable
   context (draft input, view sub-state), and a **`navigation` machine**.
3. Add the **mitt** event bus and the **IPC→XState seam**: every IPC/main message (incl. Go
   events) is translated **once** into an XState event for the owning machine. Components never
   touch a machine directly. Rule: mitt = notifications; XState events = transitions.
4. Implement a thin `subscribe(state) → DOM` render layer replacing `draw()`.
5. Wire the agents rail, agent view, and `⌘←/→` switching against the machines. Per-agent draft
   lives in the per-agent actor's context.

## Definition of done
Type into an agent's prompt, `⌘→` to another agent, `⌘←` back → **draft intact**. No DOM desync
across switches. `node --check` clean; `npm run build` green. No React in the Arcade.
