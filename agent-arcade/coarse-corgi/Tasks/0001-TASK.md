# Task 0001 — Scaffold + pipeline + Go + Electron shell

## Goal
Stand up the `agent-arcade` package so it **launches** with stubbed renderers, on the preserved
release pipeline plus the one sanctioned esbuild stage.

## Steps
1. Initialize the package in `agent-arcade/`: `package.json` name `@talkersoft-com/agent-arcade`,
   bin `agent-arcade`, version `0.1.0`. Copy from the reference and **remove** the
   electron-builder `build` block and the `dist`/`release`/`pack` scripts (no DMG).
2. Copy packaging as-is in intent: `.github/workflows/publish.yml`, `scripts/fetch-wezterm.sh`,
   `scripts/fix-node-pty.js` (keep the node-pty `spawn-helper` postinstall), and the `files`
   allowlist approach (add `studio/dist/**` and `arcade/dist/**`).
3. Bring over the Go sources as-is: `go/` (dictation-go) and `wezterm-bridge/`. Keep
   `build:go` / `build:wezterm` (CGO + ad-hoc codesign) and "verify Go binaries present".
4. Bring over Electron `main.js`, `preload.js`, `launcher/`, `arcade/main.js`, `arcade/preload.js`
   as-is. Point Studio + Arcade renderers at **stub** entry points for now.
5. Add an **esbuild `build:js`** script that produces `studio/dist/**` and `arcade/dist/**`;
   wire `npm run build` = `build:js → vendor:wezterm → build:go → build:wezterm`.
6. Run `npm run build`; sanity-check edited JS with `node --check`.

## Definition of done
`npm run build` succeeds; `agent-arcade` launches the menu-bar launcher and opens stub Studio +
Arcade windows; `npm pack --dry-run` includes `go/bin/**`, `wezterm-bridge/bin/**`,
`vendor/wezterm/**`, `studio/dist/**`, `arcade/dist/**`. No DMG/electron-builder remains.
