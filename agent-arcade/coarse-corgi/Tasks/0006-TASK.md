# Task 0006 — Arcade terminal surface

## Goal
Rebuild the terminal-facing surfaces as machines with optimistic-reconciliation for
`wezterm-bridge`, matching the reference.

## Steps
1. **Live pane peek:** poll `wezterm-bridge get-text` and render the scrape (verbatim UI).
2. **Send text** to a pane; **`^C`** interrupt.
3. **Sync mode (`⌘F`):** a `syncMode` machine forwarding every keystroke raw to the pane.
4. **Workspace shell (`⌘W`):** a `workspaceShell` machine `idle → opening → open → closed` over
   node-pty + xterm.js. Verify the node-pty `spawn-helper` exec bit (postinstall) so the shell
   actually spawns.
5. **`@`-macro picker:** arg types `select` / `text` / `flag` (flag emits its literal token or
   nothing; select/text values stay shell-quoted), run in the workspace shell.
6. **wezterm-bridge ops are optimistic-with-reconciliation:** success on exit-0, **error state on
   non-zero exit** — surfaced, never assumed success.

## Definition of done
Each surface matches the reference (peek, send, `^C`, sync `⌘F`, shell `⌘W`, macros
`select`/`text`/`flag`). A non-zero `wezterm-bridge` exit shows an error state. `⌘W` spawns a real
shell. Build green.
