# Test 0005 — E2E

## TC-001 — API list returns paged envelope
```
curl -sS http://localhost:5250/api/v1/Run?pageSize=50 | jq '{page, pageSize, total, count: (.items|length)}'
```
Pass: `count ≤ pageSize`; `total ≥ count`.

## TC-002 — VM filter is the join, not a guess
Pick a VM with run history (see Task 0005 step 1 for the psql call).
```
curl -sS "http://localhost:5250/api/v1/Run?vmId=$VMID&pageSize=50" | jq '.total'
```
Cross-check with:
```
psql -h localhost -U cloudmanager -d clouddb -tAc "SELECT COUNT(DISTINCT t.run_id) FROM vm.playbook_run_targets t JOIN vm.virtual_machines v ON v.id = t.vm_id WHERE v.public_id = '$VMID'"
```
Pass: the two numbers match.

## TC-003 — No token leakage

```
curl -sS http://localhost:5250/api/v1/Run?pageSize=200 | jq '[.items[].vaultRunToken] | map(select(. != null)) | length'
```

## TC-005 — Browser smoke
Open `http://localhost:3000/ansible/history`.
- Grid renders with up to 50 rows.
- Pagination "Next" loads page 2; URL updates to `?page=2`.
- Selecting a VM in the combobox filters; URL updates to `?vmId=vm_XXXX&page=1`.
- Clicking any row navigates to `/ansible/runs/:runId` and the existing detail view renders without errors.
- Browser Back returns to `/ansible/history` with the same filter state.

Pass: all five interactions behave as above.
