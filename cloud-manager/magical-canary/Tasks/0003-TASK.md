# Task 0003 — MCP: cloud_playbook_run_list

## Goal
Expose the new `GET /api/v1/Run` list endpoint as an MCP tool. Thin pass-through, no client-side filtering or merging.

## Files
- `cloud-manager-mcp/src/tools/ansible.ts` (extend, in the "Runs" section starting at line 125)

## Edits

Add this tool below `cloud_run_cancel` (around line 164). Reuse the file's existing `apiRequest`, `idSchema`, `formatResult`, and `z` imports.

```typescript
server.tool(
  "cloud_playbook_run_list",
  "List playbook runs across the system, paginated and filterable by VM, playbook, or status. Newest first.",
  {
    vmId: idSchema("Filter to runs that targeted this VM (any-target match)").optional(),
    playbookId: idSchema("Filter to runs of this playbook").optional(),
    status: z.enum(["Queued", "Running", "Succeeded", "Failed", "Cancelled"])
      .describe("Filter by run status").optional(),
    page: z.number().int().min(1).default(1).describe("1-based page number").optional(),
    pageSize: z.number().int().min(1).max(200).default(50).describe("Items per page (1-200; default 50)").optional(),
  },
  async ({ vmId, playbookId, status, page, pageSize }) => {
    const params = new URLSearchParams();
    if (vmId)       params.set("vmId", vmId);
    if (playbookId) params.set("playbookId", playbookId);
    if (status)     params.set("status", status);
    if (page)       params.set("page", String(page));
    if (pageSize)   params.set("pageSize", String(pageSize));
    const qs = params.toString();
    const result = await apiRequest<unknown>("GET", `/api/v1/run${qs ? `?${qs}` : ""}`);
    if (!result.ok) {
      return { content: [{ type: "text", text: `Error: ${result.error}` }], isError: true };
    }
    return { content: [{ type: "text", text: formatResult(result.data) }] };
  },
);
```

## Notes
- Tool name is `cloud_playbook_run_list` (not `cloud_run_list`) — matches the plan's MCP surface and the existing `cloud_playbook_*` naming family.
- The MCP enum mirrors the C# enum's string names exactly. Integer values not exposed via MCP (operators wouldn't think in ints).
- Page / pageSize bounds are validated client-side AND server-side (server clamps; we set the same bounds in zod so bad calls fail fast with a useful error).
