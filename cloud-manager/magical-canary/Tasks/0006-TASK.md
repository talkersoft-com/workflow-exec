# Task 0006 — Deploy + Results + Ship

## Goal
Deploy api + mcp + web to the running system, write the workflow's `Results/RESULT.md` + `Retro/LESSONS.md`, then `hv_ship` so the deck transitions.

## Steps

### 1. Deploy all targets

```
python3 .cicd/deploy-cloud-manager.py --target all
```

This rebuilds + restarts cloud-manager-api (smoke test against `/api/v1/Host`), runs `npm ci && npm run build` for mcp (no restart — operator does `/mcp`), and rebuilds + rsyncs cloud-manager-web (smoke test against `http://localhost:3000/`).

If any target fails its smoke test, write `Retro/FIX-NNN.md` and resolve before continuing.

### 4. Ship

```
hv_ship  deck: "cloud-manager"
         message: "ansible history: paginated runs grid with VM filter; new GET /api/v1/Run + cloud_playbook_run_list MCP tool"
         title:   "ansible history: paginated playbook-run grid with VM filter"
```

`hv_ship` opens PRs for every dirty / unpushed repo in the deck (`cloud-manager-api`, `cloud-manager-mcp`, `cloud-manager-web`, plus `workflow-exec` carrying this workflow). On merge it auto-transitions the deck to the next branch.

After `hv_ship` returns, print the PR URLs to the user verbatim.

## Definition of done
Deploy clean, Results + LESSONS committed in workflow-exec, PRs opened.
