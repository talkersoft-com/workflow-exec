# Task 0002 — API: GET /api/v1/Run list endpoint

## Goal
Expose `PlaybookRunService.ListAsync` over HTTP. Query string accepts `vmId`, `playbookId`, `status`, `page`, `pageSize`. Inherits the existing `[RequireFeatureFlag("playbooks")]` gate from the controller.

## Files
- `cloud-manager-api/src/CloudManager.API/Controllers/RunController.cs` (extend)

## Edits

Add a new `[HttpGet]` action with no route suffix (it lives at `GET /api/v1/Run`). Keep the per-run `[HttpGet("{runId}")]` action intact — ASP.NET routes the no-suffix action when no `runId` is present.

```csharp
[HttpGet]
public async Task<IActionResult> List(
    [FromQuery] string? vmId,
    [FromQuery] string? playbookId,
    [FromQuery] PlaybookRunStatus? status,
    [FromQuery] int page = 1,
    [FromQuery] int pageSize = 50)
{
    try
    {
        var result = await _service.ListAsync(vmId, playbookId, status, page, pageSize);
        return Ok(result);
    }
    catch (Exception ex) { _logger.LogError(ex, "list failed"); return StatusCode(500, new { Message = "Internal server error" }); }
}
```

## Notes
- `PlaybookRunStatus` is the existing enum (`Queued`, `Running`, `Succeeded`, `Failed`, `Cancelled`). ASP.NET's default `EnumConverter` accepts both the integer (`?status=2`) and the string (`?status=Running`, case-insensitive). No custom binder needed.
- `vmId` / `playbookId` are public IDs (`vm_XXXX`, `pb_XXXX`) — service is responsible for the translation.
- The controller already has `[RequireFeatureFlag("playbooks")]` at the class level. The new action inherits it. Do not add a second attribute and do not weaken the gate.
