# Task 0005 — E2E verify

## Goal
Confirm the full stack — API list endpoint, MCP tool, web grid — works end to end against the running cloud-manager. No code changes in this phase; only verification.

## Steps

### 1. API directly

```bash
curl -sS 'http://localhost:5250/api/v1/Run?pageSize=50' | jq '{ page, pageSize, total, count: (.items|length), firstStartedAt: (.items[0].startedAt // null), anyTokenLeaked: ([.items[].vaultRunToken] | map(select(. != null)) | length) }'
```

Expected:
- `count` ≤ 50
- `total` is the row count of `vm.playbook_runs`
- `firstStartedAt` is the newest run's StartedAt
- `anyTokenLeaked` == 0 (the scrub holds)

```bash
# Pick a VM that has run history
VMID=$(psql -h localhost -U cloudmanager -d clouddb -tAc "SELECT v.public_id FROM vm.virtual_machines v JOIN vm.playbook_run_targets t ON t.vm_id = v.id GROUP BY v.public_id ORDER BY COUNT(*) DESC LIMIT 1")
curl -sS "http://localhost:5250/api/v1/Run?vmId=$VMID&pageSize=10" | jq '{ total, count: (.items|length), vmId: "'"$VMID"'" }'
```

Expected: every returned run targeted that VM. Cross-check by counting `playbook_run_targets` rows for that `vm_id`.

### 2. MCP tool (operator reload required first)

After Phase 3 rebuilt `dist/`, the operator must `/mcp` reload in Claude Code before the new tool is visible. Then:

```
cloud_playbook_run_list  page: 1  pageSize: 5
cloud_playbook_run_list  vmId: <vm_id>  pageSize: 10
```

Expected: shape matches the API envelope verbatim.

### 3. Browser

Open `http://localhost:3000/ansible/history`.

- A grid of up to 50 rows renders, newest first.
- `Pagination` shows "page 1 of N".
- Clicking "Next" loads page 2 and updates the URL to `?page=2`.
- Selecting a VM in the combobox sets `?vmId=vm_XXXX&page=1` and re-renders the grid filtered.
- Clicking any row navigates to `/ansible/runs/:runId` — the existing live/historical detail page renders without errors.
- Pressing Back returns to `/ansible/history` with filters intact (URL preserved them).
