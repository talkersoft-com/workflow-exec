# Task 0001 — HV_HOME Direct Path in config.LoadSetup()

## Goal

Adjust `config.LoadSetup()` so that when `$HV_HOME` is set, it checks `$HV_HOME/` directly (in addition to `$HV_HOME/.hv/`). This allows the `workflow-configuration` repo to be used as-is without being named `.hv`.

## Target repo

`hive-deck-pro` — `internal/config/config.go`

## Steps

1. Read `config.LoadSetup()` and the search logic in `findInConfigSubdir()` / the home resolution functions.

2. Locate where `$HV_HOME` is used. Currently it prepends `$HV_HOME/.hv/` to the search path. Add a check that tries `$HV_HOME/` directly FIRST:

```go
// In the config search path, when HV_HOME is set:
// 1. Try $HV_HOME/          ← NEW: direct path (for workflow-configuration repo)
// 2. Try $HV_HOME/.hv/      ← existing behavior (legacy support)
// 3. Try CWD/.hv/
// 4. Try $HOME/.hv/
```

3. The `config.yaml` file must be found at `$HV_HOME/config.yaml` (direct) OR `$HV_HOME/.hv/config.yaml` (appended).

4. Deck files must be found at `$HV_HOME/decks/*.yaml` (direct) OR `$HV_HOME/.hv/decks/*.yaml` (appended).

5. `CGO_ENABLED=0 go build ./...` must pass.
6. `CGO_ENABLED=0 go test ./...` must pass.

## Definition of done

Build and tests pass. `HV_HOME=/some/path` finds config at `/some/path/config.yaml` directly.
