# Task 0005 — Vault bootstrap script

## Goal
A single Python script `cloud-manager-api/scripts/vault/bootstrap-policies.py` provisions all Vault state the playbook execution path needs. Idempotent — re-running produces no Vault state change.

## File
`cloud-manager-api/scripts/vault/bootstrap-policies.py` (new)

## Behavior

```
python3 bootstrap-policies.py --vault-addr https://ubuntu-server.talkersoft.com:8200 --root-token $VAULT_ROOT_TOKEN
```

Provisions:

### 1. `playbook-run` ACL policy (attached to short-lived child tokens)

```hcl
path "cloudmanager/data/playbook-secrets/*" {
  capabilities = ["create", "update", "read"]
}
path "cloudmanager/metadata/playbook-secrets/*" {
  capabilities = ["read", "list"]
}
```

### 2. `cloudmanager-playbook-runner` ACL policy (orchestrator — UPDATED to read playbook-secrets)

Read the existing policy; preserve the auth/token/create + auth/token/revoke paths. Replace the obsolete `cloudmanager/data/vm/instances/+/+/*` read paths with:

```hcl
path "cloudmanager/metadata/playbook-secrets/+/*" {
  capabilities = ["list", "read"]
}
path "cloudmanager/data/playbook-secrets/+/*" {
  capabilities = ["read"]
}
```

### 3. `playbook-run` token role

```
allowed_policies      = ["playbook-run"]
token_explicit_max_ttl = "2h"
orphan                 = true
renewable              = false
```

## Implementation hints

Use `hvac` Python library (already installed on this dev box). Read current state before writing — only write if the rendered policy/role differs. Print a one-line action per resource (`policy playbook-run: unchanged` or `policy playbook-run: updated`).

## Definition of done
1. Script runs without error against the dev-box Vault.
2. Second run prints `unchanged` for every resource.
3. `vault policy read playbook-run` shows the documented paths.
4. `vault read auth/token/roles/playbook-run` shows `allowed_policies=[playbook-run]`, `token_explicit_max_ttl=2h0m0s`, `orphan=true`, `renewable=false`.
5. README updated with how to run it.
