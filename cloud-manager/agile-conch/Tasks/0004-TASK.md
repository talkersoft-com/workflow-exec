# Task 0004 — Revert playbook content to per-run scoping

## Goal
Both `pg-14-jammy` and `pg-16-noble` playbooks roll back the six dev-only triage patches and use `{{ run_public_id }}` (injected in Phase 1) for the per-run Vault path.

## Edits

For BOTH playbooks, via `cloud_playbook_update` (NOT raw SQL):

```yaml
# Before — vault_secret_path var
vault_secret_path: "cloudmanager/data/playbook-secrets/{{ vm_public_id }}/{{ inventory_hostname }}/postgres"

# After
vault_secret_path: "cloudmanager/data/playbook-secrets/{{ run_public_id }}/{{ inventory_hostname }}/postgres"
```

```yaml
# Before — vault_kv2_write task
- name: Store the generated password in Vault (KV v2)
  community.hashi_vault.vault_kv2_write:
    url: "{{ vault_addr }}"
    auth_method: token
    token: "{{ vault_token }}"
    validate_certs: false
    engine_mount_point: cloudmanager
    path: "playbook-secrets/{{ vm_public_id }}/{{ inventory_hostname }}/postgres"
    data: { ... }
  delegate_to: localhost
  become: false
  vars:
    ansible_remote_tmp: /tmp/.ansible-porch-tmp

# After
- name: Store the generated password in Vault (KV v2)
  community.hashi_vault.vault_kv2_write:
    url: "{{ vault_addr }}"
    auth_method: token
    token: "{{ vault_token }}"
    engine_mount_point: cloudmanager
    path: "playbook-secrets/{{ run_public_id }}/{{ inventory_hostname }}/postgres"
    data: { ... }
```

```yaml
# Before — apt install
- name: Install PostgreSQL packages
  ansible.builtin.apt:
    name:
      - "postgresql-{{ pg_version }}"
      - python3-psycopg2
      - python3-hvac
      - acl
    state: present
    update_cache: true

# After
- name: Install PostgreSQL packages
  ansible.builtin.apt:
    name:
      - "postgresql-{{ pg_version }}"
      - python3-psycopg2
      - acl
    state: present
    update_cache: true
```

## Steps

For each playbook:

1. `cloud_playbook_get id=<pb_id>` to capture current content
2. Apply the three edits above to the YAML string
3. `cloud_playbook_update playbookId=<pb_id> content=<new-yaml> changeSummary="revert triage patches: run_public_id, drop delegate/become/tmp/validate_certs/hvac"`
4. `cloud_playbook_get` to confirm new revision

## Definition of done

`grep` on both playbooks' content returns nothing for `delegate_to:`, `become: false`, `ansible_remote_tmp`, `validate_certs: false`, `python3-hvac`, or `vm_public_id` in the vault_secret_path context. The path uses `{{ run_public_id }}`.
