# Task 0006 — Playbook export + import scripts

## Goal
Playbook content has a git source-of-truth at `cloud-manager-api/seed/playbooks/`. The `export-playbooks.py` script pulls the current DB state into seed files; `import-playbooks.py` is the reverse — used during bootstrap to seed a fresh DB with the canonical playbooks.

## Files
- `cloud-manager-mcp/.cicd/export-playbooks.py` (new)
- `cloud-manager-mcp/.cicd/import-playbooks.py` (new)
- `cloud-manager-api/seed/playbooks/` (new directory — populated by running export once)

## export-playbooks.py

```
GET /api/v1/playbook/list   →   array of summaries
for each:
  GET /api/v1/playbook/{id}   →   full DTO with content + meta
  write cloud-manager-api/seed/playbooks/<name>.yaml         ← just the YAML content
  write cloud-manager-api/seed/playbooks/<name>.meta.json    ← description, argumentSchema, outputSchema, current revision public_id
```

Idempotent — re-running produces identical files unless content changed.

## import-playbooks.py

```
walk cloud-manager-api/seed/playbooks/*.yaml
for each:
  GET /api/v1/playbook/list → find by name
  if missing → POST /api/v1/playbook (cloud_playbook_create call shape)
  if present and content differs → PATCH /api/v1/playbook/{id} with changeSummary "import-playbooks.py sync"
  if present and same → log "no-op"
```

Preserves operator-authored playbooks not present in the seed directory.

## Steps

1. Implement both scripts. Use the `cloud_*` MCP tools' underlying HTTP calls — direct `requests` calls against `http://localhost:5250/api/v1/...` is fine for these `.cicd` scripts (they're not consumed by workflow agents).
2. Run `export-playbooks.py` once. Verify the produced `seed/playbooks/*.yaml` matches the current DB state (the two playbooks reverted in Phase 4 are the canonical content).
3. Commit `cloud-manager-api/seed/playbooks/` to the branch.
4. Smoke-test import: `db_provision_fresh` against `clouddb_local` (or use `db_execute_sql_script` to wipe `vm.playbooks` + `vm.playbook_revisions`), then run `import-playbooks.py`, verify the playbooks reappear.

## Definition of done
1. Both scripts exist + run.
2. `seed/playbooks/` committed with `pg-14-jammy.yaml`, `pg-14-jammy.meta.json`, `pg-16-noble.yaml`, `pg-16-noble.meta.json`.
3. `import-playbooks.py` is idempotent (second run is no-op).
4. A clean-DB → import → run-playbook cycle works end-to-end.
