# Task 0003 — Extend `cloud_playbook_update` for content patches

## Goal
The `cloud_playbook_update` MCP tool accepts a `content` parameter that updates `vm.playbooks.content` and creates a new revision row in `vm.playbook_revisions`. Without this, Phase 4 falls back to raw SQL (which the workflow constraints forbid).

## Investigation
Check what the tool currently accepts:

```bash
grep -A 30 'cloud_playbook_update' /home/todd/workspace/cloud-manager/vm-infra/cloud-manager/cloud-manager-mcp/src/tools/ansible.ts
```

If `content` is already a parameter and the tool calls `PATCH /api/v1/playbook/{id}` with it, Phase 3 is a no-op — skip to Phase 4.

Otherwise, extend the chain:

1. **cloud-manager-api** — `PlaybookController.Update` accepts `content` in `UpdatePlaybookRequest`. `PlaybookService.UpdateAsync` writes the new content to `vm.playbooks.content` AND inserts a new `vm.playbook_revisions` row with an incremented `revision_number`. Match the pattern in the SQL `fix_*.sql` triage scripts: `change_summary` should be a per-update string the caller provides.
2. **cloud-manager-mcp** — extend `cloud_playbook_update`'s Zod schema to include `content` (string, optional) and `changeSummary` (string, optional). Build + restart MCP dist; operator will `/mcp` reload.

## Files
- `cloud-manager-api/src/CloudManager.API/Controllers/PlaybookController.cs`
- `cloud-manager-api/src/Services/CloudManager.Data.Services/PlaybookService.cs`
- `cloud-manager-api/src/Models/CloudManager.API.Models/UpdatePlaybookRequest.cs` (or wherever the DTO lives)
- `cloud-manager-mcp/src/tools/ansible.ts`

## Build + deploy

- API: `python3 .cicd/build-cloud-manager.py --target api` then `python3 .cicd/deploy-cloud-manager.py --target api`
- MCP: `python3 .cicd/build-cloud-manager.py --target mcp` then `python3 .cicd/deploy-cloud-manager.py --target mcp` (no live restart — operator does `/mcp` reload)

## Definition of done
`cloud_playbook_update playbookId=<id> content=<new-yaml> changeSummary=<note>` succeeds. The DB shows updated `vm.playbooks.content` AND a new `vm.playbook_revisions` row.
