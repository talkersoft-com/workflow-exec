# Task 0001 — Porch extravars + secrets_prefix

## Goal
Porch injects `run_public_id` into ansible-runner extravars and `secrets_prefix` matches the per-run Vault path cloud-manager-api scopes the run token to.

## Files
- `vorch-service/vorch-service/internal/ansible/playbook_run_exec.go` (around lines 222-228)

## Edits

```go
// Before
secretsPrefix := fmt.Sprintf("cloudmanager/vm/instances/%s/%s", target.VMId, mat.PlaybookPublicID)
extravars := map[string]interface{}{
    "secrets_prefix":     secretsPrefix,
    "vm_public_id":       target.VMId,
    "playbook_public_id": mat.PlaybookPublicID,
}

// After
secretsPrefix := fmt.Sprintf("cloudmanager/data/playbook-secrets/%s", runID)
extravars := map[string]interface{}{
    "secrets_prefix":     secretsPrefix,
    "vm_public_id":       target.VMId,
    "playbook_public_id": mat.PlaybookPublicID,
    "run_public_id":      runID,
}
```

## Build + deploy

`runID` is already the run public_id string — porch references it as `/kvm-automator/playbook-runs/<runID>/`.

1. Build vorch-service: `python3 .cicd/build-vorch.py --target porch` from `vorch-service/`. If the script doesn't exist yet, the canonical build is `cd vorch-service && CGO_ENABLED=0 go build -o /tmp/porch ./cmd/porch`.
2. Replace the running binary: `sudo systemctl stop porch && sudo cp /tmp/porch /kvm-automator/porch && sudo systemctl start porch`.
3. Smoke test by triggering any playbook run and inspecting `/kvm-automator/playbook-runs/<runId>/env/extravars` — it should contain `"run_public_id"` and `"secrets_prefix": "cloudmanager/data/playbook-secrets/<runId>"`.

## Definition of done
Built clean, porch restarted, an `extravars` file from a fresh run shows both new fields with the expected values.
