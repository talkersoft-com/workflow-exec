# Test 0005 — Vault bootstrap script

## TC-001 — First run provisions
`python3 bootstrap-policies.py --vault-addr ... --root-token ...` exits 0; prints `updated` (or `created`) for each of: `playbook-run` policy, `cloudmanager-playbook-runner` policy, `playbook-run` token role.

## TC-002 — Idempotent
Second run prints `unchanged` for every resource.

## TC-003 — Resources match spec
- `vault policy read playbook-run` shows the two `path` blocks (`data/playbook-secrets/*`, `metadata/playbook-secrets/*`).
- `vault policy read cloudmanager-playbook-runner` shows the new playbook-secrets read paths (and still has the auth/token/create + revoke paths).
- `vault read auth/token/roles/playbook-run` shows `allowed_policies=[playbook-run]`, `token_explicit_max_ttl=2h0m0s`, `orphan=true`, `renewable=false`.
