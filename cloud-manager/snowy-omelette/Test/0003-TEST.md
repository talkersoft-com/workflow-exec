# Test 0003 — Internal events wired

## TC-001 — Build clean
Pass: 0 errors.

## TC-002 — Create records `created`
Create a VM. Query `vm.vm_events` for that vm_id. Pass: exactly one `created` row, payload includes `name` + `macAddress` + `hostId`.

## TC-003 — Delete records `destroy_requested`
Soft-delete the VM. Query events. Pass: a `destroy_requested` row appears after the `created` row.

## TC-004 — IP update records `ip_address_updated`
Call `PATCH /api/v1/vm/{publicId}/ip-address` with an IP. Pass: a `ip_address_updated` event row exists with `payload.ip_address` matching.

## TC-005 — Event write never blocks primary action
Simulate a transient DB failure on the events insert (temporarily revoke INSERT on `vm.vm_events`). Confirm the VM is still successfully created / deleted; the event-write failure is logged but not surfaced as a 500.
