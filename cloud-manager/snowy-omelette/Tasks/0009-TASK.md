# Task 0009 — Web: VM activity timeline view

## Goal
Add a "VM activity" panel to the VM detail page that lists events from `GET /api/v1/vm/{publicId}/events`.

## Files
- `cloud-manager-web/src/<vm-detail page>` — panel placement and component import.
- `cloud-manager-web/src/<components>/VmActivityTimeline.tsx` (or framework equivalent) — new component.
- API client method for `GET /events` if not already present.

## Steps
1. Add the API client call: `getVmEvents(publicId, take=100) → VmEvent[]`.
2. Build the timeline component:
   - Most-recent first
   - One row per event: `event_type` badge, `actor`, relative `occurred_at` (with absolute on hover)
   - Expandable `payload` block (pretty-printed JSON; collapsed by default)
3. Mount in the VM detail page below existing primary actions.
4. Handle empty state (new VM with no events) and loading state.

## Notes
- Use the same date / badge styling as the rest of the UI.
- Payload renders as preformatted JSON; do not try to interpret per-type for v1.

## Definition of done
Loading a VM detail page on a VM with events shows the timeline. New VMs show an "empty" state. Old / soft-deleted VMs (visible via the admin filter, when present) show their full history.
