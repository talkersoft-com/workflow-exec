# Task 0002 — VmEventService

## Goal
Implement the append-only `VmEventService` with the two-method surface from the PLAN.

## Files
- `cloud-manager-api/src/Services/CloudManager.Data.Services/Interface/IVmEventService.cs` (new)
- `cloud-manager-api/src/Services/CloudManager.Data.Services/VmEventService.cs` (new)
- DI registration wherever `IVirtualMachineService` is registered.

## Interface

```csharp
public interface IVmEventService
{
    Task<VmEvent> RecordEventAsync(
        Guid vmId,
        VmEventType type,
        object payload,
        string actor,
        DateTime? occurredAt = null);

    Task<IReadOnlyList<VmEvent>> GetTimelineByVmPublicIdAsync(
        string publicId,
        int take = 100);
}
```

## Steps
1. Add interface + implementation. `RecordEventAsync` serializes `payload` to `JsonDocument`, stamps `OccurredAt = occurredAt ?? UtcNow`, sets `Created` / `CreatedBy = actor`, persists.
2. `GetTimelineByVmPublicIdAsync` looks up the VM by public_id (use `.IgnoreQueryFilters()` so soft-deleted VMs still surface their event history), returns `take` events ordered by `OccurredAt DESC`.
3. No Update or Delete methods. The service is append-only at the application layer.
4. Register in DI.
5. Build clean.

## Definition of done
Build clean. Service callable from a controller (verified next phase). No mutation methods exposed.
