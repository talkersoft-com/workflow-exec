# Task 0010 — Web: replace force-delete with conditional Retry teardown

## Goal
Remove the force-delete button from the VM detail page. Add a `Retry teardown` button visible only when the VM's most recent event is `teardown_failed`.

## Files
- `cloud-manager-web/src/<vm-detail page>` — remove force-delete button, add conditional retry button.
- API client method for `POST /api/v1/vm/{publicId}/retry-teardown` if not already present.

## Steps
1. Delete the force-delete button + its handler.
2. Add the API client call: `retryVmTeardown(publicId) → VmEvent`.
3. Compute visibility: the button renders only when the timeline's most recent event has `event_type === 'teardown_failed'`. Otherwise: button is not rendered at all (not disabled — hidden).
4. On click, POST to `/retry-teardown`. On 200, refresh the timeline (the new `retry_teardown_requested` event should appear). On 400, show the server's error message.
5. Surface the result inline (no full-page navigation).

## Notes
- The visibility check is purely client-side based on the timeline. The 400 guard on the server is the authoritative check; the UI just doesn't show a button it knows will fail.
- No confirmation modal for v1 — retry is idempotent and audit-logged.

## Definition of done
Force-delete button is gone. Retry button renders only on VMs whose last event is `teardown_failed`. Clicking it adds a `retry_teardown_requested` event to the timeline and triggers a new vorch destroy attempt.
