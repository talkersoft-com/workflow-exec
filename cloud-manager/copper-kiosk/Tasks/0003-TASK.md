# Task 0003 — Blueprint builder

## Goal
Compose, order, and publish blueprints entirely from the UI.

## Steps
1. BlueprintsPage (`/blueprints`): list all non-deleted blueprints with status badges
   (Draft/Published/Archived) + create form (name, description, image picker from the existing
   images source, default memory/disk using the wizard's option sets).
2. BlueprintDetailPage (`/blueprints/:id`): metadata edit; ordered playbook list — attach from
   existing playbooks, reorder via up/down buttons (decided default, no new dependency), detach;
   per-playbook varsOverride JSON editor reusing the existing assignment vars editor pattern.
3. Lifecycle actions: Publish / Archive buttons (`POST /blueprint/{id}/publish|archive`); Delete
   only when not Published (mirror the API rule; show the 409 reason otherwise).
4. Build green.

## Definition of done
A blueprint can be created, given two ordered playbooks, reordered, published (appears in
`/marketplace`), archived (disappears), and deleted — all from the UI.
