# Task 0001 — vorch-lib: cloud-init ufw baseline (every VM)

## Goal
Make every newly provisioned VM boot SSH-only behind ufw. Append a lockout-safe ufw block to
the cloud-init user-data `runcmd`, in the user-data builder.

## Where
`vorch-lib` — the user-data builder (`InitializeHost` / `buildUserData`). Locate the function
that assembles `runcmd` for the cloud-init user-data. Mirror the existing builder pattern.

## What to add
Append to `runcmd`, **after the netplan apply step**, in this exact lockout-safe order:
```
ufw allow 22/tcp                      # allow SSH BEFORE enabling — never lock out
ufw --force default deny incoming
ufw --force default allow outgoing
ufw --force enable
```

## Constraints
- ufw ships in the Ubuntu cloud image — **no install** step.
- Idempotent; ordering matters (allow 22 first, enable last).
- Established SSH (porch post-boot runs) must survive: 22 is allowed and ufw permits
  established connections — confirm the block does not drop the active session.
- Static in the template → **vorch-service is unchanged** (no new create-vm field, no new
  user-data input). Do not thread a new parameter through.
- No secret/value logging.

## Build
Build vorch per the `vorch-lib` repo's build. The manual hypervisor redeploy happens in
Phase 0003 — here, just ensure it compiles and the unit/golden test passes.

## Definition of done
The generated user-data contains the four ufw lines in the correct order, immediately after
netplan apply, and the empty-secrets user-data is otherwise byte-stable. `Test/0001-TEST.md`
passes. Code compiles.
