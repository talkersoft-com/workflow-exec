# Task 0003 — E2E + build + deploy, then integrate

## Goal
Prove the firewall works end-to-end on real VMs, deploy the new vorch baseline + the synced
seed playbooks, write the result + lessons, then integrate.

## Steps

### 1. Build & redeploy vorch (manual, on the hypervisor)
Build vorch from `vorch-lib` and **manually redeploy it on the hypervisor** — this is the only
way the new cloud-init baseline reaches future VMs. **Record the exact build + redeploy command
in `Results/RESULT.md`.**

### 2. Confirm the seed → DB sync from Task 0002
Re-run / confirm `import-playbooks.py` reported PATCH for the pg + rabbit playbooks and that
the DB content matches the seed YAML (`cloud_playbook_get` by name, diff against the seed).

### 3. E2E — plain VM
Provision a **plain** VM (use `cloud_vm_create` + `cloud_orchestration_provision`).
- SSH (`22`) reachable.
- All other inbound blocked — checked from the **off-subnet Mac**.

### 4. E2E — Postgres VM
Provision a VM with the Postgres playbook attached.
- SSH ok.
- `5432` reachable from an **on-subnet peer** VM.
- `5432` blocked from the **off-subnet Mac**.
- Confirm the DB playbook content matches the seed YAML.

### 5. Confirm no existing VM changed
Verify no already-running VM was modified (baseline is cloud-init first-boot only; check
`cloud_vm_events` / state of pre-existing VMs).

### 6. Tear down the throwaway E2E VMs
Destroy the plain + Postgres test VMs created above (`cloud_orchestration_destroy`).

### 7. Write outputs (BEFORE hv_integrate)
- `Results/RESULT.md` — outcome, branch, PR table (TBD until integrate), per-phase summary,
  the exact manual vorch redeploy command, the E2E reachability matrix, import-playbooks output.
- `Retro/LESSONS.md` — what would have helped; list any FIX files written.

### 8. Integrate
Run `hv_integrate` (see `../deck.md`) with a clear message + title. Only `vorch-lib` and
`cloud-manager-api` carry code changes; `workflow-exec` carries the scaffold + results. PRs are
queued for auto-merge — watch for the transition confirmation; verify with `gh pr list` /
`hv_list_pulls` if the PR lines are omitted.

## Acceptance criteria (verify — never ask)
- Plain VM: SSH reachable; all other inbound blocked from the off-subnet Mac.
- Postgres VM: SSH ok; `5432` reachable from an on-subnet peer; `5432` blocked from off-subnet.
- `import-playbooks.py` reports the pg + rabbit playbooks updated; DB matches seed YAML.
- No already-running VM was modified.

## Definition of done
All acceptance criteria met, outputs written, `Test/0003-TEST.md` passes, `hv_integrate`
completed and PRs surfaced.
