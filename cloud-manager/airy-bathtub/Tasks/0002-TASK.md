# Task 0002 — Seed playbooks: per-service subnet rules

## Goal
Each service playbook opens only its own port, to the trusted subnet only. Edit the seed YAML
(the source of truth), then sync to the live DB.

## Trusted sources
`10.0.150.0/24` + `10.0.144.140/32` — reuse the Postgres playbook's `pg_hba_sources` var
(do not hardcode a second copy; loop over the existing var).

## Edits — `cloud-manager-api/seed/playbooks/`

### Postgres — ADD a ufw task
Files: `pg-14-jammy.yaml`, `pg-16-noble.yaml` (currently no ufw task).
Add:
```yaml
- name: Allow Postgres from trusted sources
  community.general.ufw:
    rule: allow
    from_ip: "{{ item }}"
    to_port: "5432"
    proto: tcp
  loop: "{{ pg_hba_sources }}"
```
Keep `listen_addresses='*'` and the existing `pg_hba_sources` — do NOT revert to localhost-only.

### RabbitMQ — TIGHTEN the existing ufw task
Files: `rabbitmq-jammy.yaml`, `rabbitmq-noble.yaml` (already open `5672` + `15672` to anywhere).
Add `from_ip: "{{ item }}"` looped over the trusted sources so both ports are subnet-only
instead of open to anywhere. Match the variable name the rabbit playbooks already use for the
trusted sources (or introduce `ufw_sources` with the identical two values if none exists).

### Keep `.meta.json` in sync
If the edited playbook content is tracked alongside a `.meta.json`, update it as the existing
playbooks do (so `import-playbooks.py` matches by name and PATCHes correctly).

## Sync to the DB
Run:
```
python3 cloud-manager-mcp/.cicd/import-playbooks.py
```
against the live API. It matches by name → PATCH if content differs / no-op if same. Confirm
the pg + rabbit playbooks report **updated** (PATCH), not created. Never hand-edit the DB via
`cloud_playbook_update` — it drifts from git and is overwritten on the next import.

## Definition of done
Both pg seeds have the ufw 5432→sources task; both rabbit seeds loop `from_ip` over the
trusted sources for 5672 + 15672; `import-playbooks.py` reports PATCH/no-op for these
playbooks; the DB content matches the seed YAML. `Test/0002-TEST.md` passes.
