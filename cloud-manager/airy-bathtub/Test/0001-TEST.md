# Test 0001 — vorch-lib cloud-init ufw baseline

Golden / unit test of the generated user-data from the user-data builder
(`InitializeHost` / `buildUserData`).

## TC-001 — ufw block present
Generate user-data and inspect `runcmd`.
Pass: all four lines present, exactly:
- `ufw allow 22/tcp`
- `ufw --force default deny incoming`
- `ufw --force default allow outgoing`
- `ufw --force enable`
Fail: any line missing or altered.

## TC-002 — Ordering correct
Pass: the four ufw lines appear **in the above order**, and **after** the netplan apply step
in `runcmd`. `ufw allow 22/tcp` precedes `ufw --force enable`.
Fail: enable before allow-22, or the block is placed before netplan apply.

## TC-003 — No install step
Pass: there is no `apt install ufw` / package-install of ufw added (it ships in the image).
Fail: an install step was added.

## TC-004 — Empty-secrets path byte-stable except the ufw block
Generate user-data for the empty-secrets case before and after the change.
Pass: the only difference is the added ufw block — everything else is byte-identical.
Fail: any unrelated drift in the generated user-data.

## TC-005 — vorch-service unchanged
Pass: no new create-vm field / user-data input was threaded through vorch-service; the rule is
static in the template.
Fail: a new parameter or field was added.

## TC-006 — Compiles / unit tests pass
Pass: vorch-lib builds and its unit/golden tests pass.
Fail: build or test failure.
