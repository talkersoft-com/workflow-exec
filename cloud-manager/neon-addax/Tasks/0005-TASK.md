# Task 0005 — API Phase 4: PlaybookGlobalRoleRef + ArgumentSpecsTranslator hook

## Goal
Attach/detach global ansible roles to playbooks. Wire the existing
`ArgumentSpecsTranslator` so that saving `meta/argument_specs.yml` in a
global role propagates to every playbook that references it via
`PlaybookGlobalRoleRef`.

## Target repo
`cloud-manager-api`

## Reference
- `cloud-manager/planning/planning/cloud-manager-ansible/API-PLAN.md` — sections
  "Playbook ↔ global role refs" and "Service-layer hooks → argument-specs hook"
- `src/Services/CloudManager.Data.Services/Playbooks/ArgumentSpecsTranslator.cs` — already exists, unused; wire it in

## What to build

### DTOs
- `PlaybookGlobalRoleRef.cs` — `{ publicId, playbookId, ansibleRoleId, ansibleRoleName, created }` (joined with AnsibleRole for name)

### Service
- `IPlaybookGlobalRoleRefService` + impl — list per playbook, attach (rejects duplicate `(playbook, role)` with 409), detach by ref publicId.

### Hook wiring
- In `RoleFileService.UpdateAsync` (after the revision insert from Task 0003): if `file_path == "meta/argument_specs.yml"`, in the same transaction:
  - Find every `Playbook` with a `PlaybookGlobalRoleRef` pointing at this role's `AnsibleRole`.
  - For each such playbook, gather all relevant `meta/argument_specs.yml` content (this role's NEW content + every other global role still referenced + any playbook-local roles' specs).
  - Call `ArgumentSpecsTranslator.Translate(...)` (use the existing API; do not modify the translator).
  - Update `Playbook.argument_schema` with the new derived JSON.
- In `PlaybookRoleFileService.UpdateAsync` (added in Task 0006 but the same shape): the local equivalent — only the owning playbook is affected.
- On attach/detach in `PlaybookGlobalRoleRefService`: same re-derive for the affected playbook(s).

### Controller
- `PlaybookGlobalRoleRefController` at `[Route("api/v1/playbook/{playbookId}/globalRole")]`:
  - GET (list attached refs)
  - POST `{ ansibleRoleId }` (attach; 409 on duplicate)
  - DELETE/{refId}

## Steps
1. Build DTOs + service + controller + hook wiring.
2. Register in `ConfigureServices.cs`.
3. `dotnet build` — must exit 0.
4. Smoke-test:
   - Create a global role `r1` with file `meta/argument_specs.yml` containing one option (e.g., `foo: {type: str}`).
   - Create a playbook `p1`.
   - POST `/api/v1/playbook/{p1}/globalRole` `{ ansibleRoleId: r1 }` → 201; `p1.argument_schema` now includes `foo`.
   - POST same again → 409.
   - PATCH `r1`'s `meta/argument_specs.yml` to add `bar: {type: int}` → 200; `p1.argument_schema` now includes `bar` too.
   - DELETE the ref → 204; `p1.argument_schema` no longer includes `foo`/`bar`.
5. Verify by querying `playbooks.argument_schema` via the database-toolkit MCP after each step.
6. Clean up.

## Definition of done
Attach/detach work. Editing `meta/argument_specs.yml` in a globally-attached
role updates every referencing playbook's `argument_schema` in the same
transaction. `dotnet build` passes.
