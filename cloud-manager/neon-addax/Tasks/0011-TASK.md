# Task 0011 — Write Results + Retro/LESSONS, then ship

## Goal
Record outcomes and ship the implementation. After this task, two PRs exist
simultaneously: the `cloud-manager-api` PR (the actual implementation) and
the `execution-workflow` PR (Results + Lessons in this workflow folder).

## Steps
1. Write `Results/RESULT.md` (per the convention)
2. Write `Retro/LESSONS.md`
3. Run `hv_ship` per `../deck.md`. This single `hv_ship` call WILL create
   two PRs because two repos are dirty: `cloud-manager-api` and
   `execution-workflow`. That is expected — do NOT panic and try to skip one.

## Definition of done
`hv_ship` output shows both the `cloud-manager-api` PR merged and the
`execution-workflow` PR merged. All other deck repos skipped (clean / nothing
ahead).
