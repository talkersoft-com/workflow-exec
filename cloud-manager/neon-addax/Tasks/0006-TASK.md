# Task 0006 — API Phase 5: playbook-local roles full surface

## Goal
Full CRUD on `PlaybookRole`, `PlaybookRoleFile`, and `PlaybookRoleFileRevision`,
scoped under a playbook. Mirrors the global-role shape exactly, with the
argument-specs hook firing for the owning playbook only.

## Target repo
`cloud-manager-api`

## Reference
- `cloud-manager/planning/planning/cloud-manager-ansible/API-PLAN.md` — section "Playbook-local roles"

## What to build

### DTOs
- `PlaybookRole.cs` — `{ publicId, playbookId, name, description, fileCount, created, modified }`
- `PlaybookRoleFile.cs` — same shape as `RoleFile.cs`
- `PlaybookRoleFileSummary.cs`
- `PlaybookRoleFileRevision.cs` — same shape as `RoleFileRevision.cs`
- `PlaybookRoleFileRevisionSummary.cs`

### Services
- `IPlaybookRoleService` + impl — list per playbook, get, create (name unique per playbook), update, delete (cascades files + revisions).
- `IPlaybookRoleFileService` + impl — mirrors `RoleFileService` exactly, scoped to a playbook-local role. Reuses `RoleFilePathValidator`. Auto-revision on PATCH. Argument-specs hook fires for THIS playbook only when `meta/argument_specs.yml` changes.
- `IPlaybookRoleFileRevisionService` + impl — list / get / restore.

### Controllers
- `PlaybookRoleController` at `[Route("api/v1/playbook/{playbookId}/role")]`
- `PlaybookRoleFileController` at `[Route("api/v1/playbook/{playbookId}/role/{roleId}/file")]`
- `PlaybookRoleFileRevisionController` at `[Route("api/v1/playbook/{playbookId}/role/{roleId}/file/{fileId}/revision")]`

All carry `[RequireFeatureFlag("playbooks")]`.

## Steps
1. Build all DTOs / services / controllers per the spec.
2. Register all three services in `ConfigureServices.cs`.
3. `dotnet build` — must exit 0.
4. Smoke-test:
   - Create a playbook `p2`.
   - Create a local role `lr1` under `p2`.
   - Create a file `tasks/main.yml` in `lr1` → 201.
   - PATCH content → 200; one revision row appears.
   - Create `meta/argument_specs.yml` in `lr1` with one option → 201; `p2.argument_schema` updated.
   - Edit `meta/argument_specs.yml` to add another option → 200; `p2.argument_schema` updated.
   - DELETE `lr1` → 204; all local-role files + revisions cascade-deleted.
5. Verify that editing a local role's `meta/argument_specs.yml` does NOT touch any OTHER playbook's argument_schema.
6. Clean up.

## Definition of done
All three local-role controllers respond correctly. Auto-revision works. The
argument-specs hook fires for the owning playbook only. Cascade delete works.
`dotnet build` passes.
