# Task 0002 — Registry API (cloud-manager-api repo)

## Steps
1. CRUD: `api/v1/variable` (list/get/create/update/delete) + `api/v1/variable/{id}/secretRef`
   PUT/DELETE, all gated `ansible-studio`. Wire shape is C-VAR verbatim (contracts §7) —
   field names must not drift; P5 and P8 consume this shape.
2. Service validation mirrors the DB CHECK: `sensitivity: "secret"` ⇒ `defaultValue` must be
   null; reject otherwise with a clear 4xx, never silently drop the value.
3. Classification authority (PLAN.md open question 3 default): flipping `sensitivity` to
   `secret` is one-way; declassification requires an explicit force flag. Both directions
   recorded as `varev` events (`classified_secret` on the flip; an `updated` event with the
   force marker on declassify).
4. Events: every mutation records the matching `varev` type (`created`, `updated`,
   `deleted`, `secret_ref_set`, `secret_ref_removed`) — best-effort try/catch, never
   blocking the primary write (project audit-log convention).
5. SecretRef writes validate VaultPath as free-text with required prefix `cloudmanager/`
   (PLAN.md open question 1 default — picker is P9 UX).
6. Harvest assist: `POST api/v1/variable/harvest?owner=…` reads P2's decomposer output
   (`defines`/`consumed_by` var names) and proposes unregistered names for one-click
   registration, kind defaulted `internal`. Harvest NEVER auto-classifies sensitivity.
7. camelCase JSON, public ids only; ids resolved via the existing
   `PublicIdByGuidAsync`/`GuidByPublicIdAsync` helpers.
8. Build: `python3 .cicd/build-cloud-manager.py --target api` must pass.

Done when: CRUD + secretRef + harvest endpoints live and gated, C-VAR shape verbatim,
events recorded on every mutation, api build clean.
