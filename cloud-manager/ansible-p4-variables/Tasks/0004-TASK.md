# Task 0004 — Required enforcement (cloud-manager-api repo)

## Steps
1. `GET api/v1/playbook/{pid}/requiredVars?inventoryId=&hostId=` (gated `ansible-studio`):
   compute the required `vdef`s for the playbook's role closure — closure walked via P2
   `includes`/`depends_on` edges — minus those satisfied by any override or var layer
   (role/playbook defaults, P1 group vars by nesting, host vars, assignment override,
   blueprint override). Return the missing list in C-VAR record shape.
2. A `$secretRef` value satisfies a required variable; a registered definition with only a
   null default does not.
3. Run trigger (`POST …/run`) gains optional `"enforceRequired": true` — additive, default
   false this plan (flipping the default is a P5/P6 open question once gap detection is
   fully wired). When true and the gap list is non-empty, the run is rejected with the gap
   list; when false or omitted, behavior is byte-identical to today.
4. Query shape: closure + var layers batched per request — no N+1 over hosts or roles.
5. camelCase, public ids only.
6. Build: `python3 .cicd/build-cloud-manager.py --target api` must pass.

Done when: requiredVars answers "what's missing?" correctly over the role closure for a
concrete inventory/host context, enforceRequired gates the run trigger additively, api
build clean.
