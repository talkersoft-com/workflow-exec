# Task 0005 — MCP + regression (cloud-manager-mcp repo, api tests)

## Steps
1. New module `variables.ts` in cloud-manager-mcp (contracts §4):
   `cloud_variable_list`, `cloud_variable_get`, `cloud_variable_create`,
   `cloud_variable_update`, `cloud_variable_delete`, `cloud_variable_secret_ref_set`.
   Read/mutate separation per the existing profile convention; ids via `idSchema`;
   register in the profile/tool index.
2. The standing intentional exclusion stays: `GET /run/{runId}/secret` is NOT exposed as an
   MCP tool, and no new tool returns Vault cleartext or VaultPath secret values beyond what
   C-VAR carries (`secretRef.vaultPath` is a path, never a secret value).
3. Cleartext-rejection tests at the MCP boundary: `cloud_variable_create`/`update` with
   `sensitivity: "secret"` + a `defaultValue` surfaces the API rejection; override mutation
   via existing tools (e.g. `cloud_vm_playbook_update`) with cleartext on a
   secret-classified name surfaces the 409.
4. Regression: existing run flows with NO typed variables are byte-identical — trigger a
   plain playbook run and a blueprint-sequencer run against fixtures with zero `vdef` rows;
   compare materialized payloads and run behavior to pre-P4 baseline.
5. Build: `python3 .cicd/build-cloud-manager.py --target mcp` must pass (and `--target api`
   if regression fixes touched the API).

Done when: variables.ts tools registered and exercised against the API, exclusions intact,
cleartext rejection surfaced end-to-end, untyped-var regression green, mcp build clean.
