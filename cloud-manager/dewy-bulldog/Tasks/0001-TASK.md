# Task 0001 — Config Struct: PRMode Enum + Backward Compat

## Goal

Replace `AutoMerge bool` and `RequireMergedPR bool` in the `Ship` struct with `PRMode` enum. Add `MergePollInterval` and `MergePollTimeout` duration fields. Add custom `UnmarshalYAML` so old configs with `auto_merge: true` keep working.

## Target repo

`hive-deck-pro` — `internal/config/config.go`

## Steps

1. Add `PRMode` type and constants above the `Ship` struct:
```go
type PRMode string

const (
    PRModeAutoMerge  PRMode = "auto_merge"
    PRModeAwaitMerge PRMode = "await_merge"
    PRModeManual     PRMode = "manual"
)
```

2. Replace `Ship` struct fields:
   - Remove: `AutoMerge bool`, `RequireMergedPR bool`
   - Add: `PRMode PRMode yaml:"pr_mode"`, `MergePollInterval time.Duration yaml:"merge_poll_interval"`, `MergePollTimeout time.Duration yaml:"merge_poll_timeout"`
   - Keep: `DeleteBranchOnMerge`, `TeardownOnShip`, `AutoDefaultBranch`

3. Add custom `UnmarshalYAML` on `*Ship` that reads a raw map, sets `PRMode` from `pr_mode` if present, otherwise falls back to deriving it from `auto_merge` + `require_merged_pr`:
   - `auto_merge: true` → `PRModeAutoMerge`
   - `auto_merge: false, require_merged_pr: true` → `PRModeManual`
   - neither / both false → `PRModeAutoMerge` (safe default)
   - empty `pr_mode` field → `PRModeAutoMerge`

4. Add `"time"` import if not present.

5. Fix any call sites in `ops/ship.go` or elsewhere that reference `l.Setup.Ship.AutoMerge` or `l.Setup.Ship.RequireMergedPR` — update to use `l.Setup.Ship.PRMode` (stub the switch with TODOs for now; Phase 2 fills them in).

6. `CGO_ENABLED=0 go build ./...` — must pass.

## Definition of done

`go build ./...` passes. `PRMode` type and constants exist. Old `auto_merge: true` in config.yaml still parses without error.
