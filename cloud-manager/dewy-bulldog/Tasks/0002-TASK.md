# Task 0002 ‚Äî ops/ship.go: Dispatch on pr_mode

## Goal

Replace the `AutoMerge` boolean checks in `RunShip` with a switch on `PRMode`. All three modes produce correct behavior. Output format unchanged for `auto_merge` and `manual` modes.

## Target repo

`hive-deck-pro` ‚Äî `ops/ship.go`

## Steps

1. Replace the current `if l.Setup.Ship.TeardownOnShip && l.Setup.Ship.AutoMerge` / `if l.Setup.Ship.RequireMergedPR && !l.Setup.Ship.AutoMerge` logic with a switch:

```go
switch l.Setup.Ship.PRMode {
case config.PRModeAutoMerge, "":
    // auto_merge: open PR with auto-merge flag, then checkout next branch
    if l.Setup.Ship.TeardownOnShip {
        return "", teardown.Run(l, teardown.Options{RequireMergedPR: false})
    }
    nextBranch := namegen.Generate()
    return "", checkout.Run(l, checkout.Options{
        RequireMergedPR: false,
        NextBranch:      nextBranch,
    })

case config.PRModeAwaitMerge:
    // open PRs normally, return immediately ‚Äî caller uses RunAwaitMerge to poll
    return "\nawait_merge: PRs opened ‚Äî use hv_await_merge to poll until merged", nil

case config.PRModeManual:
    // open PRs and stop ‚Äî user merges and runs hv next
    return "\nmerge-gate: PRs opened ‚Äî merge them, then run hv next to transition", nil
}
```

2. Pass the `AutoMerge` option to `ship.Run` only for `PRModeAutoMerge` (the GitHub auto-merge flag on the PR itself).

3. `CGO_ENABLED=0 go build ./...` and `CGO_ENABLED=0 go test ./...` must pass.

4. Smoke test: `hv ship cloud-manager "test" --title "test"` should behave identically to before (deck is on `pr_mode: auto_merge`).

## Definition of done

Build and tests pass. Switch covers all three modes. `auto_merge` output identical to before.
