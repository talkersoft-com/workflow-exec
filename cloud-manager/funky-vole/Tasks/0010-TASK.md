# Task 0010 — Web: AnsibleRunPage blocks Run when requirements unmet

## Goal
On the Run page, after the operator selects a playbook, fetch its requirements; compute against the selected controller's host_collections; if any are unmet, disable the Run button and render an inline error naming the missing collection(s) with a link to Collections.

## Target repo(s)
`cloud-manager-web`

## Steps

1. **`AnsibleRunPage.tsx`** additions:
   - When the user picks a playbook from the dropdown, dispatch `fetchForPlaybook(pid)` (from Phase 9's slice).
   - Compute:
     ```ts
     const reqs = useAppSelector(state => selectRequirementsForPlaybook(state, selectedPlaybookId));
     const installedSet = new Set(hostCollections.filter(h => h.installStatus === 'Installed').map(h => h.collectionId));
     const missing = reqs.filter(r => !installedSet.has(r.collectionId));
     ```
   - When `missing.length > 0`:
     - Disable the Run button.
     - Render `<div style={{ color: 'var(--color-error)' }}>Run blocked: missing required collection(s): {missing.map(m => m.collectionNamespace + '.' + m.collectionName).join(', ')}. <Link to="/ansible/collections">Install in Collections</Link></div>`.

2. **TypeScript + build**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-web
   npx tsc --noEmit
   npm run build
   ```
   Both clean.

3. Write `Retro/CHANGE-0010.md`.

## Definition of done
- Run button disables and inline message renders when requirements unmet.
- Builds clean.
- `Retro/CHANGE-0010.md` exists.
