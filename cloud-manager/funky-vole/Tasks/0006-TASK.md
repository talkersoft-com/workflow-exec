# Task 0006 — Enforcement in TriggerMultiVmAsync + 400 mapping

## Goal
Before mint+publish in `TriggerMultiVmAsync`, check requirements against the chosen controller's `host_collections`. On any miss, throw `RequirementsUnmetException`. Controller maps that to a 400 with structured `missing` payload.

## Target repo(s)
`cloud-manager-api`

## Steps

1. **`RequirementsUnmetException`** new class in `src/Models/CloudManager.Entities/Exceptions/` (or `Services/CloudManager.Data.Services/`):
   ```csharp
   public record MissingRequirement(string CollectionId, string Namespace, string Name, string? MinVersion, string? MaxVersion, string Reason);
   public class RequirementsUnmetException : Exception
   {
       public IReadOnlyList<MissingRequirement> Missing { get; }
       public RequirementsUnmetException(IReadOnlyList<MissingRequirement> missing) : base("Requirements unmet") { Missing = missing; }
   }
   ```

2. **`SatisfiesVersion(string? installed, string? min, string? max)`** in `PlaybookRunService`:
   - Confirm `Semver` NuGet is referenced (if not, add `<PackageReference Include="Semver" Version="2.*" />` to `CloudManager.Data.Services.csproj`).
   - Returns true when `installed` is null and bounds are null (degenerate case).
   - Else: parse installed; check `installed >= min` (if min present) and `installed <= max` (if max present).
   - On parse failure of `installed`: return true (operator fault, don't block; document in the same file).

3. **Enforcement block** at the start of `TriggerMultiVmAsync`:
   ```csharp
   var requirements = await _context.PlaybookCollectionRequirements
       .Where(r => r.PlaybookId == playbook.Id).ToListAsync();
   if (requirements.Count > 0) {
       var controllerId = await _hostService.ResolveControllerHostIdAsync();
       var installed = await _context.HostAnsibleCollections
           .Where(h => h.HostId == controllerId && h.InstallStatus == CollectionInstallStatus.Installed)
           .ToDictionaryAsync(h => h.CollectionId);
       var missing = new List<MissingRequirement>();
       foreach (var req in requirements) {
           var coll = await _context.AnsibleCollections.AsNoTracking().FirstOrDefaultAsync(c => c.Id == req.CollectionId);
           var ns = coll?.Namespace ?? "";
           var name = coll?.Name ?? "";
           var cid = coll?.PublicId ?? req.CollectionId.ToString();
           if (!installed.TryGetValue(req.CollectionId, out var hc)) {
               missing.Add(new MissingRequirement(cid, ns, name, req.MinVersion, req.MaxVersion, "not installed"));
               continue;
           }
           if (!SatisfiesVersion(hc.Version, req.MinVersion, req.MaxVersion)) {
               missing.Add(new MissingRequirement(cid, ns, name, req.MinVersion, req.MaxVersion, $"installed version {hc.Version} out of range"));
           }
       }
       if (missing.Count > 0) throw new RequirementsUnmetException(missing);
   }
   ```
   This block runs **before** the run row is inserted — no half-state.

4. **Controller mapping** in `PlaybookController.TriggerRun`:
   - Catch `RequirementsUnmetException ex` → return `BadRequest(new { error = "requirements_unmet", missing = ex.Missing })`.

5. **Build**: `dotnet build CloudManager.sln -c Release` clean.

6. Write `Retro/CHANGE-0006.md`.

## Definition of done
- Exception class + service helper + controller mapping all present.
- Enforcement runs before run-row insert.
- `dotnet build` clean.
- `Retro/CHANGE-0006.md` exists.
