# Task 0001 — Migration: vault_run_token + hosts.is_controller + backfill

## Goal
EF Core migration that adds the two new columns required by Phases A and C, applies cleanly to clouddb.

## Target repo(s)
`cloud-manager-api`

## Steps

1. **Update the `PlaybookRun` entity** at `src/Models/CloudManager.Entities/Models/PlaybookRun.cs`:
   - Add `public string? VaultRunToken { get; set; }`

2. **Update the `Host` entity** at `src/Models/CloudManager.Entities/Models/Host.cs`:
   - Add `public bool IsController { get; set; }`

3. **DbContext bindings** in `src/Models/CloudManager.Entities/CloudManagerDbContext.cs`:
   - In the `Host` entity block: `entity.Property(e => e.IsController).HasColumnName("is_controller").HasDefaultValue(false).IsRequired();`
   - In the `PlaybookRun` entity block: `entity.Property(e => e.VaultRunToken).HasColumnName("vault_run_token").HasMaxLength(256).IsRequired(false);`

4. **Generate the migration**:
   ```
   set -a; source <(sudo grep -E "^(DB_|RABBITMQ_)" /etc/cloud-manager/cloud-manager-config.env); set +a
   export DBHOST=$DB_HOST DBUSER=$DB_USER DBPASSWD="$DB_PASSWORD" DBPORT=$DB_PORT DATABASE=$DB_NAME RABBITMQ_HOST=$RABBITMQ_HOST RABBITMQ_USER=$RABBITMQ_USER RABBITMQ_PASSWORD=$RABBITMQ_PASSWORD
   cd vm-infra/cloud-manager/cloud-manager-api/src/Models/CloudManager.Entities
   dotnet ef migrations add AddVaultRunTokenAndControllerHost -s ../../CloudManager.API
   ```

5. **Add the backfill** at the bottom of the generated `Up()`:
   ```csharp
   migrationBuilder.Sql("UPDATE bare_metal.hosts SET is_controller = true WHERE name = 'ubuntu-server';");
   ```

6. **Build**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src && dotnet build CloudManager.sln -c Release
   ```
   Expect 0 errors.

7. **Apply**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src/Models/CloudManager.Entities
   dotnet ef database update -s ../../CloudManager.API
   ```

8. Write `Retro/CHANGE-0001.md`.

## Definition of done
- Entity + DbContext updated for both fields.
- Migration generated + applied.
- `\d vm.playbook_runs` shows `vault_run_token` column.
- `\d bare_metal.hosts` shows `is_controller` column.
- `SELECT name, is_controller FROM bare_metal.hosts WHERE name='ubuntu-server';` returns `true`.
- `Retro/CHANGE-0001.md` exists.
