# Task 0003 — API mint + revoke + sweeper + DTO scrub

## Goal
`TriggerMultiVmAsync` mints the scoped child token, persists it on the run row, publishes AMQP. Worker callback PATCH + `CancelAsync` revoke on terminal status. Background sweeper handles orphaned tokens. DTO surfaces token only on `GetByIdAsync`; list endpoints scrub.

## Target repo(s)
`cloud-manager-api`

## Steps

1. **PlaybookRun DTO + Mapper**:
   - Add `public string? VaultRunToken { get; set; }` to `DtoModel.PlaybookRun`.
   - `PlaybookProfile.cs`: `.ForMember(d => d.VaultRunToken, o => o.MapFrom(s => s.VaultRunToken))`.

2. **`TriggerMultiVmAsync` mint** (per PLAN.md A.1):
   - After `_context.SaveChangesAsync()` (the existing one that persists run + targets), before `_publisher.EnqueueRun(...)`:
     - `var prefix = $"cloudmanager/data/playbook-secrets/{run.PublicId}";`
     - `var policyName = $"playbook-run-{run.PublicId}";`
     - `await _vaultClient.WritePolicyAsync(policyName, BuildRunPolicy(run.PublicId));`
     - `var child = await _vaultClient.CreateChildTokenAsync(policies: [policyName], ttl: TimeSpan.FromHours(1), displayName: policyName, noParent: true, renewable: false);`
     - `run.VaultSecretsPrefix = prefix;`
     - `run.VaultRunToken = child.ClientToken;`
     - `await _context.SaveChangesAsync();` (second save for the token columns)
     - Then publish.
   - On `VaultException` from any of those calls: catch in the calling controller (`PlaybookController.TriggerRun`) and return `503` with `{ Message = "Vault unavailable, try again" }`.

3. **`BuildRunPolicy(string publicId)`** — private static method on `PlaybookRunService` returning the exact HCL from PLAN.md A.1.

4. **List-endpoint scrub**: in `ListByAssignmentAsync` (and any other method returning lists of `PlaybookRun` DTO), iterate the results and set `dto.VaultRunToken = null` before return. Add a comment explaining why.

5. **`GetByIdAsync` keeps token**: confirm the existing implementation doesn't scrub; this is the one endpoint vorch hits via `fetchRun`.

6. **`RevokeRunVaultAsync(PlaybookRun run)`** — private async method per PLAN.md A.4. Idempotent: silent return if `VaultRunToken` is empty; log-warn-and-return on Vault errors so the sweeper can retry.

7. **Worker callback PATCH**: in `PlaybookRunsController.Patch` (or wherever the worker callback flips terminal status), after the SaveChanges, call `RevokeRunVaultAsync(run)` when the new status ∈ {Succeeded, Failed, Cancelled}.

8. **`CancelAsync` revoke**: in `PlaybookRunService.CancelAsync`, after the cancel transition is persisted, call `RevokeRunVaultAsync(run)`.

9. **`VaultRunTokenSweeper`** — new `BackgroundService` in `cloud-manager-api/src/CloudManager.API/HostedServices/VaultRunTokenSweeper.cs`:
   - 30-minute timer.
   - Query: `_context.PlaybookRuns.Where(r => r.VaultRunToken != null && r.Status != Succeeded && r.Status != Failed && r.Status != Cancelled && r.StartedAt < DateTime.UtcNow.AddHours(-2)).ToListAsync()`.
   - For each: mark `Status = Failed`, `ErrorMessage = "vault-sweep: run exceeded 2h timeout"`, save, then `RevokeRunVaultAsync(run)`.
   - Register in `Program.cs`: `builder.Services.AddHostedService<VaultRunTokenSweeper>();`

10. **Build**: `dotnet build CloudManager.sln -c Release` clean.

11. Write `Retro/CHANGE-0003.md`.

## Definition of done
- DTO has `VaultRunToken`, list endpoints scrub, get-by-id passes through.
- `TriggerMultiVmAsync` mints and persists token; the second `SaveChangesAsync` is in place; no AMQP publish on Vault failure.
- `RevokeRunVaultAsync` wired from PATCH callback + CancelAsync, idempotent.
- `VaultRunTokenSweeper` registered as a HostedService.
- `dotnet build` clean.
- `Retro/CHANGE-0003.md` exists.
