# Task 0011 — Build + deploy

## Goal
Migration applied; cloud-manager-api, vorch-service, and cloud-manager-web all running the new code on `ubuntu-server`.

## Target repo(s)
`cloud-manager-api`, `vorch-service`, `cloud-manager-web` (binaries + bundle).

## Steps

1. **Migration apply** (idempotent — skip if already applied during Phase 1):
   ```
   export DBHOST=localhost DBUSER=cloudmanager DBPASSWD='P@ssw0rd!' DBPORT=5432 DATABASE=clouddb RABBITMQ_HOST=localhost RABBITMQ_USER=cloudmanager RABBITMQ_PASSWORD='P@ssw0rd!'
   cd vm-infra/cloud-manager/cloud-manager-api/src/Models/CloudManager.Entities
   dotnet ef database update -s ../../CloudManager.API
   ```

2. **cloud-manager-api**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api
   sudo python3 scripts/api/install-api-service.py
   sudo systemctl is-active cloud-manager-api
   ```

3. **vorch-service**:
   ```
   sudo python3 scripts/vorch/install-vorch-service.py
   sudo systemctl is-active vorch_service
   ```

4. **cloud-manager-web**:
   ```
   sudo rm -rf vm-infra/cloud-manager/cloud-manager-web/dist
   cd vm-infra/cloud-manager/cloud-manager-api
   sudo python3 scripts/web/install-web-app.py
   sudo systemctl is-active cloud-manager-web
   ```

5. **Smoke checks**:
   - `curl -sk https://ubuntu-server.talkersoft.com/api/v1/host/list | python3 -m json.tool | grep -E "isController|name"` shows the new field.
   - `curl -sk https://ubuntu-server.talkersoft.com/api/v1/ansible/playbooks/pb_EMEJCH08GG/collection-requirements` returns `[]` (empty list).
   - `sudo grep -l "isController" /opt/cloud-manager-web/assets/*.js | head` finds a bundle.

## Definition of done
- Migration applied.
- All three services `active`.
- Smoke checks above succeed.
