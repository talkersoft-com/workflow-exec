# Task 0004 ŌĆö Vorch forwards token to ansible-runner

## Goal
Vorch's playbook-run subprocess receives `VAULT_TOKEN` + `VAULT_ADDR` in its env when the run row carries a non-empty `vaultRunToken`.

## Target repo(s)
`vorch-service`

## Steps

1. **`runDTO` addition** at `vorch-service/vorch-service/handlers/playbook_run_handler.go`:
   ```go
   type runDTO struct {
       ID                 string `json:"id"`
       PlaybookID         string `json:"playbookId"`
       PlaybookRevisionID string `json:"playbookRevisionId,omitempty"`
       Status             string `json:"status"`
       VaultRunToken      string `json:"vaultRunToken,omitempty"`
   }
   ```

2. **Env injection** in `playbook_run_exec.go`'s `executeOneTarget`:
   - Before the `exec.CommandContext(...)` call: build `env := os.Environ()`. If `run.VaultRunToken != ""`, append `"VAULT_TOKEN="+run.VaultRunToken` and `"VAULT_ADDR="+serverCfg.Vault.Address`.
   - Set `cmd.Env = env`.
   - **Do NOT log the token value.** It's fine to log `VAULT_TOKEN injected for run %s` without the value.

3. **Build verification**:
   ```
   cd vm-infra/cloud-manager/vorch-service/vorch-service
   go build ./...
   go vet ./...
   ```
   Expect no errors.

4. Write `Retro/CHANGE-0004.md`.

## Definition of done
- `runDTO` has `VaultRunToken` field with camelCase JSON tag.
- `executeOneTarget` sets the two env vars on the subprocess only when `vaultRunToken` is non-empty.
- `go build` + `go vet` clean.
- `Retro/CHANGE-0004.md` exists.
