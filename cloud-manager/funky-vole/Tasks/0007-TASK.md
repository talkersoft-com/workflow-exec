# Task 0007 — Host.IsController plumbing + ResolveControllerHostIdAsync

## Goal
The `Host` entity surfaces `IsController`. The `Host` DTO returns `isController` on every host list/get. `IHostService.ResolveControllerHostIdAsync()` returns the controller host id used by Phase 6.

## Target repo(s)
`cloud-manager-api`

## Steps

1. **Entity** already has the field from Phase 1.

2. **DTO** at `src/Models/CloudManager.DTO/Models/Host.cs`: add `public bool IsController { get; set; }`.

3. **AutoMapper**: ensure `Host → DtoModel.Host` map carries `IsController` (default mapping should pick it up; explicit `.ForMember(d => d.IsController, o => o.MapFrom(s => s.IsController))` is fine too).

4. **`IHostService.ResolveControllerHostIdAsync()`**:
   - Loads `_context.Hosts.AsNoTracking().Where(h => h.IsController).OrderBy(h => h.Name).FirstOrDefaultAsync()`.
   - If zero rows: throws `InvalidOperationException("no controller host configured")`.
   - Returns the host's internal `Guid` id (not the public_id — Phase 6 uses internal Guid for the join against host_collections).

5. **API list/get** — verify the controller-side endpoints (`HostController.List`, `Get`) return the new field implicitly via DTO mapping. No code change beyond the DTO.

6. **Build**: `dotnet build CloudManager.sln -c Release` clean.

7. Write `Retro/CHANGE-0007.md`.

## Definition of done
- DTO has `IsController`.
- `ResolveControllerHostIdAsync` returns the Guid for `ubuntu-server`.
- `dotnet build` clean.
- `Retro/CHANGE-0007.md` exists.
