# Task 0005 — Perf + regression (cloud-manager-api repo)

## Steps
1. Synthetic-scale perf test (epic §11 — performance is a requirement, not a nicety):
   generate a synthetic inventory of 1k variables across the layers and 200 hosts (the cap
   boundary) with realistic group nesting. Whole-inventory preview (`hostId: null`) must
   complete within the responsive-preview budget — derive the concrete threshold from the
   epic §11 wording during research, record the chosen number IN the test, and gate on it.
2. Verify no N+1: layer inputs are batched per context — assert query count against the
   synthetic inventory is bounded (constant-ish in host count, not linear in vars × hosts).
3. Cap behaviour at scale: 200 hosts resolves; 201 → 413 (boundary test against the
   constant from Task 0002).
4. Regression — flag off: with `ansible-studio` disabled, existing run flows are
   byte-identical to pre-change: trigger a run and compare the porch-bound extravars
   payload, `VarsSnapshot` content, run DTO responses, and event stream against pre-change
   fixtures. The engine must be invisible when disabled (no rsnap capture, no preview, no
   enforce path).
5. Regression — flag on, no opt-ins: flag enabled but no `enforceRequired` and no preview
   calls → run flows behaviourally identical except for additive rsnap capture and additive
   DTO fields.
6. Build: `python3 .cicd/build-cloud-manager.py --target api` must pass with the perf suite
   wired into the normal test run (CI-runnable, no manual steps).

Done when: perf test green at 1k vars / 200 hosts with a recorded threshold, query count
bounded, flag-off byte-identical regression green, api build clean.
