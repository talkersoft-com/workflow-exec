# Task 0002 — Preview endpoint (cloud-manager-api repo)

## Steps
1. C-RES DTOs VERBATIM from contracts §8 — field names are binding:
   `variables[]` with `name`, `state` (`resolved | undefined | missingRequired`),
   `sensitivity` (`plain | secret`), `effectiveValue` (`"***"` literal when masked),
   `winner { precedenceLevel, source, sourcePublicId }`,
   `overridden [{ precedenceLevel, source, sourcePublicId }]`;
   plus `gaps [{ name, kind, reason }]`. camelCase on the wire; public ids only.
2. Controller: `POST api/v1/resolution/preview` with body
   `{ playbookId, inventoryId, hostId, extraVars }` → 200 with the C-RES shape. The
   endpoint is a thin shell over `IVariableResolutionService` — masking decisions live in
   the engine (Task 0001), never re-derived here.
3. Side-effect-free: an idempotent read; POST only because the context body is structured.
   No state writes, no events recorded.
4. Feature-flag gating: `ansible-studio` — flag off → the same not-available behaviour the
   other ansible-studio endpoints use (match P1/P2 gating convention found in research).
5. Per-host matrix mode: `hostId: null` resolves for ALL hosts of the inventory and returns
   a per-host gap matrix (epic §5 "per target"). Cap whole-inventory preview at 200 hosts by
   default → 413 beyond (PLAN.md perf stance; make the cap a constant the perf phase can
   reference).
6. Validation: unknown playbookId/inventoryId/hostId → 404 with the standard error shape;
   hostId not attached to the inventory → 422.
7. Integration tests: resolved/undefined/missingRequired contexts, masked secret round-trip
   through the HTTP layer (response JSON contains `"***"`, never the cleartext), matrix
   mode, 413 cap, flag-off behaviour.
8. Build: `python3 .cicd/build-cloud-manager.py --target api` must pass.

Done when: preview endpoint returns C-RES verbatim for a real context, gated, capped,
side-effect-free, integration tests green, api build clean.
