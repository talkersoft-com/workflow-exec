# Task 0002 — MCP: camelCase guard exemption for `secretBindingParams[].params`

**Repo:** `cloud-manager-mcp` (TypeScript)
**Plan:** Design §#2; phase 0002.

## Goal
Stop `apiRequest`'s recursive camelCase key check from rejecting legitimate
`secretBindingParams[].params` map keys (placeholder names like `vm_public_id`, which must match
the `{vm_public_id}` placeholder). Those keys are free-form **user data**, not wire fields, so
`cloud_marketplace_provision` with `params:{vm_public_id:…}` must not error.

## Steps

### 1. Locate the guard
- Find the recursive camelCase key check in `apiRequest` (config.ts / the request helper) that
  throws on snake_case keys (`snake_case key "vm_public_id" … not allowed`).

### 2. Narrow exemption
- Exempt the **inner `secretBindingParams[].params` map keys** from the check — and only those.
- Implement narrowly: once recursion is under a `secretBindingParams` element, do **not** recurse
  into the `params` object's keys (skip key validation for that object's direct keys).
- The guard MUST still apply to every structural/wire field name — including `secretBindingId`,
  `params` itself, and all other request fields. Only the keys *inside* a `params` object are
  exempt. Do not globally relax the guard.

### 3. Build
- `python3 .cicd/build-cloud-manager.py --target mcp` — must pass clean.
- Note: the running MCP server is not restarted by the build; operator `/mcp` reload is required
  for the running server to accept the previously-rejected calls (surface this in Results).

## Definition of done
`secretBindingParams[].params` keys with snake_case placeholder names pass the guard; all other
snake_case keys (structural/wire fields) are still rejected; mcp builds clean.
`Test/0002-TEST.md` passes.
