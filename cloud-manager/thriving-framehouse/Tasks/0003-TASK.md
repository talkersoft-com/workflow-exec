### 4. E2E — reproduce the cross-VM templated flow THROUGH the MCP
Requires the operator to `/mcp` reload first so the running MCP carries the guard fix.
- Provision a source Postgres VM, then a consumer VM from a templated `{vm_public_id}` binding via
  `cloud_marketplace_provision` passing
  `secretBindingParams:[{secretBindingId, params:{vm_public_id: <source vm public id>}}]`.
- **Pass A:** the call does NOT raise the camelCase guard error (bug #2 fixed).
- **Pass B:** `cloud_vm_secret_binding_list` (and `GET /vm/{publicId}/secret-binding`) for the
  consumer returns the **source** VM's resolved path (e.g. `…/<source_vm_public_id>/postgres`),
  not the consumer's id (bug #3 fixed).
- Secret values never appear in output — only the Vault path.
