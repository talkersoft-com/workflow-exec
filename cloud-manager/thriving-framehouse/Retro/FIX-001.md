# FIX-001 — Migration apply failed: wrong DB password in the documented connection string

## Failure
Task 0003 step 1 / Test 0003 TC-001. Running the migration with the connection string
documented in `Exec.md` / `Tasks/0003-TASK.md`:

```
dotnet ef database update --connection "Host=localhost;User ID=cloudmanager;Password=P@ssw0rd!;Port=5432;Database=clouddb;SslMode=Prefer;Trust Server Certificate=true"
```

failed with:

```
28P01: password authentication failed for user "cloudmanager"
```

## Root cause
`P@ssw0rd!` is the password for **`clouddb_local`** (the port **5433** dev DB), not for the
live **`clouddb`** on port **5432**. The hive db-toolkit config
(`/home/todd/.hv/toolkit/db/config/database.json`) is authoritative:

- `clouddb`        → localhost:**5432**, password `s94bEyloxB5Fho4gPpt6gHFjOyWoaRdN`
- `clouddb_local`  → localhost:**5433**, password `P@ssw0rd!`

`db_test_connection(clouddb)` connects successfully, confirming the 5432 credential. The
documented connection string simply carried the stale dev password.

## Fix
Apply the migration against the live `clouddb` using the real password from the db-toolkit
config (host/port/db/user unchanged):

```
dotnet ef database update --connection "Host=localhost;User ID=cloudmanager;Password=s94bEyloxB5Fho4gPpt6gHFjOyWoaRdN;Port=5432;Database=clouddb;SslMode=Prefer;Trust Server Certificate=true"
```

The migration itself is unchanged (additive nullable `resolved_vault_path`); only the
credential was corrected. No retry was attempted before writing this FIX.
