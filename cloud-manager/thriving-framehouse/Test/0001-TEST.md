# Test 0001 — API persists + returns the resolved Vault path

## TC-001 — Migration is reversible and additive
Inspect the generated EF migration.
Pass: `Up` adds a nullable `resolved_vault_path` (text) column to `vm.vm_secret_bindings`; `Down`
drops it; no other table is altered.
Fail: column is non-nullable without a default, other tables changed, or no `Down`.

## TC-002 — Provision populates the column from the resolver
Trace the provision path (or run a provision against the local/test DB with `--connection`).
Pass: a newly-inserted `vm_secret_bindings` row has `resolved_vault_path` set to the path the
`SecretBindingResolver` computed (the source VM's path for a cross-VM templated binding).
Fail: column is null on a freshly-provisioned binding, or holds the listing VM's recomputed path.

## TC-003 — Read endpoint returns the stored value
`GET /api/v1/vm/{publicId}/secret-binding` for a binding with a stored path.
Pass: `resolvedVaultPath` equals the stored column value (the true source path); camelCase JSON;
no secret value present.
Fail: endpoint recomputes/ignores the stored value, or returns the listing VM's id.

## TC-004 — Null-column fallback preserved
For a row with `resolved_vault_path` null (pre-change row).
Pass: endpoint falls back to today's recompute and returns a best-effort path (no crash/null).
Fail: endpoint errors or returns null for a legacy row.

## TC-005 — Build passes
Run `python3 .cicd/build-cloud-manager.py --target api`.
Pass: build completes with no errors.
Fail: any build error.
