# FIX-001 — Phase 4 round-trip: provision 500 on a templated binding

## Symptom
Test 0004 step 1 (provision a VM from a blueprint with a secret binding attached) returned
**HTTP 500**. API log:

```
CloudManager.Vault.Client.VaultSecretNotFoundException:
  Vault secret not found at path 'cloudmanager/data/vm/instances/vm_PPR04V26XF/njp-postgres'
```

The blueprint `bp_VEEC7P60WX` had two **templated** bindings
(`sb_P2BC7Y4JJW`, `sb_62PAR086VS`) whose `pathTemplate` is
`cloudmanager/data/vm/instances/{vm_public_id}/...`.

## Root cause (NOT a code defect in this workflow)
`SecretBindingResolver` reads the Vault secret **at instantiate time** to inject it. A templated
path only resolves once the VM public id exists (generated during instantiate), so the path
`cloudmanager/data/vm/instances/vm_PPR04V26XF/njp-postgres` cannot be pre-seeded — no value lives
there, so injection throws `VaultSecretNotFoundException` and the instantiate fails.

This is a **test-fixture** problem, not a bug in Task 0001/0002/0003 (the new read endpoint, the
two new MCP tools). The failed VM record `vm_PPR04V26XF` was left in a Failed state.

## Fix
Pivot the round-trip fixture to a **static** binding, whose path is a fixed, pre-seedable Vault key:
1. `cloud_secret_create` → managed secret `sec_PS6HKC219V` at `cloudmanager/data/manual/njp-e2e`
   (value written to Vault, status Active).
2. `POST /api/v1/secret-binding {"name":"njp-e2e-static","secretId":"sec_PS6HKC219V"}` → static
   binding `sb_W6HN0WCP05` (pathTemplate = the secret's fixed vault_path).
3. New blueprint `bp_29ZRV0YERX` with only that static binding, published.
4. `cloud_marketplace_provision` → `vm_89PF4XWDCZ`.

## Re-run result — PASS
After the VM reached orchestrationStatus=Completed, the consumer wrote the usage row and
`GET /api/v1/vm/vm_89PF4XWDCZ/secret-binding` returned:

```json
[{"publicId":"vsb_RSN2EMMPM6","credName":"DBPASSWD","secretBindingId":"sb_W6HN0WCP05",
  "secretBindingName":"njp-e2e-static","resolvedVaultPath":"cloudmanager/data/manual/njp-e2e",
  "createdAt":"2026-06-26T21:27:46.413237Z"}]
```

All Test 0004 assertions then passed (metadata + path, no value, public ids only).
