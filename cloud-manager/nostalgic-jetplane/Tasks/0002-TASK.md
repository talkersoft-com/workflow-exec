# Task 0002 ‚Äî MCP: `cloud_vm_secret_binding_list`

## Goal
Give an agent the **verification tool** for the end-to-end test ‚Äî wrap the Task 0001 endpoint so it
can confirm which secret bindings were injected into a provisioned VM.

## Reference
`../../../workflow-plans/cloud-manager/nostalgic-jetplane/PLAN.md` ‚Äî Design ¬ß2.

## Steps
- Add `cloud_vm_secret_binding_list` to `cloud-manager-mcp`. Mirror the structure of an existing
  read tool that takes a single VM id (e.g. `cloud_vm_playbook_list` / `cloud_vm_get`).
- Input: `id` (a `vm_‚Ä¶` public id). No other inputs.
- Call `GET /api/v1/vm/{id}/secret-binding` and return the usage rows verbatim (camelCase):
  `publicId`, `credName`, `secretBindingId`, `secretBindingName`, `resolvedVaultPath`, `createdAt`.
- **No secret value** in the tool output ‚Äî the endpoint never returns one; do not add any field that
  would. Description should state it returns metadata + the Vault path only.
- Register the tool in the same place the other `cloud_vm_*` tools are registered; keep the
  description/wording consistent with the existing surface.

## Build
`python3 .cicd/build-cloud-manager.py --target mcp` must pass cleanly.

## Definition of done
`cloud_vm_secret_binding_list({id})` returns the VM's usage rows against the live API (`[]` when
none), surfaces no value, and the mcp builds. Test 0002 passes.
