# Task 0001 — API: VM secret-binding read endpoint

## Goal
Expose the `vm.vm_secret_bindings` usage rows #2 records on provision — the verification data that
nothing (API or MCP) currently reads. An agent must be able to ask "what creds were injected into
this VM?" without ever seeing a secret value.

## Reference
`../../../workflow-plans/cloud-manager/nostalgic-jetplane/PLAN.md` — Design §1.

## Steps
- Add `GET /api/v1/vm/{publicId}/secret-binding` to `VirtualMachineController`
  (`cloud-manager-api`). Mirror the existing `SecretBindingController` read actions for shape and
  the controller's existing public-id → Guid resolution.
- Add the backing service method (on the VM service, mirroring the existing `SecretBinding*`
  service): query `vm_secret_bindings` for the VM, join `secret_bindings` for the binding name.
  **Soft-delete aware** (honor `deleted_at` / the global query filter).
- Return one object per usage row, camelCase, public_ids only:
  - `publicId` — the usage row (`vsb_…`)
  - `credName` — the systemd credential name injected
  - `secretBindingId` — `sb_…`
  - `secretBindingName` — joined from `secret_bindings`
  - `resolvedVaultPath` — the binding's path with `{vm_public_id}` substituted (templated) or the
    referenced `Secret.vault_path` (static). **A PATH, never a value.** (Open question default:
    return the **resolved/substituted** path. The raw `path_template` is a trivial switch if asked.)
  - `createdAt`
- **404** if the VM does not exist (use the controller's existing not-found convention).
- **No secret value** is read or returned — this endpoint never touches Vault; it only reports
  metadata + the path string. Never log a value (there is none to log here).

## Build
`python3 .cicd/build-cloud-manager.py --target api` must pass cleanly.

## Definition of done
The endpoint returns the usage rows for a VM that has them (with `secretBindingName` joined and a
resolved `resolvedVaultPath`), returns `[]` for a VM with none, 404s for an unknown VM, leaks no
`Guid` and no secret value, and the api builds. Test 0001 passes.
