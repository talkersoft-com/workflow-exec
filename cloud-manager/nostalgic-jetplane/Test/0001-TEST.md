# Test 0001 — API: VM secret-binding read endpoint

## TC-001 — Returns usage rows for a VM that has them
Pick (or provision) a VM with a `vm_secret_bindings` row, then
`GET /api/v1/vm/{publicId}/secret-binding`.
Pass: returns one object per usage row with `publicId` (`vsb_…`), `credName`, `secretBindingId`
(`sb_…`), `secretBindingName` (joined), `resolvedVaultPath`, `createdAt`. Fail: missing fields or
wrong join.

## TC-002 — Empty list for a VM with no bindings
`GET` on a VM that has no usage rows.
Pass: `[]` (200). Fail: error or non-empty.

## TC-003 — 404 for unknown VM
`GET` with a non-existent `publicId`.
Pass: 404. Fail: 200/500.

## TC-004 — No value, no Guid
Inspect the response.
Pass: `resolvedVaultPath` is a Vault **path** (no secret value anywhere in the body); every id is a
public id (`vsb_…`/`sb_…`), no raw `Guid`. Fail: any secret value or `Guid` present.

## TC-005 — Soft-delete aware
Soft-delete a binding/usage row (or use one already soft-deleted) and re-query.
Pass: soft-deleted rows are excluded. Fail: they appear.

## TC-006 — Build
`python3 .cicd/build-cloud-manager.py --target api`. Pass: clean. Fail: compile error.
