# Test 0004 — Round-trip, deploy, ship

## TC-001 — Attach → update round-trip
Attach a binding, update its `credName`/`position` via `cloud_blueprint_secret_binding_update`,
verify with `cloud_blueprint_secret_binding_list`.
Pass: the update is reflected; only the provided fields changed. Fail: no change / wrong field changed.

## TC-002 — Provision → list-usage verification
Provision a VM from that blueprint, then `cloud_vm_secret_binding_list` on it.
Pass: the injected binding appears with the expected `credName`, `secretBindingName`, and a resolved
`resolvedVaultPath` — and no value. Fail: missing row / wrong data / any value present.

## TC-003 — Edge cases
Pass: unknown VM id → 404 surfaced; VM with no bindings → `[]`. Fail: otherwise.

## TC-004 — Build + deploy healthy
Pass: `--target all` builds clean; `--target api` deployed (`cloud_health_check` green); `--target
mcp` dist rebuilt; **no** `dotnet ef database update` run (no migration added). Fail: build/deploy
issue or a migration was applied.

## TC-005 — Outputs + operator note + PRs
Pass: `Results/RESULT.md` (with the `/mcp` reload operator note near the top) and `Retro/LESSONS.md`
exist BEFORE integrate; `hv_integrate` opened PRs for `cloud-manager-api` + `cloud-manager-mcp`
(links reported). Fail: missing outputs/note or no PRs.
