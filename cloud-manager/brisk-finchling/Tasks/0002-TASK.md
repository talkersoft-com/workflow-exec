# Task 0002 тАФ Controller

## Goal

Expand `UpdateFlagRequest` to accept optional health fields and add the `/api/v1/admin/feature-flags/{key}` route alias so the vorch-service probe can reach the endpoint.

## Target repo

`cloud-manager-api`

## Files to change

- `src/API/CloudManager.API/Controllers/FeatureFlagsController.cs`

## Steps

1. Replace the `UpdateFlagRequest` record with a nullable version:
   ```csharp
   public record UpdateFlagRequest(
       bool? Enabled,
       bool? Healthy,
       DateTime? LastProbedAt
   );
   ```

2. Add the admin route alias and update the action to dispatch both fields:
   ```csharp
   [HttpPatch("{key}")]
   [Route("api/v1/admin/feature-flags/{key}")]
   public async Task<IActionResult> UpdateFlag(string key, [FromBody] UpdateFlagRequest request)
   {
       try
       {
           DtoModel.FeatureFlag? updated = null;
           if (request.Enabled.HasValue)
               updated = await _service.UpdateEnabledAsync(key, request.Enabled.Value);
           if (request.Healthy.HasValue)
               updated = await _service.UpdateHealthAsync(key, request.Healthy.Value, request.LastProbedAt);
           if (updated is null)
               return BadRequest(new { Message = "Request must include at least one of: enabled, healthy" });
           return Ok(updated);
       }
       catch (KeyNotFoundException)
       {
           return NotFound(new { Message = $"Feature flag '{key}' not found" });
       }
       catch (Exception ex)
       {
           _logger.LogError(ex, "Error updating feature flag {Key}", key);
           return StatusCode(500, new { Message = "Internal server error" });
       }
   }
   ```

3. Run `dotnet build` тАФ must compile with no errors.

## Definition of done

`dotnet build` passes. The controller accepts requests from both routes. Sending only `{healthy}` does not change `Enabled`. Sending only `{enabled}` does not change `Healthy`.
