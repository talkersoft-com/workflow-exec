# Task 0001 — Service Layer

## Goal

Add `UpdateHealthAsync` to the feature-flag service interface and implementation so the controller can persist health state independently of the enabled toggle.

## Target repo

`cloud-manager-api`

## Files to change

- `src/Services/CloudManager.Data.Services.Interface/IFeatureFlagService.cs`
- `src/Services/CloudManager.Data.Services/FeatureFlagService.cs`

## Steps

1. Add to `IFeatureFlagService`:
   ```csharp
   Task<FeatureFlag> UpdateHealthAsync(string key, bool healthy, DateTime? lastProbedAt);
   ```

2. Implement in `FeatureFlagService`:
   ```csharp
   public async Task<DtoModel.FeatureFlag> UpdateHealthAsync(string key, bool healthy, DateTime? lastProbedAt)
   {
       var flag = await _context.FeatureFlags.FirstOrDefaultAsync(f => f.Key == key)
           ?? throw new KeyNotFoundException($"Feature flag '{key}' not found");
       flag.Healthy = healthy;
       flag.LastProbedAt = lastProbedAt ?? DateTime.UtcNow;
       await _context.SaveChangesAsync();
       _logger.LogInformation("Feature flag {Key} health updated: Healthy={Healthy}", key, healthy);
       return _mapper.Map<DtoModel.FeatureFlag>(flag);
   }
   ```

3. Run `dotnet build` from the repo root — must compile with no errors.

## Definition of done

`dotnet build` passes. `IFeatureFlagService` has `UpdateHealthAsync`. `FeatureFlagService` implements it. No existing tests broken.
