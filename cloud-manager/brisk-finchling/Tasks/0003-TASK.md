# Task 0003 — Tests, Smoke, Ship

## Goal

Verify the changes work end-to-end, write Results and Lessons, then ship.

## Target repo

`cloud-manager-api`

## Steps

1. Run the existing test suite — no regressions:
   ```
   dotnet test
   ```

2. Write unit tests for `UpdateHealthAsync` (in the existing feature-flag test file or a new one):
   - Happy path: valid key → `Healthy` and `LastProbedAt` updated, returns DTO
   - `lastProbedAt` null → falls back to `DateTime.UtcNow`
   - Unknown key → throws `KeyNotFoundException` → controller returns 404
   - Sending only `{healthy}` does not change `Enabled`
   - Sending only `{enabled}` does not change `Healthy`

3. Smoke test against the running API (if available):
   ```bash
   # Health-only PATCH via admin route (probe path)
   curl -s -X PATCH http://localhost:5000/api/v1/admin/feature-flags/playbooks \
     -H "Content-Type: application/json" \
     -d '{"healthy": true, "last_probed_at": "2026-05-28T12:00:00Z"}' | jq .

   # Verify GET reflects the update
   curl -s http://localhost:5000/api/v1/featureflags | jq '.playbooks'

   # Enabled-only PATCH via original route (web UI path) — must not touch Healthy
   curl -s -X PATCH http://localhost:5000/api/v1/featureflags/playbooks \
     -H "Content-Type: application/json" \
     -d '{"enabled": false}' | jq .
   ```

4. Write `../Results/RESULT.md`
5. Write `../Retro/LESSONS.md`
6. Call `hv_ship` per `../deck.md`

## Definition of done

All unit tests pass. Smoke test (if API is running) confirms both routes work and fields update independently. `Results/RESULT.md` and `Retro/LESSONS.md` exist. `hv_ship` completes successfully.
