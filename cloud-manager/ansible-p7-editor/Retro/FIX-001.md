# FIX-001 — AnsibleRoleDetailPage crashes on real data: `file_path` vs camelCase wire JSON

## Failure
Task 0006 smoke, role-detail fragment check. Navigating to
`/ansible/roles/ansr_BZEZ2FMSPC` (smoke role created via API) crashes the page:

```
TypeError: Cannot read properties of undefined (reading 'split')
  at RoleFileTree useMemo (index-BSFlLnLz.js)
```

## Root cause
The API returns **camelCase wire JSON** (`filePath`) per epic conventions
(ANSIBLE-EPIC-CONTRACTS.md §11), confirmed live:

```
GET /api/v1/ansibleRole/{id}/file → [{"id":"rf_…","filePath":"tasks/fragment.yml",…}]
```

but the web client types and consumers (`RoleFileSummary.file_path`,
`RoleFile.file_path`, `RoleFileTree`'s `f.file_path.split("/")`,
`AnsibleRoleDetailPage`'s `selectedFile?.file_path`, `api.ts` response types)
use snake_case `file_path`. `RoleFileTree` splits `undefined` → render crash.

Pre-existing defect (the page shipped read-only in an earlier phase and was
evidently never exercised against live role files), exposed now because P7's
smoke is the first to open the page with real data. It blocks TC-002, and the
fix is in-scope: cloud-manager-web only.

## Fix
Rename the client-side field to the actual wire name `filePath` across
`api.ts` response types, `ansibleRolesSlice` (`RoleFileSummary`, `RoleFile`),
`RoleFileTree`, and `AnsibleRoleDetailPage`. No API change.

## Re-run
Rebuild + redeploy web, repeat the role-detail smoke.
