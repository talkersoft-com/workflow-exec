# Task 0002 — YAML + Ansible semantics: lang-yaml, keyword/module overlay, indent guides (cloud-manager-web repo)

## Steps
1. RESEARCH (read, no changes): `@codemirror/lang-yaml`'s Lezer tree shape — node names for
   keys, values, anchors/aliases, block scalars, comments — and how highlight tags map to it;
   CM6 patterns for tree-walking decoration plugins (ViewPlugin + RangeSet).
2. Wire `@codemirror/lang-yaml` as the host language in `AnsibleYamlEditor`: keys/values,
   anchors/aliases, block scalars, and comments each get distinct highlight tags through the
   theme from Task 0001.
3. Ansible semantics overlay — a styling pass over the YAML tree, NOT a grammar fork:
   - Play/task keywords (`hosts`, `vars`, `tasks`, `handlers`, `when`, `notify`, `become`,
     `loop`, `register`, `name`, …) highlighted distinctly when they appear in key position.
   - Module-name position: the first non-meta key of a task mapping (skip `name`, `when`,
     `tags`, `become`, etc.) styled as a module name.
   The overlay degrades gracefully on syntactically broken YAML (no exceptions thrown).
4. Indent guides: vertical guides aligned to YAML indentation levels, theme-token colored.
5. Component tests: a representative playbook fixture highlights keywords and the module-name
   key distinctly from plain keys; anchors/aliases/block scalars/comments carry their tags;
   broken YAML renders without errors.
6. Build: `python3 .cicd/build-cloud-manager.py --target web` must pass.

Done when: YAML structure and Ansible semantics highlight per the plan, indent guides render,
overlay survives broken input, web build clean.
