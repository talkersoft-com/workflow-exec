# Task 0004 — Fragments + diagnostics: fragmentMode wrap/map, client lint, diagnostics merge, gutter/panel (cloud-manager-web repo)

## Steps
1. RESEARCH (read, no changes): which consumers hand the editor fragment chunks (role files /
   fragment entities that are not valid standalone YAML documents); CM6 `linter()` /
   `lintGutter()` APIs and how diagnostics positions map to the document.
2. `fragmentMode` (C-ED prop): when true, virtually wrap the chunk before parse — synthetic
   document context + re-indent as designed in PLAN.md — and map all positions (highlighting,
   diagnostics, selections) back to the raw buffer so a fragment never errors with "missing
   document start". The raw buffer remains what is displayed and saved: relative indentation
   preserved byte-for-byte on save (epic §7).
3. Client-side lint source: derive YAML syntax errors from the Lezer tree (error nodes) and
   surface them as CM6 diagnostics.
4. Caller-supplied diagnostics: merge the `diagnostics` prop
   (`EditorDiagnostic { from, to, severity, message, rule? }`) into the same `linter()`
   surface. Host pages may fetch P6 `plint` findings and pass them when the API exposes them —
   absence is graceful; the editor itself never fetches (PLAN.md open question 3 default:
   host page owns the fetch).
5. Surfaces: squiggles per severity, gutter markers, and the lint panel; severities map
   error/warning/info distinctly.
6. Component tests: fragment fixture parses cleanly with positions mapped; syntax-error
   fixture produces a client diagnostic at the right range; supplied diagnostics render and
   re-render on prop change; fragment save round-trips byte-identical.
7. Build: `python3 .cicd/build-cloud-manager.py --target web` must pass.

Done when: fragments highlight and lint without false "missing document" errors, client +
supplied diagnostics share one surface with gutter and panel, fragment saves are
byte-identical, web build clean.
