# Task 0003 — Embedded Jinja2: in-repo Lezer grammar, nested mounting, highlight tags (cloud-manager-web repo)

## Steps
1. RESEARCH (read, no changes): Lezer nested-parser mounting (`parseMixed`) and how
   `@codemirror/lang-yaml` exposes scalar-value nodes for embedding; survey what the plan
   already decided — in-repo grammar over unmaintained npm options (PLAN.md open question 2
   default), expressions + statement delimiters first (open question 1 default).
2. Write a small in-repo Lezer grammar (~150 lines) for Jinja2 covering:
   - `{{ … }}` expressions: delimiters, identifiers, dotted attribute access, `|` filters
     with arguments, string/number literals.
   - `{% … %}` statement blocks: delimiters and the leading statement keyword highlighted;
     deep statement parsing deferred (P8/P9 may extend).
   Grammar compiled via `@lezer/generator` (build-time integration consistent with the Vite
   setup).
3. Mount the Jinja2 parser into YAML scalar values (plain, quoted, and block scalars) with
   `parseMixed` so `{{ api_token }}` is a parsed template node carrying a decoration target —
   not plain string content. Non-template scalars are untouched.
4. Highlight tags: template delimiters, identifiers, and filters get their own tags, themed
   from the existing tokens.
5. Component tests: expression inside a plain scalar, inside a quoted scalar, inside a block
   scalar; filter chains; `{%` statement delimiters; a scalar with unbalanced `{{` does not
   throw and degrades to plain text.
6. Build: `python3 .cicd/build-cloud-manager.py --target web` must pass.

Done when: Jinja2 inside YAML scalars parses as an embedded language with distinct highlight
tags, malformed templates degrade gracefully, web build clean.
