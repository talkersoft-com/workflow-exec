# Task 0006 — Deploy web, results, ship

## Steps
1. Deploy web only: `python3 .cicd/deploy-cloud-manager.py --target web`. No API, MCP, or
   vorch deploys — this workflow changes cloud-manager-web only (deck.md). No migrations.
   A failed deploy is a stop condition — write a FIX file under `Retro/`, resolve, retry.
2. Smoke (through the deployed stack at https://ubuntu-server.talkersoft.com — localhost:3000
   has no API proxy): open a playbook role file in `AnsiblePlaybookEditorPage` — CM6 editor
   renders with YAML + Ansible + Jinja2 highlighting in the active theme; flip
   `[data-theme]` and confirm the editor follows; edit → dirty indicator → Cmd/Ctrl-S saves
   with a change summary and the saved content round-trips byte-identical; open a fragment
   file on `AnsibleRoleDetailPage` — no "missing document start" diagnostic; introduce a
   YAML error — squiggle + gutter marker appear; a list page loads without fetching the
   editor chunk (network tab).
3. Write `Results/RESULT.md`: outcome, branch, PR table (TBD until ship), phase summary,
   deploy target + smoke attempts + HTTP codes, and the recorded editor-chunk bundle budget.
4. Write `Retro/LESSONS.md` (what would have helped; FIX file list — or one line if clean).
5. Ship (see `../deck.md`):
   ```
   hv_ship  deck: "cloud-manager"
            message: "feat: CodeMirror 6 Ansible YAML editor + embedded Jinja2 (P7)"
            title:   "Ansible Epic P7: real editor — CM6, Jinja2, fragments, inline lint"
            stage:   "exec"
   ```
6. Fill the PR table in `Results/RESULT.md` from the hv_ship output (amend on the same
   branch; push the amendment via hv_ship again, which will push-only). Per deck behaviour
   the fill may land as a follow-up PR — verify actual PR state with `gh` if the ship
   output omits PR lines.

Done when: web deployed with passing smoke checks, RESULT.md and LESSONS.md written, hv_ship
opened PRs for every repo ahead of main.
