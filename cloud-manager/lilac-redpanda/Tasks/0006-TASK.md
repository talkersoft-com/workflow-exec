# Task 0006 — vorch-service subscribers + handlers + widen ReadWritePaths

## Goal
Two subscribers consuming `collection-installs` and `collection-removes`, two handlers that shell out to `ansible-galaxy`, verify, and PATCH back. Systemd unit widens `ReadWritePaths` to `/usr/share/ansible/collections` so the install can write there.

## Target repo(s)
`vorch-service` (handler/subscriber code), `cloud-manager-api` (systemd unit template in `install-vorch-service.py`)

## Steps

1. **Subscribers** under `vorch-service/vorch-service/messagesub/`:
   - `collection_install_sub.go` — copy of `playbook_run_sub.go`, swap queue name + handler reference.
   - `collection_remove_sub.go` — same for the remove queue.

2. **Install handler** at `vorch-service/vorch-service/handlers/collection_install_handler.go`:
   ```go
   func (h *CollectionInstallHandler) Handle(ctx, body) error {
       // 1. decode msg → runID
       // 2. fetchRun(runID) → returns {collectionNamespace, collectionName, requestedVersion, force, installPath?}
       // 3. PATCH run → status=Running, startedAt=now
       // 4. cmd := ansible-galaxy collection install <ns>.<name>[:<ver>] -p /usr/share/ansible/collections
       //    if force: append --force
       // 5. capture combined stdout+stderr
       // 6. on non-zero exit: PATCH {status:Failed, output, errorMessage, completedAt}; return nil (ack)
       // 7. on zero exit: run `ansible-galaxy collection list <ns>.<name> --format json`
       //    parse version + install_path from the JSON output
       // 8. PATCH {status:Succeeded, output, version, installPath, completedAt}
   }
   ```

3. **Remove handler** at `vorch-service/vorch-service/handlers/collection_remove_handler.go`:
   ```go
   func (h *CollectionRemoveHandler) Handle(ctx, body) error {
       // 1. decode msg → runID
       // 2. fetchRun(runID) → returns {collectionNamespace, collectionName, installPath}
       // 3. PATCH run → status=Running
       // 4. SAFETY GUARD: if !strings.HasPrefix(installPath, "/usr/share/ansible/collections/ansible_collections/")
       //                 → PATCH {status:Failed, errorMessage:"install path failed safety check"}; return nil
       // 5. os.RemoveAll(installPath)
       // 6. verify: run `ansible-galaxy collection list <ns>.<name> --format json`; expect empty
       //    if the collection still shows up: PATCH {status:Failed, errorMessage:"removal verification failed"}; return nil
       // 7. PATCH {status:Succeeded, output:"removed", completedAt}
   }
   ```

4. **Register subscribers + handlers** in `vorch-service/vorch-service/app.go` next to the existing playbook-run subscriber registration.

5. **Widen ReadWritePaths**: edit `cloud-manager-api/scripts/vorch/install-vorch-service.py`:
   ```
   ReadWritePaths={self.install_dir} /var/lib/libvirt/images /mnt/cloud-storage /usr/share/ansible/collections
   ```

6. **Build verification**:
   ```
   cd vm-infra/cloud-manager/vorch-service/vorch-service
   go build ./...
   go vet ./...
   ```

7. **Write `Retro/CHANGE-0006.md`** noting the two-repo change (vorch-service + cloud-manager-api install script).

## Definition of done
- Two new subscribers, two new handlers, registration wired in `app.go`.
- `install-vorch-service.py` widens ReadWritePaths.
- `go build ./...` and `go vet ./...` clean.
- Safety guard for the remove path verified by grep.
- `Retro/CHANGE-0006.md` exists.
