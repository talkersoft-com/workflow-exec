# Task 0008 — Build + deploy

## Goal
All three services running the new code on `ubuntu-server`, with the migration applied to `clouddb`.

## Target repo(s)
`cloud-manager-api` (binary + migration), `vorch-service` (binary), `cloud-manager-web` (bundle).

## Steps

1. **Migration apply** (idempotent; skip if Phase 1 already applied):
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src/Models/CloudManager.Entities
   dotnet ef database update -s ../../CloudManager.API
   ```

2. **cloud-manager-api**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api
   sudo python3 scripts/api/install-api-service.py
   sudo systemctl is-active cloud-manager-api
   ```
   Expect `active`. Verify a known endpoint:
   ```
   curl -sk https://ubuntu-server.talkersoft.com/api/v1/ansible/collections | python3 -m json.tool | head
   ```
   Expect a JSON array including the two seeded rows.

3. **vorch-service**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api
   sudo python3 scripts/vorch/install-vorch-service.py
   sudo systemctl is-active vorch_service
   ```
   Expect `active`. Verify the unit has the new `ReadWritePaths`:
   ```
   sudo grep ReadWritePaths /etc/systemd/system/vorch_service.service
   ```
   Must include `/usr/share/ansible/collections`.

4. **cloud-manager-web**:
   ```
   sudo rm -rf vm-infra/cloud-manager/cloud-manager-web/dist
   cd vm-infra/cloud-manager/cloud-manager-api
   sudo python3 scripts/web/install-web-app.py
   sudo systemctl is-active cloud-manager-web
   ```
   Expect `active`. Verify the new tab string is in the bundle:
   ```
   sudo grep -l "Collections" /opt/cloud-manager-web/assets/*.js | head -3
   ```

5. **Smoke check**: Playwright-navigate to `/ansible/collections` and confirm the page renders with the two seeded rows. Capture a screenshot to `Results/post-deploy-list.png`.

## Definition of done
- Migration applied; `\dt ansible.*` shows four tables; `SELECT count(*) FROM ansible.collections;` returns ≥ 2.
- All three services `active`.
- `curl /api/v1/ansible/collections` returns the seeded rows.
- `/etc/systemd/system/vorch_service.service` has `/usr/share/ansible/collections` in `ReadWritePaths`.
- `/ansible/collections` page renders.
