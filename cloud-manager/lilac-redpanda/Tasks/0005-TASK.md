# Task 0005 — vorch-lib message structs

## Goal
Two Go structs for the new queue payloads, with camelCase JSON tags per the established convention.

## Target repo(s)
`vorch-lib`

## Steps

1. Add to `vorch-lib/models/messages/`:
   - `collection_install_run.go`:
     ```go
     package messages

     // CollectionInstallRunMessage is the body shape of the collection-installs
     // RabbitMQ queue. The C# ICollectionInstallPublisher in cloud-manager-api
     // serializes { "runId": "acrun_xxx" }.
     type CollectionInstallRunMessage struct {
         RunID string `json:"runId" yaml:"runId"`
     }
     ```
   - `collection_remove_run.go`:
     ```go
     // CollectionRemoveRunMessage is the body shape of the collection-removes queue.
     type CollectionRemoveRunMessage struct {
         RunID string `json:"runId" yaml:"runId"`
     }
     ```

2. Both structs use camelCase JSON tags (`runId`) per the convention shipped in vorch-lib PR #3. NOT snake_case.

3. Build verification:
   ```
   cd vm-infra/cloud-manager/vorch-lib
   go build ./...
   go vet ./...
   ```
   Expect no output.

4. Write `Retro/CHANGE-0005.md`.

## Definition of done
