# Task 0004 — API AMQP publishers

## Goal
Two new publishers that declare `collection-installs` and `collection-removes` queues and publish `{ "runId": "acrun_..." }` (camelCase) bodies. Endpoints from Phase 3 inject the publishers and call `.Publish(runPublicId)` after the DB write.

## Target repo(s)
`cloud-manager-api`

## Steps

1. **`CollectionInstallPublisher.cs`** under `src/Services/CloudManager.AMQP.Publisher/`:
   - Copy `PlaybookRunPublisher.cs` shape.
   - `private const string QueueName = "collection-installs";`
   - `Publish(string runPublicId)` → declare queue durable + persistent message + body `JsonSerializer.Serialize(new { runId = runPublicId })`.

2. **`CollectionRemovePublisher.cs`** — same shape, `QueueName = "collection-removes"`.

3. **Interfaces** under `Interface/`: `ICollectionInstallPublisher`, `ICollectionRemovePublisher`. Mirror `IPlaybookRunPublisher`.

4. **DI registration** in `ConfigureServices.cs` for the AMQP publisher project: register both as singletons.

5. **Wire into endpoints**: `HostCollectionsController.RequestInstallAsync` and `RequestReinstallAsync` call `_installPublisher.Publish(run.PublicId)`. `RequestRemoveAsync` calls `_removePublisher.Publish(run.PublicId)`. Both publish AFTER the DB SaveChangesAsync succeeds.

6. **Build**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src
   dotnet build CloudManager.sln -c Release
   ```
   Expect 0 errors.

7. **Append to `Retro/CHANGE-0004.md`**.

## Definition of done
- Two publisher classes + two interfaces.
- `dotnet build` clean.
- Publish call sites visible in `HostCollectionsController.cs`.
- `Retro/CHANGE-0004.md` exists.
