# Task 0003 — API endpoints (10 routes under /api/v1/ansible)

## Goal
Three new controllers exposing the ten endpoints listed in PLAN.md. Validation, error mapping (KeyNotFoundException → 404, InvalidOperationException → 400/409 as appropriate), and consistent error envelopes match the existing controller style.

## Target repo(s)
`cloud-manager-api`

## Steps

1. **`AnsibleCollectionsController.cs`** at `src/CloudManager.API/Controllers/`:
   - `GET /api/v1/ansible/collections` → `ListAsync`
   - `POST /api/v1/ansible/collections` → `CreateAsync` with `{ namespace, name, description?, sourceUrl? }`. Validation: lowercase, dot-free in each segment; namespace and name both required.
   - `GET /api/v1/ansible/collections/{cid}` → detail incl. nested `host_collections` rows
   - `DELETE /api/v1/ansible/collections/{cid}` → 409 if any `host_collections` row still references it

2. **`HostCollectionsController.cs`**:
   - `GET /api/v1/ansible/hosts/{hid}/collections` → list per host
   - `POST /api/v1/ansible/hosts/{hid}/collections` → body `{ collectionPublicId, requestedVersion? }`, optional `?force=true`. 409 if a row already exists unless `force`. Returns the new `acrun_…` run id.
   - `POST /api/v1/ansible/host-collections/{hacollId}/reinstall` → new install run with `Force=true`. Status flips back to `Installing`.
   - `DELETE /api/v1/ansible/host-collections/{hacollId}` → request remove. 409 if status is already `Removing` or `Removed`.

3. **`CollectionInstallRunsController.cs`**:
   - `GET /api/v1/ansible/collection-install-runs/{cid}` → status + output
   - `PATCH /api/v1/ansible/collection-install-runs/{cid}` → worker callback. Body `{ status, output?, errorMessage?, version?, installPath?, completedAt? }`. Same auth model as the existing playbook-run callback.

4. **Error envelope**: every catch returns `new { Message = ex.Message }` for known exceptions, `new { Message = "Internal server error" }` for unknown 500s. Log `_logger.LogError(ex, "<op> failed")` on 500s.

5. **Build**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src
   dotnet build CloudManager.sln -c Release
   ```
   Expect 0 errors.

6. **Append to `Retro/CHANGE-0003.md`** with one line per file added.

## Definition of done
- Three controller files, ten endpoint actions.
- `dotnet build` clean.
- `Retro/CHANGE-0003.md` exists.
- Routes can be hit (the actual install path is wired in Phase 4 — publisher; this phase the POST endpoints just write DB rows + return).
