# Task 0007 — Web: Collections sub-nav tab + list/detail + modal + slices + polling

## Goal
Fifth sub-nav tab ("Collections") that lands on a list page with status badges + Install/Remove/Reinstall actions, a detail page showing install runs history, and an add-collection modal. Three Redux slices wire the API. Transitional states (Installing, Removing) poll every 5s.

## Target repo(s)
`cloud-manager-web`

## Steps

1. **`AnsibleSubNav.tsx`** — add fifth tab:
   ```tsx
   { label: "Collections", path: "/ansible/collections", match: ["/ansible/collections"] },
   ```

2. **`App.tsx`** — add two routes:
   ```tsx
   <Route path="/ansible/collections" element={<AnsibleLayout><AnsibleCollectionsPage /></AnsibleLayout>} />
   <Route path="/ansible/collections/:cid" element={<AnsibleLayout><AnsibleCollectionDetailPage /></AnsibleLayout>} />
   ```

3. **Redux slices** under `src/store/slices/`:
   - `collectionsSlice.ts` — thunks: `fetchCollections`, `fetchCollection`, `createCollection`, `deleteCollection`.
   - `hostCollectionsSlice.ts` — thunks: `fetchHostCollections(hostId)`, `requestInstall(hostId, collectionId, requestedVersion?)`, `requestReinstall(hacollId)`, `requestRemove(hacollId)`.
   - `collectionInstallRunsSlice.ts` — thunks: `fetchCollectionInstallRun(runId)`. Has a polling thunk `pollUntilTerminal(runId)` invoked when the UI sees `Installing` or `Removing` status; polls every 5s, stops on `Succeeded`/`Failed`.

4. **API client** in `src/services/api.ts`: add the ten endpoint helpers under an `ansible.collections.*` namespace.

5. **`AnsibleCollectionsPage.tsx`** at `src/pages/`:
   - Header: title + `+ Add collection` button (opens modal).
   - Table: columns `namespace.name`, description, latest known version, status badge for the controller host, actions cell.
     - Status badge sources from `host_collections.install_status` for the single controller host. Empty/unknown shows "Not installed".
     - Action buttons depend on state (per PLAN.md): Install (None/Removed/Failed) | Remove + Reinstall (Installed) | Spinner only (Pending/Installing/Removing).
   - Wrap in `<FeatureGate flag="playbooks">`.

6. **`AnsibleCollectionDetailPage.tsx`** — metadata + install runs table + action buttons. Pencil icon launches modal-edit (optional v1 — text fields only).

7. **Add-collection modal** — reuse the `Modal` component from `PlaybooksPage`. Fields: namespace, name, description (optional), source_url (optional). On submit → `createCollection` thunk → close modal → refresh list.

8. **Polling** — `useEffect` in the list page: for every visible row whose `install_status` is `Installing` or `Removing`, dispatch `pollUntilTerminal(install_run_id)`. Implementation: setTimeout chain inside the thunk; cancellation via component-unmount.

9. **Build verification**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-web
   npx tsc --noEmit
   npm run build
   ```
   Both must be clean.

10. **Append to `Retro/CHANGE-0007.md`**.

## Constraints
- Don't add Collections affordances to the Run page, History page, or playbook-detail page. Tab-scoped only.
- Don't add the `playbook_collection_requirements` UI in this phase — schema exists, surface comes later.
- Don't fetch from Galaxy.ansible.com — manual catalog entry only.

## Definition of done
- `AnsibleSubNav` shows 5 tabs (Playbooks / Roles / Run / History / Collections).
- Two new pages + three new slices.
- `tsc --noEmit` and `npm run build` clean.
- `Retro/CHANGE-0007.md` exists.
