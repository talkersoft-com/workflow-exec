# Task 0001 — EF Core migration: ansible schema, 4 tables, enum, seed

## Goal
A new EF Core migration that creates the `ansible` schema, the four tables exactly as DATA-MODEL.md specifies, the `CollectionInstallStatus` Postgres enum, all FK constraints + indexes, and seeds `community.postgresql` + `community.hashi_vault` into `ansible.collections`. Migration applies cleanly against the live `clouddb`.

## Target repo(s)
`cloud-manager-api`

## Steps

1. **Add entity classes** under `src/Models/CloudManager.Entities/Models/`:
   - `AnsibleCollection.cs` — `Namespace`, `Name`, `Description`, `SourceUrl`, `LatestKnownVersion`. UNIQUE composite (`Namespace`, `Name`).
   - `HostAnsibleCollection.cs` — `HostId` (FK → `bare_metal.hosts`), `CollectionId` (FK → `ansible.collections`), `Version`, `InstallPath`, `InstallStatus` (the new enum), `LastInstallRunId` (nullable FK → `collection_install_runs`), `ErrorMessage`. UNIQUE composite (`HostId`, `CollectionId`).
   - `AnsibleCollectionInstallRun.cs` — `HostId` (FK), `CollectionId` (FK), `RequestedVersion`, `Status` (`PlaybookRunStatus`), `Output`, `ErrorMessage`, `StartedAt`, `CompletedAt`, `Force` (bool — needed by handler to set `--force`), `IsRemoval` (bool — distinguishes install vs remove runs).
   - `PlaybookCollectionRequirement.cs` — `PlaybookId` (FK → `vm.playbooks`), `CollectionId` (FK), `MinVersion`, `MaxVersion`. UNIQUE composite (`PlaybookId`, `CollectionId`).
   - All inherit `Audit`. Public-id prefixes registered in `EntityPrefixRegistry.cs`: `acoll_`, `hacoll_`, `acrun_`, `pcreq_`.

2. **Add the enum**: `src/Models/CloudManager.Entities/Enums/CollectionInstallStatus.cs` with values `Pending, Installing, Installed, Failed, Removing, Removed`. String storage.

3. **Update `CloudManagerDbContext.cs`**:
   - Add `DbSet<>` properties for all four new entities.
   - Configure each entity with `ToTable("<name>", "ansible")`.
   - Configure FKs: `ON DELETE CASCADE` for `host_collections.host_id`, `ON DELETE RESTRICT` for `host_collections.collection_id` and all other FKs.
   - Configure UNIQUE indexes for `(namespace, name)`, `(host_id, collection_id)`, `(playbook_id, collection_id)`.
   - Configure non-unique indexes on `collection_install_runs.host_id` and `.collection_id`.
   - Configure the new enum as a Postgres enum: `modelBuilder.HasPostgresEnum<CollectionInstallStatus>("ansible");`.

4. **Generate the migration**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src/Models/CloudManager.Entities
   dotnet ef migrations add AddAnsibleCollectionsSchema -s ../../CloudManager.API
   ```
   - Verify the generated `Up()` creates the schema, the enum, the four tables, indexes, FKs.
   - Verify `Down()` is reversible (drop tables, drop enum, drop schema).

5. **Add seed inserts** in the generated migration's `Up()` (or via `HasData` in the DbContext):
   ```sql
   INSERT INTO ansible.collections (id, public_id, namespace, name, description, source_url, latest_known_version, created, created_by, modified, modified_by) VALUES
     (uuid_generate_v4(), 'acoll_PGSQL00001', 'community', 'postgresql', 'PostgreSQL modules for Ansible', 'https://galaxy.ansible.com/community/postgresql', NULL, NOW(), 'migration', NOW(), 'migration'),
     (uuid_generate_v4(), 'acoll_HVAULT0001', 'community', 'hashi_vault', 'HashiCorp Vault modules for Ansible', 'https://galaxy.ansible.com/community/hashi_vault', NULL, NOW(), 'migration', NOW(), 'migration');
   ```
   Comment in the migration: "Seeded because the `pg` playbook references these two collections; demo path should not require manual catalog entry."

6. **Build**:
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src
   dotnet build CloudManager.sln -c Release
   ```
   Expect 0 errors.

7. **Apply migration** (deploys to live clouddb):
   ```
   cd vm-infra/cloud-manager/cloud-manager-api/src/Models/CloudManager.Entities
   dotnet ef database update -s ../../CloudManager.API
   ```
   Expect success.

8. **Write `Retro/CHANGE-0001.md`** with one line per file added/modified.

## Definition of done
- Four entity classes, one enum class, DbContext updated, migration file generated.
- `dotnet build` clean (0 errors).
- `dotnet ef database update` succeeds.
- `\dt ansible.*` shows all four tables. `\dT ansible.*` shows the enum.
- `SELECT count(*) FROM ansible.collections;` returns 2.
- `Retro/CHANGE-0001.md` exists.
