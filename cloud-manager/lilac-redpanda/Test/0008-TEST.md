# Test 0008 — Build + deploy

## TC-001 — All services active
Run:
```
sudo systemctl is-active cloud-manager-api vorch_service cloud-manager-web
```
Pass: `active` for all three.
Fail: any failed/inactive.

## TC-002 — ansible schema present in DB
Run:
```
PGPASSWORD='P@ssw0rd!' psql -h localhost -U cloudmanager -d clouddb -tAc "SELECT count(*) FROM information_schema.tables WHERE table_schema='ansible';"
```
Pass: ≥ 4.
Fail: less.

## TC-003 — API returns the catalog with seeded rows
Run:
```
curl -sk https://ubuntu-server.talkersoft.com/api/v1/ansible/collections | python3 -c "import json,sys;d=json.load(sys.stdin);print(len(d), [(r['namespace'],r['name']) for r in d])"
```
Pass: at least 2 rows; both `community.postgresql` and `community.hashi_vault` appear.
Fail: empty or missing seeds.

## TC-004 — Vorch unit has new ReadWritePaths
Run:
```
sudo grep ReadWritePaths /etc/systemd/system/vorch_service.service
```
Pass: line includes `/usr/share/ansible/collections`.
Fail: missing.

## TC-005 — Web bundle has the Collections tab string
Run:
```
sudo grep -l "Collections" /opt/cloud-manager-web/assets/*.js | head
```
Pass: at least one file.
Fail: empty.

## TC-006 — Collections page renders
Use Playwright. Navigate to `/ansible/collections`. Capture `Results/post-deploy-list.png`.
Pass: page renders without error; sub-nav shows 5 tabs with Collections active; two seeded rows visible.
Fail: blank, error, or sub-nav missing.
