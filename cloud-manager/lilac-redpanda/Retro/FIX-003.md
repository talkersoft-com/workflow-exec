# Fix 003 — point ansible-galaxy's $HOME at /tmp under PrivateTmp

## Failed test
Task 0009, Flow B install run `acrun_S1Z0EXV8AQ` failed with `Unable to create local directories(/root/.ansible/tmp): Read-only file system`.

## Root cause
The vorch systemd unit has `ProtectHome=yes`, which means `/root` is mounted read-only inside the service. ansible-galaxy wants to write its tmp dir at `~/.ansible/tmp` (resolving `~` to `/root`), and the write fails. The hardening flag is correct; ansible just needs a different HOME.

## Fix applied
Add `Environment=HOME=/tmp` to the vorch systemd unit (and to the installer template). `PrivateTmp=yes` means `/tmp` is a per-service tmpfs that's writable and isolated, so ansible-galaxy gets a writable cache without exposing real user homes.

## Outcome
Pass — ansible-galaxy creates `/tmp/.ansible/tmp` and the install proceeds.
