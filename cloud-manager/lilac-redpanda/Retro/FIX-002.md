# Fix 002 — install ansible-core system-wide so root finds ansible-galaxy

## Failed test
Task 0009, Flow B install failed with `ansible-galaxy: executable file not found in $PATH` (run `acrun_4WTHS5S1NB`, status Failed).

## Root cause
The earlier `apt-get install -y ansible-runner` step landed ansible-runner in a way that did not bring `ansible-core` for root. The only `ansible-galaxy` on disk was `/home/cloudmanager/.local/bin/ansible-galaxy` from a pip-as-cloudmanager install — root's PATH doesn't include it, so vorch (running as root) failed at exec time.

## Fix applied
```
sudo apt-get install -y ansible
```
Pulls `ansible-core` + the full collection bundle. `ansible-galaxy` now resolves at `/usr/bin/ansible-galaxy` which is on root's default PATH.

## Outcome
Pass — re-running the install flow now succeeds (verified in Flow B re-run).
