# Fix 001 — pre-create /usr/share/ansible/collections before vorch start

## Failed test
Task 0008, post-deploy `systemctl is-active vorch_service` returns `activating`.

## Root cause
The widened systemd `ReadWritePaths=...` line now includes `/usr/share/ansible/collections`. systemd refuses to start a unit that lists a non-existent `ReadWritePaths` entry:
```
vorch_service.service: Failed to set up mount namespacing: /usr/share/ansible/collections: No such file or directory
```
The directory only gets created when ansible-galaxy first installs a collection there — which is the operation vorch is supposed to perform. Chicken-and-egg.

## Fix applied
1. Pre-create the directory once at install time:
   ```
   sudo mkdir -p /usr/share/ansible/collections
   sudo chmod 755 /usr/share/ansible/collections
   ```
2. Add the same pre-create step into `install-vorch-service.py` so future deploys don't hit this on a fresh host.

## Outcome
Pass — vorch starts cleanly, healthprobe runs, queue subscribers up.
