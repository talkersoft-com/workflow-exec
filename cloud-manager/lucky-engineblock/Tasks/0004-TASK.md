# Task 0004 — MCP marketplace tools

## Goal
The marketplace is fully drivable from the cloud-manager MCP.

## Steps (cloud-manager-mcp)
1. New `src/tools/marketplace.ts` following the patterns in `vms.ts` (client, camelCase assertion,
   error envelopes, zod schemas with descriptive `.describe()` text):
   - `cloud_blueprint_list`, `cloud_blueprint_get`, `cloud_blueprint_create`,
     `cloud_blueprint_update`, `cloud_blueprint_delete`
   - `cloud_blueprint_publish`, `cloud_blueprint_archive`
   - `cloud_blueprint_playbook_attach` (playbookId, position?, varsOverride?),
     `cloud_blueprint_playbook_reorder` (bppId, position),
     `cloud_blueprint_playbook_detach`
   - `cloud_marketplace_list`
   - `cloud_marketplace_provision` (blueprintId, hostId, name, description?, memory?, diskSizeGb?)
     → returns virtualMachineId + runPlan
2. Register the module in `src/index.ts` and add it to the `hv.cloud` profile in `profiles.json`.
3. Rebuild dist: `python3 .cicd/build-cloud-manager.py --target mcp` (lockfile must not drift —
   the build uses npm ci semantics).

## Definition of done
Build green; tool list of a freshly spawned server includes all 12 tools; payload field names match
the API contract exactly (camelCase, public ids).
