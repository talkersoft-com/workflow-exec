# Task 0002 — API: Blueprint + Marketplace controllers

## Goal
Full CRUD + lifecycle + ordered playbook management on `/api/v1/blueprint`, and the storefront +
instantiate on `/api/v1/marketplace`, all gated by `[RequireFeatureFlag("marketplace")]`, with
BlueprintEvents recorded on every state change. Instantiate is MVP: VM + copied assignments +
provision; auto-run lands in Phase 0003.

## Repo
`cloud-manager-api`

## Conventions
Model the stack on Playbook/PlaybookAssignment (exemplar): DTOs, AutoMapper profile, service,
controller. camelCase wire JSON; public ids only (convert via `PublicIdByGuidAsync` /
`GuidByPublicIdAsync`); BlueprintEvents are append-only, best-effort try/catch, never blocking the
primary write.

## Steps
1. `BlueprintController` (`/api/v1/blueprint`):
   - `GET /list` — all non-deleted blueprints (admin/builder view)
   - `GET /{id}` — detail incl. ordered playbooks
   - `POST /` — create `{name, description, vmi, defaultMemory, defaultDiskSizeGb, config?}`
   - `PATCH /{id}` — update mutable fields (Draft/Published both editable; edits to Published log
     an `updated` event)
   - `DELETE /{id}` — soft delete; only Draft/Archived (Published must be archived first → 409)
   - `POST /{id}/publish`, `POST /{id}/archive` — Draft→Published→Archived; Archived→Published
     allowed; every transition logs a BlueprintEvent
   - `POST /{id}/playbook` `{playbookId, position?, varsOverride?}` — position defaults to end
   - `PATCH /{id}/playbook/{bppId}` `{position?, varsOverride?}` — reorder renumbers atomically
   - `DELETE /{id}/playbook/{bppId}`
2. `MarketplaceController` (`/api/v1/marketplace`):
   - `GET /` — published, non-deleted blueprints only: name, description, image summary,
     playbook count
   - `POST /{blueprintId}/instantiate` `{hostId, name, description?, memory?, diskSizeGb?}` —
     overrides win over blueprint defaults. Transactionally: create VirtualMachine from image +
     defaults with `BlueprintId` set, **reusing the exact create path the wizard uses** (MAC/IP
     allocation, VmEvent logging); copy each BlueprintPlaybook → `VmPlaybookAssignment`
     (VarsOverride copied); trigger provision via existing `IProvisionService`; log `instantiated`
     BlueprintEvent + VmEvent. Returns `{virtualMachineId, runPlan: [assignmentIds in order]}`.
3. Gate every new endpoint with `[RequireFeatureFlag("marketplace")]`.
4. Record BlueprintEvents for created/updated/published/archived/playbook_attached/
   playbook_detached/reordered/instantiated.
5. Build: `python3 .cicd/build-cloud-manager.py --target api` — must pass cleanly.

## Definition of done
All endpoints implemented per the plan's shapes, flag-gated, events recorded; api build green.
Run `Test/0002-TEST.md`.
