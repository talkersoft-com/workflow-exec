# Task 0001 — Data model + migration + seeds

## Goal
The `marketplace` schema exists in code and in a reversible EF migration: Blueprint,
BlueprintPlaybook, BlueprintEvent entities plus nullable `VirtualMachine.BlueprintId`, with
public-id prefixes registered and `jammy` / `postgres-jammy` blueprints + `marketplace` feature
flag seeded.

## Repo
`cloud-manager-api` (`/home/todd/workspace/cloud-manager/vm-infra/cloud-manager/cloud-manager-api`)

## Steps
1. Study the exemplars first: `VirtualMachine` (soft-delete + filtered unique index), `VmEvent`
   (append-only event log), `Playbook`/`VmPlaybookAssignment` (join + JSONB vars),
   `EntityPrefixRegistry`, the DbContext entity configurations, and an existing migration + seed.
2. Add `BlueprintStatus` enum: `Draft | Published | Archived` (default Draft).
3. Add **Blueprint** entity (`bp_` prefix; Audit base; soft-delete via `DeletedAt` + global query
   filter like VirtualMachine):
   - Name string(255) required, unique where `deleted_at is null`
   - Description string(4096) required
   - VirtualMachineImageId Guid FK → VirtualMachineImage, required, Restrict
   - DefaultMemory int required (MB, same semantics as VirtualMachine.Memory)
   - DefaultDiskSizeGb int required (5–500 validation, same as VM)
   - Config JSONB optional
   - Status BlueprintStatus, default Draft
4. Add **BlueprintPlaybook** entity (`bpp_` prefix; Audit base; no soft delete):
   - BlueprintId FK → Blueprint required Cascade; PlaybookId FK → Playbook required Restrict
   - Position int required (0-based run order)
   - VarsOverride JSONB required default `{}`
   - Constraints: `unique(BlueprintId, PlaybookId)`, `unique(BlueprintId, Position)`,
     index (BlueprintId, Position)
5. Add **BlueprintEvent** entity (`bpev_` prefix; mirrors VmEvent exactly; append-only, no soft
   delete): BlueprintId FK, EventType
   (`created|updated|published|archived|playbook_attached|playbook_detached|reordered|instantiated`),
   Payload JSONB, Actor, OccurredAt.
6. Add nullable `BlueprintId` Guid FK (Restrict) to `VirtualMachine` — additive only; no other
   change to VM entity, indexes, or filters.
7. Register `bp`, `bpp`, `bpev` in `EntityPrefixRegistry`.
8. Wire DbSets + entity configurations into the DbContext (new `marketplace` schema).
9. Create ONE EF migration covering all of the above, with a working reversible `Down()`.
10. Seed (same mechanism as `seed/playbooks/`):
    - blueprint **jammy** — ubuntu jammy `vmi`, zero playbooks, Published
    - blueprint **postgres-jammy** — same image + `pg-14-jammy` playbook at Position 0, Published
    - feature flag row `marketplace`, disabled
11. Build: `python3 .cicd/build-cloud-manager.py --target api` — must pass cleanly.

## Definition of done
Entities, prefixes, DbContext config, one reversible migration, seeds, and flag all in place;
api build green. Run `Test/0001-TEST.md`.
