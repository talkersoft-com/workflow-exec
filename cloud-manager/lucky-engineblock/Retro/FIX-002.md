# FIX-002 — deploy-cloud-manager.py api smoke test polls a non-existent route

## Failure
`deploy-cloud-manager.py --target api` reported "smoke test failed for
http://localhost:5250/api/v1/Host" although the service deployed and started cleanly
(journalctl shows "Now listening on :5250", app healthy). `smoke_http` expects
200/304/401/403 but `HostController` only routes `GET api/v1/Host/list` — the bare
`/api/v1/Host` URL has no route and always returns 404, so the smoke test can never pass.

## Fix
Changed the smoke URL in `cloud-manager-mcp/.cicd/deploy-cloud-manager.py` from
`/api/v1/Host` to `/api/v1/Host/list` (the actual list route, returns 200). Re-ran
`--target api` to confirm a green deploy.

## Status
Resolved — deploy re-run passes its smoke test.
