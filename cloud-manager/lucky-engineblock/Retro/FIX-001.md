# FIX-001 — Test API instance failed to start (port 5250 hardcoded)

## Failure
Phase 0002 testing: launching a second API instance on port 5251 failed —
`Program.cs:138` calls `builder.WebHost.UseUrls("http://ubuntu-server.talkersoft.com:5250")`,
which overrides `ASPNETCORE_URLS`/`--urls`, so the test instance tried to bind 5250 where the
live deployed API already listens (`AddressInUseException`).

## Fix
No code change (the hardcoded UseUrls is pre-existing behavior this plan must not alter).
Resolution: deploy early instead of running a parallel instance — apply the migration to live
clouddb and deploy the new api build (Phase 0005's documented procedure, executed at the end of
Phase 0002). All new endpoints are behind the `marketplace` feature flag, which is seeded
disabled, so the early deploy does not expose unverified behavior. Test/0002 TCs then run
against the deployed API on 5250 (flag toggled on for the test). TC-006 (live instantiate) and
the apply-path part of TC-007 remain in Phase 0005 as planned. Phase 0005 re-runs the deploy
script for the final build.

## Status
Resolved. Test DB instance `clouddb_apitest` dropped after seed verification.
