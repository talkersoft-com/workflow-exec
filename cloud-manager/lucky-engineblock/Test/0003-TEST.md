# Test 0003 — Run sequencer

Logic-level verification may use unit/integration tests against the sequencer's decision function;
the full live-VM pass happens in Test 0005. Verify at minimum:

## TC-001 — Build green
`python3 .cicd/build-cloud-manager.py --target api` passes.

## TC-002 — First-run trigger
VM with BlueprintId set reaches OrchestrationStatus=Completed → exactly one run enqueued, for the
assignment matching blueprint Position 0.

## TC-003 — Chain advance
Sequenced run at position N reaches Succeeded → run for position N+1 enqueued; after the last
position succeeds, nothing further is enqueued.

## TC-004 — Stop on failure
A sequenced run fails → no further run enqueued; VmEvent `blueprint_run_chain_failed` recorded.
Manual `apply` afterwards still works (existing path untouched).

## TC-005 — Non-blueprint VMs untouched
VM with BlueprintId == null completing orchestration or finishing runs → sequencer does nothing.

## TC-006 — Zero-playbook blueprint
Blueprint with no playbooks (jammy): provision completes → no run enqueued, no failure event.

## TC-007 — Re-entrancy
Duplicate/repeated status writeback for the same transition does not double-enqueue a run.
