# Test 0002 — Blueprint + Marketplace API

Exercise against a locally-run API (or the deployed dev API) with the `marketplace` flag enabled
unless a TC says otherwise. All ids on the wire must be public ids; all JSON camelCase.

## TC-001 — Build green
`python3 .cicd/build-cloud-manager.py --target api` passes.

## TC-002 — Flag gating
With flag `marketplace` disabled: every `/api/v1/blueprint*` and `/api/v1/marketplace*` endpoint
refuses (the standard RequireFeatureFlag response). With it enabled: they respond.

## TC-003 — Blueprint CRUD
Create a blueprint (valid vmi, defaults) → 200 with `bp_` id; GET /{id} returns it; PATCH updates
mutable fields; GET /list includes it. DELETE on a Draft soft-deletes (row keeps deleted_at; gone
from /list).

## TC-004 — Lifecycle
Draft→publish→200 + `published` event; publish→archive→200 + `archived` event;
Archived→publish allowed. DELETE on a Published blueprint → 409. Marketplace GET / shows only
Published, non-deleted.

## TC-005 — Ordered playbook management
Attach playbook A (no position → position 0), attach B (→ 1). Reorder B to 0 → positions renumber
atomically (A=1, B=0; no unique-constraint violation). Duplicate attach of A → 409. Detach A →
positions stay consistent. Events logged for attach/detach/reorder.

## TC-006 — Instantiate (MVP)
POST /marketplace/{postgres-jammy}/instantiate with a valid hostId + name:
- Returns `{virtualMachineId, runPlan}` with assignment ids in blueprint order
- VM exists with the blueprint's image, defaults (or caller overrides), BlueprintId provenance set
- VmPlaybookAssignments copied (VarsOverride included)
- Provision was triggered through the existing IProvisionService path (VM reaches the same
  orchestration states a wizard-created VM does)
- `instantiated` BlueprintEvent + VmEvent logged
Overrides: passing memory/diskSizeGb wins over blueprint defaults.

## TC-007 — Admin flow regression
Wizard-style `POST /virtualmachine/create/{hostId}` and direct playbook attach/apply on a non-
blueprint VM behave exactly as before (responses byte-identical in shape; BlueprintId stays null).
