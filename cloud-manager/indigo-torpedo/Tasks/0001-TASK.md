# Task 0001 — Create bundle folder + write README.md

## Goal
Create the planning bundle directory and write its index README.

## Target repo
`planning` (writes under `cloud-manager/planning/planning/cloud-manager-ansible/`)

## Steps
1. Create directory `/home/todd/workspace/cloud-manager/planning/planning/cloud-manager-ansible/`
2. Write `cloud-manager-ansible/README.md` with these sections:
   - **Problem statement** — one paragraph. Why this bundle exists: the data
     model is shipped, but no API / UI / event-processor exists to manage
     ansible playbooks and roles end-to-end. This bundle is the cross-project
     design for closing that gap.
   - **Bundle contents** — bullet list linking to each of the four sibling
     plans (`API-PLAN.md`, `WEB-PLAN.md`, `VORCH-PLAN.md`,
     `DATA-MODEL-DELTAS.md`) with a one-line summary each.
   - **What's already shipped** — one paragraph naming the deployed entities
     (12 ansible-related tables in `vm` schema, `membership.feature_flags`),
     the `EntityPrefixRegistry` entries, and the live publisher
     (`IPlaybookRunPublisher` wired into `RunController.Apply`). Point at
     `../DATA-MODEL.md` for the ER diagram.
   - **Relationship to existing plans** — call out which sections of
     `../PLAYBOOK-INTEGRATION-PLAN.md` remain authoritative (feature-flag
     gating, vault path conventions, secrets strategy, run-log storage
     trade-offs, UI phasing) and which are superseded by this bundle
     (data-model additions, worker design — both replaced because they
     describe the git-backed shape and the Python worker recommendation).
     `../ANSIBLE-INSTALL-PLAN.md` remains fully authoritative.
   - **Implementation order** — recommend a sequence across the four plans
     (typically: data deltas first if any, then API, then web in parallel
     with vorch).

## Definition of done
`cloud-manager-ansible/README.md` exists and contains all five sections above
with concrete content (not placeholders).
