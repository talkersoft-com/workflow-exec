# Task 0001 — Metadata pipeline: C-VAR fetch + cache, identifier scan, decoration emission (cloud-manager-web repo)

## Steps
1. RESEARCH (read, no changes): contracts §7 (C-VAR) — the `vdef` record shape (`kind`:
   required/default/internal, `sensitivity`: plain/secret) and the definition endpoints P4
   shipped; contracts §9 (C-ED) — the `EditorDecoration` shape and `decorations` prop the P7
   editor accepts; the P7 Lezer grammar's exposed Jinja2 identifier nodes
   (`cloud-manager-web/src/components/AnsibleYamlEditor/`, read-only); how editor pages know
   their document owner (playbook/role) and revision.
2. Create `useVarDecorations(documentContext)` hook in cloud-manager-web:
   - Fetch C-VAR definitions for the document's owner closure (playbook/role → its `vdef`
     scopes) in ONE batched call; cache per revision so re-renders and re-scans don't refetch.
   - Scan the editor's parse tree for Jinja2 identifier nodes via the P7 grammar's node types
     and match against definitions by name. Matching is by C-VAR registration only — no
     name-pattern heuristics (`*_token`, `*_password` etc. must NOT influence classification).
   - Emit `EditorDecoration[]` in the exact C-ED shape: `kind: "secret"` for
     sensitivity=secret, `kind: "required"` for kind=required (the missing-state gutter
     amplification arrives with resolution data in later tasks), `kind: "vaultRef"` for
     `$secretRef` values and `vault_*` lookups.
   - Unregistered variables: no decoration; expose them on the hook result so a quiet tooltip
     hint can render (registration nudge — full harvest UX is P9, not here).
3. Re-scan cadence: debounced live re-scan on document change (300ms, matching lint behavior);
   plan open question 3 default.
4. Unit tests: batched-fetch shape (one call per owner closure per revision), cache hit on
   unchanged revision, identifier matching (registered → correct kind; unregistered → none;
   no heuristic leakage), `EditorDecoration[]` conforms to the C-ED type.
5. Build: `python3 .cicd/build-cloud-manager.py --target web` must pass.

Done when: the hook produces correct C-ED decorations from C-VAR metadata for a document
context, cached per revision, with unregistered names surfaced separately; web build clean.
No editor-component or API changes.
