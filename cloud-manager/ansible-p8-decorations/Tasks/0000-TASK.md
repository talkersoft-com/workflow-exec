# Task 0000 — Initialize Workspace

## Goal
Verify the workspace is clean, verify the dependency series (P4, P5, P7) is merged, and create
the feature branch across all repos in the deck.

## Steps
1. Run `hv_status` on the deck (see `../deck.md` for the exact call)
2. Verify prerequisites are merged to main: P4 (C-VAR), P5 (C-RES), and P7 (C-ED) PRs.
   Concretely: `AnsibleYamlEditor` with `decorations`/`hoverProvider` props exists in
   `cloud-manager-web` on origin/main, the C-VAR definition endpoints and the C-RES
   `POST api/v1/resolution/preview` endpoint respond on the deployed API. If any prerequisite
   is missing → STOP and surface to operator (this is the P5-blocked-on-P4 failure mode again).
3. If all repos are clean and on the default branch → run `hv_init`
4. If all repos are clean and on a feature branch with all PRs merged → run `hv_next`
5. If any repo is dirty → STOP. Investigate before proceeding.
   - Exception: dirty repo with a merged PR → hv_stash → hv_next → hv_unstash
6. Run `hv_status` again to confirm all repos are on the new feature branch, all clean
7. Record the generated branch name in `../deck.md`

## Definition of done
All repos on the new feature branch, all clean. P4/P5/P7 contract surfaces verified present.
Branch name recorded in deck.md.
