# Task 0003 — Hover provenance: context bar, C-RES calls, provenance tooltip (cloud-manager-web repo)

## Steps
1. RESEARCH (read, no changes): contracts §8 (C-RES) — the `POST api/v1/resolution/preview`
   request/response: per-variable state, masked `effectiveValue`, winner + overridden chain;
   the P7 editor's `hoverProvider` extension point (C-ED hoverTooltip bridge); the inventory
   and host list endpoints needed by the pickers.
2. Context bar: an optional bar on the editor pages with inventory + host pickers selecting
   the operation context for resolution. Last chosen context is remembered per playbook via
   localStorage (plan open question 1 default); starts restored when present, empty otherwise.
3. `hoverProvider` implementation supplied via the C-ED prop:
   - On hover over a matched identifier WITH a context selected: call C-RES preview for the
     page's operation context — debounced, response cached per context (one preview call
     serves all hovers in that context until doc revision or context changes).
   - Render: winner (e.g. `host_vars → hvar_… level 14`), the overridden chain, and
     `effectiveValue` — for secrets this is the engine's literal `"***"` plus the `sref_…`
     provenance. The client NEVER receives or displays cleartext (contracts §8 invariant);
     do not post-process the masked value.
   - WITHOUT a context: hover shows classification + definition info only — NO resolution
     call is made.
   - `missingRequired` from the preview response now feeds Task 0002's bold gutter dot for
     required-and-missing variables in the selected context.
4. Tests: hover-with-context renders winner/chain/masked value from a mocked preview
   response; hover-without-context makes zero resolution calls; per-context cache means
   repeated hovers trigger no duplicate calls; secret hover shows `***` (assert cleartext
   absent from DOM); context persistence round-trips localStorage per playbook.
5. Build: `python3 .cicd/build-cloud-manager.py --target web` must pass.

Done when: hovering a variable with a chosen context shows effective value (secrets masked)
plus the winner/overridden provenance chain from C-RES; no context → definition info only;
required-missing gutter amplification live; web build clean.
