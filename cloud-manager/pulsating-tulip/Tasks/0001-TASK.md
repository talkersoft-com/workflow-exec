# Task 0001 — Restructure ClaudeSettings to Claude-native schema

## Target repo
`hive-deck-pro` (outside the cloud-manager deck — commit manually per `../deck.md`)

## Steps

### 1. `internal/config/config.go`
Replace the flat `Allow` and `DefaultMode` fields with a nested `Permissions` struct:

```go
type ClaudePermissions struct {
    Allow       []string `yaml:"allow"                  json:"allow"`
    Deny        []string `yaml:"deny,omitempty"         json:"deny,omitempty"`
    DefaultMode string   `yaml:"defaultMode,omitempty"  json:"defaultMode,omitempty"`
}

type ClaudeSettings struct {
    Enabled        bool              `yaml:"enabled"`
    ForceOverwrite *bool             `yaml:"force_overwrite"`
    Permissions    ClaudePermissions `yaml:"permissions"`
}
```

### 2. `internal/claude/claude.go`
Rewrite `writeSettingsFile`/`mergePermissions` to use `cs.Permissions` directly:
- When `force_overwrite` is true: write `{"permissions": <marshal cs.Permissions>}` directly — no field-by-field building
- When false: read existing file, merge only allow entries additively (keep deny and defaultMode from existing file unless absent)

### 3. `.hv/config.yaml.example`
Update `claude_settings` block:
```yaml
claude_settings:
  enabled: false
  force_overwrite: true
  permissions:
    allow:
      # - "Bash(*)"
      - "Edit(*)"
      - "Write(*)"
      - "Read(*)"
      - "Glob(*)"
      - "Grep(*)"
      - "WebFetch(*)"
      - "WebSearch(*)"
    # defaultMode: bypassPermissions   # uncomment to bypass all Claude permission prompts
```

### 4. `~/.hv/config.yaml`
Update `claude_settings` to the same new shape, moving `allow` under `permissions:` and removing `default_mode`.

## Definition of done
`ClaudeSettings` has no `Allow` or `DefaultMode` at top level. `config.yaml.example` uses `permissions:` subtree. `~/.hv/config.yaml` updated to match.
