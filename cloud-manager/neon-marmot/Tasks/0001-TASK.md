# Task 0001 — Implement force_overwrite in hive-deck-pro

## Goal
Add `ForceOverwrite bool` to `ClaudeSettings`, update `MaybeWrite` to use it, and document the field in `config.yaml.example`.

## Target repo
`hive-deck-pro` (outside the cloud-manager deck — commit manually per `../deck.md` Manual steps)

## Steps

### 1. `internal/config/config.go`
Add `ForceOverwrite bool \`yaml:"force_overwrite"\`` to the `ClaudeSettings` struct.

### 2. `internal/claude/claude.go`
Update `writeSettingsFile` so that when `cs.ForceOverwrite` is `true` (or unset — treat the zero value as `true`), the function skips reading the existing file and writes the full config-derived content directly. When `false`, keep the current read-merge-write path.

The struct zero value for bool is `false`. To make `true` the default when the field is omitted from YAML, use a pointer (`*bool`) or a custom unmarshal. The simplest approach: add a `ForceOverwrite *bool` pointer field; a `nil` pointer means "not set" → treat as `true`.

### 3. `.hv/config.yaml.example`
Add the following under `claude_settings`, after `enabled:`:
```yaml
  force_overwrite: true   # when true, always replace settings.local.json on init (default); false = additive merge
```

## Definition of done
All three files updated. `ForceOverwrite` nil or true → file replaced. `ForceOverwrite` false → additive merge preserved.
