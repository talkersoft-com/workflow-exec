# Task 0007 — Write Results + LESSONS, then hv_ship

## Goal
Capture outcome + retrospective in files that get included in the PR,
then ship.

## Steps
1. **Write `Results/RESULT.md`** — top of file MUST include this operator
   instruction verbatim:

   > **Operator action required:** Restart the cloud-manager-mcp via
   > `/mcp` in Claude Code to pick up the new dist. The three new tools
   > (`cloud_vm_update`, `cloud_vm_ip_address_update`, `cloud_vm_events`)
   > only become callable after that restart.

   Then standard sections: Outcome (SHIPPED / PARTIAL / FAILED), Branch,
   Pull Requests (TBD until hv_ship returns), Phase summary (one
   paragraph per phase), Deploy log (smoke-test results from Task 0006).

2. **Write `Retro/LESSONS.md`** — what went smoothly, what was friction
   (specific missing tool, ambiguous instruction, false-positive test).
   List FIX files in the Retro/ dir. If none were needed, say so in
   one line.

3. **Call `hv_ship`:**
   ```
   hv_ship  deck: "cloud-manager"
            message: "mcp-coverage-tier-1: cloud_vm_update + ip_address_update + events"
            title:   "mcp coverage tier 1: 3 new VM tools (update, ip-address, events)"
   ```

4. After hv_ship returns, fill the PR table in `Results/RESULT.md` with
   the returned PR URLs.

## Definition of done
- RESULT.md exists, contains the operator-restart notice at the top.
- LESSONS.md exists.
- hv_ship returned a merged PR for cloud-manager-mcp.
- Deck transitioned to next branch via auto-merge mode.
