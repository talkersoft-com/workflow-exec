# Task 0001 — Wrapper hardening (camelCase + 4xx surfacing)

## Goal
Ensure the shared API-client wrapper in cloud-manager-mcp refuses to send
snake_case keys to cloud-manager-api and surfaces 4xx response bodies cleanly
to MCP callers.

## Steps
1. Open `cloud-manager-mcp/src/runner.ts` (or the shared HTTP client module
   it delegates to — find the single point where the body is serialised
   and the response is unwrapped).
2. **camelCase guard:** before serialising the request body, walk top-level
   keys and throw a TypeError naming any key containing `_`. Example:
   `if (/_/.test(k)) throw new TypeError("snake_case key on the wire: " + k)`.
3. **4xx surfacing:** when `response.ok === false`, await `response.text()`
   (or json if Content-Type is JSON), build an error that includes status +
   body, and throw it. Do NOT swallow the body.
4. If either guard is already present and behaves as above, mark the
   sub-step done with no code change.

## Files touched
- `cloud-manager-mcp/src/runner.ts` (and/or sibling HTTP-client module)

## Definition of done
- A request body containing a snake_case key throws before hitting the
  network.
- A 4xx response surfaces the body text in the thrown error message.
- `npm run build` clean.
