# Task 0004 — Implement `cloud_vm_events`

## Goal
Expose `GET /api/v1/VirtualMachine/{publicId}/events?take=100` as an MCP tool.

## Steps
1. Open `cloud-manager-mcp/src/tools/vms.ts`.
2. Add `cloud_vm_events` with input schema:
   - `publicId: string` (required)
   - `take?: number` (default 100, max 500 — clamp client-side; the API
     also enforces the cap)
3. Build the URL with the query string: `?take=${take}` (omit if default).
4. Call: `GET /api/v1/VirtualMachine/${publicId}/events`.
5. Return the response JSON as-is — an array of `VmEvent` DTOs ordered
   `occurred_at` DESC.
6. Register in `cloud-manager-mcp/src/index.ts`.
7. Description should mention "audit log / lifecycle timeline" so the LLM
   reaches for it during incident triage.
8. `npm run build` — clean.

## Definition of done
- Tool registered, builds clean.
- `take > 500` is clamped to 500 client-side (avoids a needless 4xx).
