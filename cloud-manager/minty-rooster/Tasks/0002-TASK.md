# Task 0002 — Implement `cloud_vm_update`

## Goal
Expose `PATCH /api/v1/VirtualMachine/{publicId}` as an MCP tool.

## Steps
1. Open `cloud-manager-mcp/src/tools/vms.ts`.
2. Add `cloud_vm_update` with input schema:
   - `publicId: string` (required, sent in the URL only, NOT in the body)
   - `name?: string`
   - `description?: string`
   - `memory?: number`
   - `diskSizeGb?: number`
   - additional mutable fields the existing PATCH DTO accepts —
     mirror the API DTO field list, camelCase only.
3. Strip `publicId` before serialising the body.
4. Call the shared API client: `PATCH /api/v1/VirtualMachine/${publicId}`.
5. Return the response JSON unchanged.
6. Map 404 → propagate the API's body; other 4xx → propagate per the
   wrapper's standard 4xx surfacing from Task 0001.
7. Register the tool in `cloud-manager-mcp/src/index.ts` (or whichever
   barrel file lists tools).
8. `npm run build` — must be clean.

## Definition of done
- Tool registered, builds clean.
- Calling with a known good `publicId` and `{ name: "renamed" }` returns
  the updated DTO (verified manually against running API in Task 0006).
