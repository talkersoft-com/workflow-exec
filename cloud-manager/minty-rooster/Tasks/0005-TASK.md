# Task 0005 — Coverage assertion

## Goal
Confirm every endpoint on the `VirtualMachine` controller either has a
matching MCP tool or is explicitly documented as intentionally excluded.

## Steps
1. List endpoints from
   `cloud-manager-api/src/CloudManager.API/Controllers/VirtualMachineController.cs`
   (one row per `[HttpGet]/[HttpPost]/[HttpPatch]/[HttpDelete]` attribute).
2. List MCP tool names from
   `cloud-manager-mcp/src/tools/vms.ts`.
3. For each endpoint, match it to:
   - an existing tool, OR
   - an entry in the "intentionally excluded" list in this task file
     (currently: `cloud_vm_retry_teardown`, `cloud_run_get_secret`).
4. Update `cloud-manager-mcp/README.md` (or `COVERAGE.md` if it exists) with
   a short table of endpoint → tool / excluded mappings. If neither file
   exists, create `cloud-manager-mcp/COVERAGE.md` with that table.
5. Any endpoint with no tool AND no excluded-list entry → failure. Stop
   and surface to operator.

## Intentionally excluded
- `POST /api/v1/VirtualMachine/{publicId}/retry-teardown` — operator UI only.
- `GET /api/v1/Run/{publicId}/secret` — would leak Vault tokens; never in
  MCP surface.

## Definition of done
- 11 of 12 VM-controller endpoints have a tool (after Tier 1 adds 3).
- 1 (retry-teardown) is explicitly listed as excluded.
- Coverage table committed.
