# Test 0001 — Wrapper hardening

## TC-001 — snake_case rejected before network
Write a one-off node script (or inline jest test) that calls the wrapper
with a body like `{ public_id: "x" }`.
- Pass: throws TypeError mentioning the offending key. No HTTP request made.
- Fail: request goes out, or wrapper coerces the key silently.

## TC-002 — 4xx body surfaces in error message
Stub fetch to return `{ ok: false, status: 400, text: () => "validation: name required" }`.
- Pass: thrown error message contains both `400` and `validation: name required`.
- Fail: error message is generic (e.g. just `Request failed`) or omits the body.

## TC-003 — Build is clean
Run `npm run build` in `cloud-manager-mcp/`.
- Pass: exit 0, no TS errors.
- Fail: any compile error.
