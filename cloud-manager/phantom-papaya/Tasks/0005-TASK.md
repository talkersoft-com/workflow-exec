# Task 0005 — Finalize Results + Lessons; ship

## Goal
Finalize `Results/RESULT.md`, write `Retro/LESSONS.md`, then call `hv_ship` to open the PR(s).

## Target repo(s)
`workflow-exec` (always), plus any repo touched in Task 0003.

## Steps

1. **Finalize `Results/RESULT.md`** (if not already complete from Task 0004) with:
   - Outcome: SHIPPED | PARTIAL | FAILED
   - Branch: `phantom-papaya`
   - PRs: leave as `TBD — filled from hv_ship output` (this task fills them in after ship)
   - Phase summary, one short paragraph per phase

2. **Write `Retro/LESSONS.md`** with:
   - What would have helped (be specific — missing MCP call, ambiguous task, missing doc, false-positive test)
   - List of FIX files written (if any)
   - If the bug was a regression: which PR introduced it and what could have caught it (test, lint, CI check)

3. **Run `hv_status`** — verify only the expected repos are dirty (the touched code repo(s) + `workflow-exec`).

4. **Run `hv_ship`** with:
   ```
   hv_ship  deck: "cloud-manager"
            message: "fix: VM creation error in cloud-manager"
            title:   "fix: VM creation error in cloud-manager"
   ```

5. **Update `Results/RESULT.md`** with the PR URLs from the ship output. (`hv_ship` advances the branch automatically; the file edit goes onto the next branch and is fine to leave as the first commit there — or you may skip if the URLs land in conversation history.)

6. **Stop.** Do not start the next task.

## Definition of done
- `Results/RESULT.md` populated with outcome + phase summaries
- `Retro/LESSONS.md` populated
- `hv_ship` returned successfully, PR(s) merged or open
- All ORCH.md task checkboxes are checked
