# Task 0003 — Implement the minimal fix

## Goal
Apply the change specified in `Retro/DIAGNOSIS.md` to the responsible repo(s) and verify the local build is clean. No deployment in this task — that's Task 0004.

## Target repo(s)
Determined by `Retro/DIAGNOSIS.md`. One of (or a subset of):
- `cloud-manager-web`
- `cloud-manager-api`
- `vorch-service`
- `vorch-lib`

## Steps

1. **Read `Retro/DIAGNOSIS.md`** and confirm the offending file:line and proposed minimal change.

2. **Apply the edit.** Use `Edit` (not `Write`) on the named file. The change must:
   - Be the minimum needed to fix the bug
   - Not introduce silent fallbacks, disabled feature flags, or "while we're here" cleanups
   - Surface real errors (rather than swallow them) if the failure was due to a swallowed error

3. **Per-repo verification commands.** Run the verifier(s) matching each touched repo:

   - **`cloud-manager-web`**:
     ```
     cd vm-infra/cloud-manager/cloud-manager-web
     npx tsc --noEmit
     npm run build
     ```
     Expected: zero TS errors; build emits `dist/`.

   - **`cloud-manager-api`**:
     ```
     cd vm-infra/cloud-manager/cloud-manager-api/src
     dotnet build CloudManager.sln -c Release
     ```
     Expected: `Build succeeded. 0 Error(s)`.

   - **`vorch-service`**:
     ```
     cd vm-infra/cloud-manager/vorch-service/vorch-service
     go build ./...
     go vet ./...
     ```
     Expected: no output (clean) on both.

   - **`vorch-lib`**:
     ```
     cd vm-infra/cloud-manager/vorch-lib
     go build ./...
     go vet ./...
     ```
     Then also build `vorch-service` (the consumer that uses it via local replace).

4. **Write a one-line note in `Retro/CHANGE.md`** summarizing what was edited:
   ```
   <repo>:<file>:<line> — <one-line description of the change>
   ```
   One line per edit. This drives the PR description at ship time.

## Definition of done
- The edit(s) named in `DIAGNOSIS.md` are applied.
- All verifiers for touched repos pass clean.
- `Retro/CHANGE.md` exists with one line per file changed.
- No edits to repos NOT named in `DIAGNOSIS.md`.
