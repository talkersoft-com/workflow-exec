# Task 0001 — Reproduce + capture the VM-creation failure

## Goal
Trigger the bug end-to-end on `ubuntu-server.talkersoft.com` and capture the verbatim error message, HTTP status, stack traces, and journal excerpts from the responsible services. Output: `Retro/REPRODUCTION.md` with all artifacts referenced.

## Target repo(s)
None (capture-only phase — no source changes).

## Steps

1. **Pre-flight: confirm services are up.**
   - `sudo systemctl is-active cloud-manager-api` → expect `active`
   - `sudo systemctl is-active vorch_service` → expect `active`
   - `sudo systemctl is-active cloud-manager-web` → expect `active`
   - `sudo systemctl is-active rabbitmq-server` → expect `active`
   - If any are inactive, restart and capture journal — that may itself be the bug.

2. **Snapshot the baseline journals** so the reproduction window is bounded.
   - Note the current time (`date -Iseconds`). Save as `Retro/repro-start.txt`.

3. **Trigger the failure via the UI.**
   - Use Playwright MCP tools to navigate to `https://ubuntu-server.talkersoft.com/create`.
   - Fill the form with minimal valid values:
     - Name: `repro-vm-<short-timestamp>` (e.g. `repro-vm-1530`)
     - Host: first available
     - Image: first available
     - Memory: default
     - Disk: default
     - Description: `phantom-papaya reproduction`
   - Submit.
   - Capture:
     - `mcp__playwright__browser_take_screenshot` of the error state → `Retro/screenshot-error.png`
     - `mcp__playwright__browser_network_requests` → save the POST to `/api/v1/virtualmachine/create/*` (URL, request body, response status, response body) into `Retro/network.json` or `Retro/network.md`
     - `mcp__playwright__browser_console_messages` → save any errors to `Retro/console.txt`

4. **If the UI reproduction is blocked** (e.g. cannot reach the form, no images available, login required, etc.), fall back to a direct API curl:
   - Query the DB for an available `host_id` and `image_id`:
     ```
     mcp__database-toolkit-mcp__db_execute_sql_query
       database: clouddb
       sql: "SELECT host_id, name FROM vm.hosts LIMIT 1; SELECT virtual_machine_image_id, name FROM vm.virtual_machine_images LIMIT 1;"
     ```
   - Then:
     ```
     curl -i -X POST https://ubuntu-server.talkersoft.com/api/v1/virtualmachine/create/<host_id> \
       -H 'Content-Type: application/json' \
       -d '{"vmi":"<image_id>","virtualMachine":{"name":"repro-vm-1530","description":"phantom-papaya","memory":"2048","diskSizeGb":20}}'
     ```
   - Save full curl output (including headers) to `Retro/curl.txt`.

5. **Capture server-side journals** bracketing the reproduction window.
   - `sudo journalctl -u cloud-manager-api --since "$(cat Retro/repro-start.txt)" --no-pager` → `Retro/api.log`
   - `sudo journalctl -u vorch_service --since "$(cat Retro/repro-start.txt)" --no-pager` → `Retro/vorch.log`
   - `sudo journalctl -u rabbitmq-server --since "$(cat Retro/repro-start.txt)" --no-pager` → `Retro/rmq.log`

6. **If the API returned 5xx with no stack trace**, increase log verbosity:
   - Inspect `/etc/systemd/system/cloud-manager-api.service` for `ASPNETCORE_ENVIRONMENT`.
   - If `Production`, set to `Development` temporarily (`sudo systemctl edit cloud-manager-api`), reload + restart, re-trigger, then revert.
   - This is the only systemd change permitted in this task.

7. **Check whether ANY VM has been created recently** to establish whether this is a regression or has always been broken in this codebase state:
   ```
   mcp__database-toolkit-mcp__db_execute_sql_query
     database: clouddb
     sql: "SELECT virtual_machine_id, name, orchestration_status, created FROM vm.virtual_machines ORDER BY created DESC LIMIT 10;"
   ```
   Save result to `Retro/recent-vms.md`.

8. **Write `Retro/REPRODUCTION.md`** with:
   - Reproduction steps (exact UI clicks or curl command)
   - Verbatim error visible to the user
   - HTTP status + response body
   - Top relevant lines from each journal (don't dump everything — extract the lines that mention the failed request)
   - One-paragraph hypothesis about which layer emitted the error
   - List of all captured artifact files

## Definition of done
- `Retro/REPRODUCTION.md` exists and includes verbatim error + HTTP status + a one-paragraph hypothesis.
- All referenced artifact files exist in `Retro/`.
- The original ASPNETCORE_ENVIRONMENT is restored (if it was changed).
- No source code under `vm-infra/cloud-manager/` has been modified.
