# Test 0004 — Deployed + VM creation succeeds

## TC-001 — All touched services active
Run:
```
sudo systemctl is-active cloud-manager-api vorch_service cloud-manager-web
```
Pass: `active` for all three.
Fail: any is `failed` / `inactive`.

## TC-002 — VM-creation request now succeeds
Repeat the Task-0001 reproduction (UI or curl). Capture the new POST response.
Pass: HTTP 2xx; response body contains `virtualMachineId`; no UI error.
Fail: same error as before, or a new error.

## TC-003 — VM row exists in the database
Query clouddb:
```
SELECT virtual_machine_id, name, orchestration_status, created
FROM vm.virtual_machines
WHERE name = '<post-fix-repro-name>'
ORDER BY created DESC LIMIT 1;
```
Pass: row exists; `orchestration_status` is one of `Initialized`, `Started`, `Booting`, `Completed`.
Fail: no row, or status is `Failed` / `Error`.

## TC-004 — vorch consumed the message
Run:
```
sudo journalctl -u vorch_service --since "2 minutes ago" | grep -iE "create-vm|received message" | tail -20
```
Pass: at least one line confirms vorch received and processed the create-vm message for the new VM.
Fail: vorch log shows no consumption (= AMQP publish issue) or shows a panic / error for this VM.

## TC-005 — Results/RESULT.md populated
Run:
```
test -s Results/RESULT.md && grep -iE "outcome|after" Results/RESULT.md
```
Pass: file exists, is non-empty, contains an Outcome line and references the post-fix evidence.
Fail: file missing, empty, or lacks Outcome.
