# Task 0001 — Part A record entities + migration (cloud-manager-api)

## Steps
1. RESEARCH (read, no changes): existing entity patterns in
   `cloud-manager-api/src/Models/CloudManager.Entities` — `PlaybookRevision` (`pbrev`) and how
   revisions are cut on save, `AnsibleRole`/`PlaybookRole` + role file revisions (`rfrev`),
   `VmEvent` (append-only pattern), `EntityPrefixRegistry`, EF configurations, the P1 `ansible`
   schema entities (inventory records the `targets` edges will reference).
2. Add four entities in schema `ansible`, exactly per contracts §1 and PLAN.md Part A:
   - `PlaybookPlay` (`play`) — FK PlaybookRevision (Cascade); Position; Name; HostsPattern;
     raw attrs jsonb
   - `PlaybookTask` (`ptask`) — FK to play OR handler parent; Position; Name; ModuleName;
     args jsonb; SourceSpan (start line/col, end line/col against the revision's YAML)
   - `PlaybookHandler` (`hdlr`) — FK PlaybookRevision; Name; tasks hang off it
   - `VarBlock` (`vblk`) — FK owner (play / role via owner type+id); Kind:
     `play_vars` | `role_defaults` | `role_vars`; entries jsonb
3. SourceSpan must be present on `PlaybookTask` — P8 anchors editor decorations to it.
4. Add a `decompositionState` marker on the revision (or companion record) so a parse failure
   can be recorded without blocking the save (consumed by Task 0002).
5. Register prefixes `play` `ptask` `hdlr` `vblk` in `EntityPrefixRegistry` (collision-check
   first).
6. Add the EF migration for the four tables. Do NOT add the `edge` entity here — that is
   Task 0003's migration.
7. Build: `python3 .cicd/build-cloud-manager.py --target api`

Done when: build passes; migration compiles and contains the four `ansible` schema tables with
the FKs and cascade behavior from PLAN.md; prefixes registered with no collisions.
