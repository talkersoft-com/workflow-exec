# Task 0002 — Part A decomposer service (cloud-manager-api)

## Steps
1. RESEARCH (read, no changes): every code path that cuts a new `PlaybookRevision` or
   `RoleFileRevision` today (playbook create/update, role file create/update/restore); how
   YamlDotNet is already used in the repo (existing dependency — do not add a new parser).
2. Add `IPlaybookDecomposer` + implementation:
   - Parse the revision's YAML with YamlDotNet.
   - Rebuild that revision's `play`/`ptask`/`hdlr`/`vblk` records **in one transaction**
     (delete-and-recreate for the revision is acceptable — records are a derived read model).
   - Populate SourceSpan from the parser's line/col marks.
   - Decompose BOTH playbook revisions and role file revisions (tasks/handlers inside roles) —
     used-by is weak without role task records (PLAN.md open question 2, default: both).
3. Hook the decomposer into every save path found in step 1, after the revision row is written.
4. Parse failure must NOT block the save: the blob is still stored; mark the revision
   `decompositionState: failed` and surface that state in the relevant GET responses.
5. Records are NEVER recomposed into YAML — no write path from records back to the blob.
6. Build: `python3 .cicd/build-cloud-manager.py --target api`

Done when: build passes; saving a playbook or role file rebuilds its records transactionally;
a deliberately malformed YAML save still succeeds and marks `decompositionState: failed`.
