# Test 0010 — Build + deploy + migrate + backfill

## TC-001 — Migration applied
```bash
PGPASSWORD='P@ssw0rd!' psql -h localhost -U cloudmanager -d clouddb -c "\d vm.virtual_machines" | grep ip_address
```
Pass: shows `ip_address | character varying(45) | ...`.

## TC-002 — Backfill populated rows
```bash
PGPASSWORD='P@ssw0rd!' psql -h localhost -U cloudmanager -d clouddb -tAc "SELECT count(*) FROM vm.virtual_machines WHERE ip_address IS NOT NULL"
```
Pass: count > 0.

## TC-003 — All services active
```bash
systemctl is-active porch && systemctl is-active vorch_service && systemctl is-active cloud-manager-api
```
Pass: all print `active`.

## TC-004 — Porch unit has ProtectHome=yes + ANSIBLE_RUNNER_BIN
```bash
systemctl cat porch.service | grep -E 'ProtectHome|ANSIBLE_RUNNER_BIN'
```
Pass: shows `ProtectHome=yes` and `Environment="ANSIBLE_RUNNER_BIN=..."`.

## TC-005 — ansible-runner is system-installed
```bash
which ansible-runner
```
Pass: path is under `/usr/bin/` or `/usr/local/bin/`.
Fail: shows a /home/.../.local path or a /usr/local/bin/ansible-runner.real shim.

## TC-006 — No recent exceptions
```bash
sudo journalctl -u porch -u vorch_service -u cloud-manager-api --since "60 seconds ago" --no-pager | grep -iE 'error|exception|panic|fatal' | grep -v 'feature-flag failed' | head -5
```
Pass: zero or only known-benign.

## TC-007 — Single consumer on playbook-runs (no regression)
```bash
sudo rabbitmqctl list_consumers | grep playbook-runs | wc -l
```
Pass: `1`.
