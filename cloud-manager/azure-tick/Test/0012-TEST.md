# Test 0012 — End-to-end verification

The acceptance criteria from PLAN.md, one per TC. Replace `<runId>` with the actual new runId.

## TC-001 — Run reaches Succeeded
```sql
SELECT status FROM vm.playbook_runs WHERE public_id = '<runId>';
```
Pass: `status = 3`.

## TC-002 — Vault columns
```sql
SELECT vault_secrets_prefix, vault_run_token IS NULL AS cleared FROM vm.playbook_runs WHERE public_id = '<runId>';
```
Pass: prefix = `cloudmanager/data/playbook-secrets/<runId>`, cleared = t.

## TC-003 — Assignment row updated
```sql
SELECT last_run_status, last_applied_at FROM vm.vm_playbook_assignments
WHERE last_run_id = (SELECT id FROM vm.playbook_runs WHERE public_id = '<runId>');
```
Pass: last_run_status = 3, recent timestamp.

## TC-004 — Vault KV has postgres credentials
```bash
VAULT_TOKEN=<root> vault kv get cloudmanager/playbook-secrets/<runId>/postgres-test/postgres
```
Pass: fields `host`, `port`, `db`, `user`, `password` all present.

## TC-005 — postgres-16 installed on the VM
```bash
ssh cloudmanager@<postgres-test-ip> 'dpkg -l postgresql-16'
```
Pass: package shown as `ii` (installed).

## TC-006 — app DB + appuser exist
```bash
ssh cloudmanager@<postgres-test-ip> 'sudo -u postgres psql -l | grep -E "^\s*app\s*\|"'
ssh cloudmanager@<postgres-test-ip> 'sudo -u postgres psql -tAc "SELECT 1 FROM pg_roles WHERE rolname=\"appuser\""'
```
Pass: `app` listed; `1` returned for the role.

## TC-007 — Password roundtrip (Vault → psql works)
```bash
PWD=$(VAULT_TOKEN=<root> vault kv get -field=password cloudmanager/playbook-secrets/<runId>/postgres-test/postgres)
ssh cloudmanager@<postgres-test-ip> "PGPASSWORD=$PWD psql -h localhost -U appuser -d app -tAc 'SELECT 1'"
```
Pass: `1`.

## TC-008 — No hvs.* leaks in porch journal

sudo journalctl -u porch --since "10 minutes ago" | grep -E 'hvs\.[A-Za-z0-9]' | head -3
```
Pass: zero matches.

## TC-009 — No hvs.* leaks in run log file

sudo grep -E 'hvs\.[A-Za-z0-9]' /kvm-automator/playbook-logs/<runId>.log | head -3
```
Pass: zero matches.

## TC-010 — Single consumer on playbook-runs

sudo rabbitmqctl list_consumers | grep playbook-runs | wc -l

systemctl cat porch.service | grep -E 'ProtectHome|ANSIBLE_RUNNER_BIN'
