# Task 0010 — Build + deploy + migrate + backfill

## Goal
Roll everything out to the running services. Apply the EF migration. Backfill VM IPs. Reinstall porch (which re-runs the install script — apt or pipx --global ansible-runner + ProtectHome=yes).

## Target repo(s)
(no code changes — rollout only)

## Steps
1. **Publish API**:
   ```bash
   cd cloud-manager-api
   dotnet publish src/CloudManager.API/CloudManager.API.csproj -c Release -o /tmp/cm-api-azure-tick --nologo -v q
   sudo systemctl stop cloud-manager-api
   sudo cp -r /tmp/cm-api-azure-tick/* /opt/cloud-manager-api/
   ```
2. **Apply migration**:
   ```bash
   cd cloud-manager-api/src/Models/CloudManager.Entities
   dotnet ef database update --context CloudManagerDbContext
   ```
   Verify with:
   ```bash
   PGPASSWORD='P@ssw0rd!' psql -h localhost -U cloudmanager -d clouddb -c "\d vm.virtual_machines" | grep ip_address
   ```
3. **Start API**:
   ```bash
   sudo systemctl start cloud-manager-api
   sleep 3
   curl -s http://localhost:5250/api/v1/FeatureFlags | head -3
   ```
4. **Backfill VM IPs**:
   ```bash
   sudo python3 cloud-manager-api/scripts/porch/backfill-vm-ip-addresses.py
   ```
   Should print one line per VM with the new IP. Skipped VMs are documented in `Retro/CHANGE-0010.md`.
5. **Rebuild + reinstall porch** (runs the updated install script with ansible-runner ensure + ProtectHome=yes):
   ```bash
   sudo systemctl stop porch
   sudo python3 cloud-manager-api/scripts/porch/install-porch-service.py
   ```
6. **Rebuild + redeploy vorch-service** (the IP writeback change):
   ```bash
   cd vorch-service/vorch-service
   go build -o /tmp/vorch-azure-tick ./cmd/vorch-service
   sudo systemctl stop vorch_service
   sudo cp /tmp/vorch-azure-tick /kvm-automator/vorch-service
   sudo systemctl start vorch_service
   ```
7. **Start porch**:
   ```bash
   sudo systemctl start porch
   sleep 3
   systemctl status porch | head -10
   sudo journalctl -u porch --since "10s ago" --no-pager | head -10
   ```

## Definition of done
- API published + restarted, migration applied.
- `vm.virtual_machines` has `ip_address` column.
- Backfill script ran; non-NULL `ip_address` count > 0 (records exceptions in CHANGE-0010).
- Porch reinstalled with `ProtectHome=yes`; `systemctl cat porch.service | grep -E "ProtectHome|ANSIBLE_RUNNER_BIN"` shows both.
- All three services (porch, vorch_service, cloud-manager-api) `active`.
- No exceptions in journals over the last 60 seconds.
