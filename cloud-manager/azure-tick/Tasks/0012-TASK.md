# Task 0012 — End-to-end verification

## Goal
Prove the full pg-on-postgres-test demo works: trigger from UI → postgres-16 installed on the VM → app DB + appuser exist → password is at the expected Vault KV path → no `hvs.*` leaks anywhere.

## Target repo(s)
(no code changes — live verification only)

## Steps
1. Navigate the UI to `/ansible/run?playbookId=pb_EMEJCH08GG`. Select `postgres-test`. Click Run.
2. Wait for terminal status (Succeeded). Record the new runId, e.g. `run_XXXXXXXXXX`.
3. Verify run row:
   ```sql
   SELECT public_id, status, vault_secrets_prefix, vault_run_token IS NULL AS cleared
   FROM vm.playbook_runs WHERE public_id = '<runId>';
   ```
   Pass: `status = 3`, prefix = `cloudmanager/data/playbook-secrets/<runId>`, `cleared = t`.
4. Verify assignment row updated:
   ```sql
   SELECT last_run_status, last_applied_at FROM vm.vm_playbook_assignments
   WHERE last_run_id = (SELECT id FROM vm.playbook_runs WHERE public_id = '<runId>');
   ```
   Pass: `last_run_status = 3`, recent timestamp.
5. Verify Vault KV (operator with root token):
   ```bash
   VAULT_TOKEN=<root> vault kv get cloudmanager/playbook-secrets/<runId>/postgres-test/postgres
   ```
   Pass: fields `host`, `port`, `db`, `user`, `password` present.
6. Verify postgres installed on the VM:
   ```bash
   POSTGRES_IP=$(PGPASSWORD='P@ssw0rd!' psql -h localhost -U cloudmanager -d clouddb -tAc "SELECT ip_address FROM vm.virtual_machines WHERE name='postgres-test'")
   ssh -o StrictHostKeyChecking=no cloudmanager@$POSTGRES_IP "dpkg -l postgresql-16 | tail -1 && sudo -u postgres psql -l | grep -E 'app|appuser'"
   ```
   Pass: postgresql-16 listed as ii (installed), `app` DB present.
7. Verify password roundtrip:
   ```bash
   PWD_FROM_VAULT=$(VAULT_TOKEN=<root> vault kv get -field=password cloudmanager/playbook-secrets/<runId>/postgres-test/postgres)
   ssh cloudmanager@$POSTGRES_IP "PGPASSWORD=$PWD_FROM_VAULT psql -h localhost -U appuser -d app -c 'SELECT 1;'"
   ```
   Pass: query returns `1`.
8. Verify no `hvs.*` leaks:
   ```bash
   sudo journalctl -u porch --since "10 minutes ago" | grep -E 'hvs\.[A-Za-z0-9]'
   sudo grep -E 'hvs\.[A-Za-z0-9]' /kvm-automator/playbook-logs/<runId>.log
   ```
   Both pass: zero matches.
9. Verify single consumer on `playbook-runs`:
   ```bash
   sudo rabbitmqctl list_consumers | grep playbook-runs | wc -l
   ```
   Pass: `1`.
10. Verify `porch.service` config:
    ```bash
    systemctl cat porch.service | grep -E 'ProtectHome|ANSIBLE_RUNNER_BIN'
    ```
    Pass: `ProtectHome=yes` AND `ANSIBLE_RUNNER_BIN=` set.

## Definition of done
All 10 checks pass. If any fails: write `Retro/FIX-NNN.md` and retry once. If still failing on retry, stop and surface to operator.
