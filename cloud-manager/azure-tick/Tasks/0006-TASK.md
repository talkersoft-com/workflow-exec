# Task 0006 — Porch inventory generation

## Goal
Porch writes `<workdir>/inventory/hosts` per target before exec, sourcing IP from the target DTO. Failure mode: if `target.IpAddress` is empty, fail the target cleanly with a clear message.

## Target repo(s)
`vorch-service`

## Steps
1. Extend `playbookRunTargetDTO` in `internal/ansible/playbook_run_exec.go` (around line 24) with:
   ```go
   IpAddress string `json:"ipAddress,omitempty"`
   ```
2. In `executeOneTarget`, just before the `cmd := exec.CommandContext(...)` line:
   ```go
   if target.IpAddress == "" {
       msg := "vm ip_address not set — vorch did not record it during VM create. backfill required."
       h.failTarget(ctx, runID, target.ID, msg)
       return messages.PlaybookRunStatusFailed
   }
   inventoryDir := filepath.Join(workdir, "inventory")
   _ = os.MkdirAll(inventoryDir, 0o700)
   user := os.Getenv("PORCH_ANSIBLE_USER")
   if user == "" { user = "cloudmanager" }
   inventoryLine := fmt.Sprintf("%s ansible_host=%s ansible_user=%s ansible_ssh_private_key_file=env/ssh_key ansible_ssh_common_args='-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null'\n",
       target.VMName, target.IpAddress, user)
   if err := os.WriteFile(filepath.Join(inventoryDir, "hosts"), []byte(inventoryLine), 0o600); err != nil {
       h.failTarget(ctx, runID, target.ID, fmt.Sprintf("write inventory: %v", err))
       return messages.PlaybookRunStatusFailed
   }
   ```
3. Add `--inventory` flag to the exec call:
   ```go
   cmd := exec.CommandContext(ctx, bin, "run", workdir,
       "--inventory", filepath.Join(workdir, "inventory", "hosts"),
       "--project-dir", workdir, "-p", "site.yml", "--json")
   ```
   (The `--project-dir` will be removed in Phase 7 — Phase 6 only adds inventory.)
4. `go build ./...` — exit 0.

## Definition of done
- `target.IpAddress` is part of the DTO.
- `inventory/hosts` is written before exec.
- `--inventory` flag is passed.
- Empty `IpAddress` fails the target with the documented message.
- Build clean.
