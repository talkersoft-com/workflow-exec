# Task 0009 — ansible-runner system install + ProtectHome=yes

## Goal
Install ansible-runner system-wide so porch can drop `ProtectHome=no`. Prefer apt; fall back to pipx --global. Re-enable `ProtectHome=yes` on the unit. Clean up the FIX-001 symlink chain in uninstall.

## Target repo(s)
`cloud-manager-api`

## Steps

### 9a — Verify package availability
```
apt-cache policy ansible-runner
```
Record the result in `Retro/CHANGE-0009.md`:
- If `apt-cache policy` shows a candidate version → apt path (preferred).
- Otherwise → pipx --global path.

### 9b — Update install-porch-service.py
Open `cloud-manager-api/scripts/porch/install-porch-service.py`.

1. Add a new step before "Build Application" (call it `ensure_ansible_runner`):
   ```python
   def ensure_ansible_runner(self):
       logger.info("Ensuring ansible-runner is system-installed...")
       # Prefer the apt package if available.
       chk = subprocess.run(["apt-cache", "policy", "ansible-runner"], capture_output=True, text=True)
       if "Candidate:" in chk.stdout and "(none)" not in chk.stdout:
           self.run_command(["sudo", "apt-get", "install", "-y", "ansible-runner"])
           self.ansible_runner_bin = "/usr/bin/ansible-runner"
       else:
           # Fallback to pipx --global (Python 3.12+).
           self.run_command(["sudo", "pipx", "install", "--global", "ansible-runner"], check=False)
           self.ansible_runner_bin = "/usr/local/bin/ansible-runner"
       # Verify
       if not Path(self.ansible_runner_bin).exists():
           logger.error(f"ansible-runner not found at {self.ansible_runner_bin} after install")
           return False
       logger.info(f"ansible-runner installed at {self.ansible_runner_bin}")
       return True
   ```
2. Add it to the `steps` list in `install()`:
   ```python
   ("Ensure ansible-runner", self.ensure_ansible_runner),
   ```
3. In `create_systemd_service`, change:
   - Add `Environment="ANSIBLE_RUNNER_BIN={self.ansible_runner_bin}"` to the unit body.
   - Change `ProtectHome=no` → `ProtectHome=yes`.
4. Default `self.ansible_runner_bin = "/usr/bin/ansible-runner"` in `__init__` in case `ensure_ansible_runner` is skipped.

### 9c — Clean up FIX-001 leftovers in uninstall
Open `cloud-manager-api/scripts/porch/uninstall-porch-service.py`. Add:
```python
run(["sudo", "rm", "-f", "/usr/local/bin/ansible-runner", "/usr/local/bin/ansible-runner.real"])
```
(After the existing porch binary removal.)

### 9d — Verify
```
python3 -m py_compile scripts/porch/install-porch-service.py scripts/porch/uninstall-porch-service.py
```

## Definition of done
- `install-porch-service.py` has `ensure_ansible_runner` step that prefers apt > pipx --global.
- Generated systemd unit has `ProtectHome=yes` and `Environment="ANSIBLE_RUNNER_BIN=..."`.
- `uninstall-porch-service.py` removes the FIX-001 symlinks.
- Python files compile.
- Decision recorded in `Retro/CHANGE-0009.md`.
