# Task 0003 — Materialize under project/

## Goal
Change `PlaybookMaterializationService` so every materialized file's `RelativePath` lives under `project/`. After this, ansible-runner's default project discovery finds `project/site.yml` without an explicit `--project-dir` override.

## Target repo(s)
`cloud-manager-api`

## Steps
1. Open `src/Services/CloudManager.Data.Services/PlaybookMaterializationService.cs`.
2. Find every `RelativePath = "..."` assignment (lines 55-90 in the current code).
3. Prefix each with `"project/"`:
   - `"site.yml"` → `"project/site.yml"`
   - `$"roles/{gr.Name}/{f.FilePath}"` → `$"project/roles/{gr.Name}/{f.FilePath}"`
   - `$"roles/{lr.Name}/{f.FilePath}"` → `$"project/roles/{lr.Name}/{f.FilePath}"`
   - Any other RelativePath occurrence also gets the `project/` prefix.
4. Do NOT touch the argument-specs file write (the conditional at line 97 around `ArgumentSpecsPath`). Those are metadata, not materialized files — verify with code reading.
5. `dotnet build` — exit 0.

## Definition of done
`grep -nE 'RelativePath\s*=\s*"' src/Services/CloudManager.Data.Services/PlaybookMaterializationService.cs` shows every materialized path starts with `project/`. Build clean.
