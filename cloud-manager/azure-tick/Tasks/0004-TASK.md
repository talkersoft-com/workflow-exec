# Task 0004 — Vorch IP writeback

## Goal
When `CreateVMCommand` succeeds, vorch PATCHes `cloud-manager-api` with the VM's IP. Best effort — log and continue on HTTP failure; don't fail VM create.

## Target repo(s)
`vorch-service`

## Steps
1. Open `vorch-service/internal/vm/vm_handlers.go`.
2. Find `CreateVMCommand`. After the libvirt create succeeds (just before the function returns success), make an HTTP PATCH call:
   ```go
   apiURL := os.Getenv("CLOUD_MANAGER_API_URL")
   if apiURL != "" && message.VM.IPAddress != "" {
       body := bytes.NewReader([]byte(fmt.Sprintf(`{"ipAddress":%q}`, message.VM.IPAddress)))
       req, err := http.NewRequest(http.MethodPatch, fmt.Sprintf("%s/api/v1/vm/%s/ip-address", apiURL, message.VM.PublicID), body)
       if err == nil {
           req.Header.Set("Content-Type", "application/json")
           if resp, herr := http.DefaultClient.Do(req); herr != nil {
               log.Printf("vm-create: ip writeback failed for %s: %v", message.VM.PublicID, herr)
           } else {
               resp.Body.Close()
               if resp.StatusCode >= 400 {
                   log.Printf("vm-create: ip writeback returned %d for %s", resp.StatusCode, message.VM.PublicID)
               }
           }
       }
   }
   ```
3. Confirm `message.VM.PublicID` exists on the model (`internal/models/command_messages.go`). If absent, fall through to `message.VM.VmName`; the API can resolve by name as a secondary key — but PublicID is preferred. If only the name is available, add `PublicID string \`yaml:"publicId"\`` to the YAML model and have the API populate it when publishing.
4. `go build ./...` — exit 0.

## Definition of done
`grep -nE 'ip-address|UpdateIpAddress|ipAddress' internal/vm/vm_handlers.go` shows the new PATCH call. Build clean. The call is non-fatal — `CreateVMCommand` returns nil whether or not the PATCH succeeds.
