# Task 0012 — Write Results / Lessons, then `hv_ship`

## Goal
Record outcomes and lessons, then ship via the MCP.

## Steps
1. Write `Results/RESULT.md` per the workflow template — Outcome, Branch (`raging-numbat`), Pull Requests table, Phase summary (one block per phase 0001 → 0011).
2. Write `Retro/LESSONS.md` — anything ambiguous, missing, or surprising during execution; FIX files written and why; one line if nothing went wrong.
3. Confirm both files exist and are committed-ready BEFORE calling ship.
4. Call:
   ```
   hv_ship  deck: "cloud-manager"
            message: "vm-events: append-only audit log + retry-teardown action; remove force-delete"
            title:   "vm-events: per-VM audit log + retry-teardown, retire force-delete"
   ```
5. Print every PR URL from the ship output directly to the user.
6. The deck will auto-advance on merge — record the new branch name in `Results/RESULT.md`.

## Notes
- `hv_ship` commits and pushes everything currently in the working tree. If `Results/RESULT.md` and `Retro/LESSONS.md` do not exist yet at ship time, they land on the NEXT branch and miss the PR.
- After ship, remind the operator to restart the MCP via `/mcp` so the new cloud-manager-mcp dist (without `cloud_vm_force_delete`) takes effect.

## Definition of done
PRs opened on all affected repos. Both result + lessons files included. Operator notified about `/mcp` restart.
