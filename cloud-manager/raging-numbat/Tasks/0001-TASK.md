# Task 0001 — Schema + entity

## Goal
Create the `vm.vm_events` table via an EF migration, add the `VmEvent` entity, and wire it into the DbContext.

## Files
- `cloud-manager-api/src/Models/CloudManager.Entities/Models/VmEvent.cs` (new) — inherits `Audit`; fields `Id`, `VirtualMachineId`, `EventType`, `Payload (JsonDocument)`, `Actor`, `OccurredAt`.
- `cloud-manager-api/src/Models/CloudManager.Entities/Enums/VmEventType.cs` (new) — `created`, `destroy_requested`, `teardown_succeeded`, `teardown_failed`, `retry_teardown_requested`, `restored`, `snapshot_taken`, `snapshot_reverted`, `ip_address_updated`, `playbook_run_started`, `playbook_run_completed`.
- `cloud-manager-api/src/Models/CloudManager.Entities/CloudManagerDbContext.cs` — register `DbSet<VmEvent> VmEvents`; map `vm.vm_events`; columns + both indexes per PLAN.
- `cloud-manager-api/src/Models/CloudManager.Entities/Migrations/<timestamp>_AddVmEvents.cs` (generated) — `dotnet ef migrations add AddVmEvents`.

## Steps
1. Author entity + enum.
2. Add DbContext block (table, columns, indexes, public_id wiring per existing pattern).
3. Run `dotnet ef migrations add AddVmEvents` from `src/Models/CloudManager.Entities/`.
4. Inspect generated migration: `CREATE TABLE vm.vm_events` + both indexes (`(virtual_machine_id, occurred_at DESC)` and `(event_type, occurred_at DESC)`) + `FK ON DELETE RESTRICT` to `vm.virtual_machines`.
5. `dotnet build src/CloudManager.sln` — must be clean.

## Definition of done
Build clean. `dotnet ef migrations script <prev> AddVmEvents` produces the expected DDL (CREATE TABLE + 2 indexes + FK).
