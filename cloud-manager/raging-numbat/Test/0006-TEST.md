# Test 0006 — Timeline endpoint

## TC-001 — Build clean
Pass: 0 errors.

## TC-002 — Returns events for a VM
Create a VM (produces `created`), call `GET /api/v1/vm/{publicId}/events`.
Pass: response is an array with one element, `event_type=created`, public_id present, no internal UUIDs in the response.

## TC-003 — Ordered by `occurred_at DESC`
With multiple events on a VM, hit the endpoint.
Pass: results are ordered most-recent first.

## TC-004 — `take` is honored and capped
Hit with `?take=5` — returns at most 5. Hit with `?take=10000` — returns at most 500 (server cap).
Pass: behavior matches.

## TC-005 — `payload` is nested JSON, not stringified
Inspect a response with a non-empty payload.
Pass: `payload` is an object / structured value, not a JSON-encoded string.

## TC-006 — Works on soft-deleted VMs
After a VM is soft-deleted, the endpoint still returns its event history.
Pass: timeline accessible for soft-deleted VMs.
