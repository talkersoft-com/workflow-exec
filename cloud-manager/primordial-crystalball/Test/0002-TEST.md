# Test 0002 — API: resolve attached bindings at instantiate

## TC-001 — Static binding resolves
A binding with `secret_id` set (managed Secret) attached to a blueprint.
Pass: resolution computes the Secret's `vault_path`, reads it, emits `{ credName, b64Value }` with
the correct base64 value. Fail: wrong path/value or skipped.

## TC-002 — Templated binding resolves
A templated binding (`{vm_public_id}`, `secret_id` NULL) attached to a blueprint.
Pass: `{vm_public_id}` is substituted from the provision context, the path is read, and the pair is
emitted. Fail: placeholder unsubstituted or wrong path.

## TC-003 — No bindings → empty list
Instantiate a blueprint with no attached bindings.
Pass: resolved list is empty; create-vm assembly unchanged from today. Fail: non-empty/altered.

## TC-004 — Missing secret is a clear error
Attach a binding whose path has no Vault secret.
Pass: provision surfaces a clear "secret not found" error (not silent skip). Fail: silent.

## TC-005 — No value in logs + build
Pass: value never logged; `--target api` builds clean. Fail: leak or build error.
