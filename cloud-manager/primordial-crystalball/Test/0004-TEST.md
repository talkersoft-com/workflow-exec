# Test 0004 — vorch: cloud-init injection + first-boot seal (unit/golden)

## TC-001 — write_files entry generated
Render user-data for a VM with one secret (`credName=DBPASSWD`).
Pass: a `write_files` entry writes the base64 value to `/run/cm-seed/DBPASSWD`, mode 0600, root.
Fail: missing/wrong path/mode.

## TC-002 — Seal + shred runcmd generated
Pass: a `runcmd`/first-boot step runs `systemd-creds encrypt --with-key=host --name=DBPASSWD …
→ /etc/credstore.encrypted/DBPASSWD`, then `shred -u /run/cm-seed/DBPASSWD`. Fail: missing.

## TC-003 — Value never in runcmd/logs
Pass: the secret value does not appear in any `runcmd` arg or log line (seal script reads the file).
Fail: value interpolated into a command/log.

## TC-004 — credName validation
Render with `credName="../evil"` (or with shell metachars).
Pass: rejected before path use. Fail: accepted/interpolated.

## TC-005 — Empty is byte-identical
Render user-data with no secrets.
Pass: output identical to the pre-change template (no cm-seed write_files, no extra runcmd).
Fail: any diff.

## TC-006 — Builds
Pass: `vorch-lib` + `vorch-service` Go builds clean. Fail: error.
