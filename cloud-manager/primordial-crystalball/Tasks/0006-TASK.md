# Task 0006 — E2E, build, deploy, ship

## Goal
Prove the full chain on a real VM, then build, deploy, and integrate.

## Reference
`../../../workflow-plans/cloud-manager/primordial-crystalball/PLAN.md`.

## Steps

### 1. End-to-end (live)
- Ensure a managed Secret exists (via #1.5 Secret Manager) and a Secret Binding references it with a
  `cred_name` (e.g. `DBPASSWD`); attach the binding to a test blueprint.
- Instantiate the blueprint → a VM is provisioned.
- On the VM: confirm `/etc/credstore.encrypted/<credName>` exists, the plaintext
  `/run/cm-seed/<credName>` is **gone**, and the value decrypts (e.g. a test unit with
  `LoadCredentialEncrypted` reads it, or `systemd-creds decrypt` on-host).
- Confirm the secret value appears in **no** log (api, vorch, run output, cloud-init logs).
- Confirm `vm.vm_secret_bindings` has the usage row, and deleting that binding is now **blocked**.
- **Regression:** provision a VM from a blueprint with **no** binding → succeeds exactly as before.

### 2. Build
`python3 .cicd/build-cloud-manager.py --target api` clean; `vorch-lib` + `vorch-service` Go builds clean.

### 3. Deploy
- `python3 .cicd/deploy-cloud-manager.py --target api`; confirm `cloud_health_check` green.
- **vorch redeploy (manual, hypervisor host)** — rebuild + restart `vorch-service`; record the exact
  command + restart confirmation in `Results/RESULT.md`.

### 4. Write outputs BEFORE hv_integrate
- `Results/RESULT.md` — outcome, branch, PR table, phase summary, E2E evidence (cred present /
  plaintext gone / no value in logs), deploy + vorch-restart record.
- `Retro/LESSONS.md`.

### 5. Ship
`hv_integrate` per `../deck.md` Ship block (stage `exec`). Report PR links.

## Definition of done
E2E green (cred sealed, plaintext gone, no value leaked, usage row + gate enforced, no-binding
regression clean), builds pass, api deployed + vorch restarted, outputs written, PRs opened.
Test 0006 passes.
