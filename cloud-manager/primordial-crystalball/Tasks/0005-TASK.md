# Task 0005 — API: populate vm_secret_bindings on provision success

## Goal
Record which VM uses which binding — the delete-gate's lower joint created (empty) in #1.5.

## Reference
`../../../workflow-plans/cloud-manager/primordial-crystalball/PLAN.md` — build item #5.

## Steps
- After a successful provision of a VM that had resolved bindings, insert one
  `vm.vm_secret_bindings` row per injected binding:
  `{ virtual_machine_id, secret_binding_id, cred_name }` (+ `public_id` `vsb_…`, `created_at`).
- **Idempotent on retry** — re-provisioning / retry must not create duplicate usage rows
  (upsert or pre-check on `(virtual_machine_id, secret_binding_id)`).
- This is what makes "can't delete a binding while a VM uses it" real (the RESTRICT FK from #1.5).
- camelCase; public_ids only.

## Build
`python3 .cicd/build-cloud-manager.py --target api` clean.

## Definition of done
Provisioning a VM with a binding writes exactly one usage row per binding (idempotent on retry);
attempting to delete that binding is then blocked by the RESTRICT FK. Test 0005 passes.
