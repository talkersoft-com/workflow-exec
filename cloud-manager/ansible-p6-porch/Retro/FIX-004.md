# FIX-004 — deploy-app.sh scp/ssh to self denied (publickey)

## Failure
Task 0006 step 3 retry: build succeeded (after FIX-003) but the script's
`scp/ssh todd@ubuntu-server.talkersoft.com` steps failed `Permission denied (publickey)` —
the workstation IS ubuntu-server and todd has no self-SSH key authorized, so the script's
remote-copy model cannot run from this host.

## Fix
Performed the script's copy step locally (functionally identical outcome):
`sudo install -m 755 bin/release/linux/amd64/{porch,vorch-service} /kvm-automator/`.
Config files were NOT overwritten — this PR changes no config, and the live
`/kvm-automator/config-server.yaml` carries the production Vault token.

## Lesson
The vorch deploy fragment assumes execution from a workstation other than the server. When
running ON the server, replace the scp/ssh leg with a local `sudo install`; binaries-only
deploys should skip the config copy entirely unless the PR changed config.
