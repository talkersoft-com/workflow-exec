# Task 0002 — API P1: instantiate-time parameter resolution

Repo: `cloud-manager-api`. Builds on 0001 — once params come from the operator, resolution no longer
needs the new VM's id.

## Steps
1. **`InstantiateRequest` gains optional `secretBindingParams`** — per-binding param values keyed by
   `secretBindingId`:
   ```jsonc
   "secretBindingParams": [
     { "secretBindingId": "sb_…", "params": { "vm_public_id": "vm_<sourceId>" } }
   ]
   ```
   `vm_public_id` supplies the SOURCE VM (e.g. the Postgres server), not the VM being created.
2. **`SecretBindingResolver.ResolveForBlueprintAsync` takes the supplied params.** For each templated
   binding, substitute placeholders from the supplied values. **Never** fall back to the new VM's id.
3. **Required-param-missing → 422** validation error naming the binding. A `source: vm` param that is
   required but not supplied must fail cleanly — never brick.
4. Templated `{vm_public_id}` with a supplied source VM resolves to that VM's Vault path, reads the
   secret, injects + seals as today, and records the `vm_secret_bindings` row.
5. Conventions: camelCase wire; public_ids only (`GuidByPublicIdAsync` to resolve the supplied
   `vm_public_id`); secret value never echoed.
6. Build: `python3 .cicd/build-cloud-manager.py --target api` green.

## Open questions (carried from PLAN — default unless operator says otherwise)
- Param key = `secretBindingId` (not per-attachment `bsb_…`). Switch only if the operator prefers
  attachment keying (same binding attached twice).
- Missing-param policy = hard 422 (no "provision anyway, skip the cred" mode).

## Definition of done
`secretBindingParams` accepted + validated; templated `{vm_public_id}` resolves to the supplied
source VM; required-missing → 422 naming the binding; no fallback to the new VM's id; api build green.
