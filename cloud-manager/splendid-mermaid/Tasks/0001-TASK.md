# Task 0001 тАФ API P0: resolve-before-create + clean failure

Repo: `cloud-manager-api`. Root cause is verified in the PLAN тАФ `MarketplaceController.Instantiate`
creates the VM record (~line 122) BEFORE resolving secret bindings (~line 156), so a templated
`{vm_public_id}` binding throws `VaultSecretNotFoundException` at the new VM's own (empty) path and
orphans the VM at `orchestrationStatus=0` with no create-vm command and no failure event.

## Steps
1. **Reorder `Instantiate`:** resolve the blueprint's secret bindings BEFORE creating the VM
   record. Move the `_secretBindingResolver.ResolveForBlueprintAsync(...)` call ahead of the VM
   insert + `blueprint_instantiated` event.
2. **Clean 422 on resolution failure:** when resolution fails (e.g. missing secret), return a 422
   carrying the binding **name** + the Vault **path** that was missing тАФ never the secret value.
   No VM record is created, so there is nothing to orphan.
3. **Defense in depth:** if any code path still creates a VM before a later failure, record a VM
   **failure event** (`IVmEventService.RecordEventAsync`, best-effort try/catch) and mark the VM
   failed тАФ never leave an eventless `orch=0` orphan.
4. Keep the existing resolve path for static + concrete-path templated bindings working unchanged.
5. Build: `python3 .cicd/build-cloud-manager.py --target api` green.

## Definition of done
Bindings resolve before VM creation; missing-secret тЖТ 422 with binding name + path and no VM row;
no eventless orphan possible; static / concrete-path / no-binding paths unchanged; api build green.
