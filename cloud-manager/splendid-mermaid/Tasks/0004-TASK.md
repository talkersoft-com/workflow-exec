# Task 0004 — E2E + build + deploy + ship

## Steps
1. **Build everything changed:** `python3 .cicd/build-cloud-manager.py --target all` green
   (api + mcp).
2. **Deploy api:** `python3 .cicd/deploy-cloud-manager.py --target api` (no migration this workflow,
   so no `dotnet ef database update` step). Confirm the localhost smoke poll passes.
   Rebuild mcp dist via the deploy script's mcp target so the operator can `/mcp` reload.
3. **E2E (against the deployed system, MCP + db/psql + vault tools):**
   - **Happy path (P1):** provision a consumer-client blueprint, supplying the Postgres VM
     `pg-test123` as the binding's `vm_public_id` source → secret injected + sealed; verify the
     `vm_secret_bindings` row and the create-vm command both exist; VM reaches a healthy
     orchestration state.
   - **Clean 422 (P0):** provision a templated `{vm_public_id}` blueprint with NO param → 422 naming
     the binding + Vault path; verify **no VM row** was created (no orphan, no eventless `orch=0`).
   - **Regression:** a no-binding blueprint and a static-binding blueprint both still provision
     normally.
4. **Write outputs BEFORE shipping:**
   - `Results/RESULT.md` — outcome, branch, PR table (filled from hv_integrate), per-phase summary,
     deploy targets + smoke HTTP codes, and the `/mcp` reload operator instruction (mcp dist changed).
   - `Retro/LESSONS.md` — what would have helped + FIX file list.
5. **Ship:** `hv_integrate` per `../deck.md`. A failed deploy/build is a stop condition — write a
   FIX file under `Retro/`, resolve, retry.

## Definition of done
api+mcp build green; api deployed + smoke-passing; all three E2E checks pass (happy path injected,
missing-param 422 with zero orphan, regressions intact); Results + LESSONS written; hv_integrate
opened PRs.
