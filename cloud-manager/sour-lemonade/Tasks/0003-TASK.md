# Task 0003 — Web: Secret Bindings page + blueprint Attached Secrets

## Goal
A "Secret Bindings" management UI and a blueprint "Attached Secrets" section, reusing the
existing @rjsf / list-page patterns.

## Steps
1. **Secret Bindings page** (list + create/edit): fields name, description, path_template, params editor.
   - As path_template changes, **scan for `{placeholders}`** and render one param row per placeholder.
   - Each param row: name (from placeholder), a **source picker** (only `"vm"` today; a placeholder
     named `vm_public_id` defaults to `source: vm`), and a label.
   - Empty placeholders -> empty params -> no rows (the manual case).
2. **Blueprint detail page**: add an **"Attached Secrets"** section — attach (pick a Secret Binding +
   cred_name), detach, reorder — parallel to the existing playbook-attachment UI.
3. NO instantiate-form changes.
4. Build: `python3 .cicd/build-cloud-manager.py --target web`.

## Definition of done
Web builds; both example bindings authorable in the UI; attach/detach/reorder on a blueprint persists.
Run `Test/0003-TEST.md`.
