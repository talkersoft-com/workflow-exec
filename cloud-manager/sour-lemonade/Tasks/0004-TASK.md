# Task 0004 — Verify, build, deploy, ship

## Goal
Prove the authoring round-trip end to end, deploy, then integrate.

## Steps
1. Apply the migration to the live DB (BEFORE the api restart) — see Exec.md deploy section.
2. Deploy: `python3 .cicd/deploy-cloud-manager.py --target all` (api + web; mcp build only).
3. Author BOTH example bindings:
   - manual: name "manual stored secret", path_template `/manual/secret/i/stored` -> params `[]`.
   - postgres: name "VM postgres credential", path_template
     `cloudmanager/data/vm/instances/{vm_public_id}/postgres` -> one param `{name:vm_public_id, source:vm}`.
4. Attach the postgres binding to a blueprint (cred_name `DBPASSWD`), set position.
5. Round-trip via the MCP tools (`cloud_secret_binding_list`, `cloud_blueprint_secret_binding_list`)
   and confirm the API/UI agree (camelCase, public_ids, no Guid leak).
6. Write `Results/RESULT.md` (outcome, PR table placeholder, the `/mcp` reload note) + `Retro/LESSONS.md`.
7. `hv_integrate` (message/title per deck.md).

## Definition of done
Both bindings round-trip across API/MCP/UI; services healthy; Results + Retro written; hv_integrate opened PRs.
Run `Test/0004-TEST.md`.
