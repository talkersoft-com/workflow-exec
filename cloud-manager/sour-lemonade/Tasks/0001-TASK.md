# Task 0001 — API: SecretBinding entity + migration + endpoints

## Goal
Add the `SecretBinding` entity + `BlueprintSecretBinding` join in cloud-manager-api, one
reversible migration, and the CRUD + attach endpoints. Mirror `blueprint_playbooks` throughout.

## Entities (schema `marketplace`)
- **secret_bindings**: id (uuid), public_id (`sb_`), name, description, path_template (text),
  params (jsonb, array of `{name,source,label}`, default `[]`), injection_provider (varchar(32),
  default `"systemd-creds"`), status (varchar(32), `Draft`/`Published`), config (jsonb, default `{}`),
  deleted_at (timestamp, soft-delete).
- **blueprint_secret_bindings**: id (uuid), public_id (`bsb_`), blueprint_id (uuid FK, cascade),
  secret_binding_id (uuid FK), position (int, unique per blueprint), cred_name (varchar).

## Endpoints
- Secret Bindings: list, get(publicId), create, update, soft-delete — camelCase, public_ids only.
- Blueprint <-> binding: attach (blueprintId, secretBindingId, position?, credName), detach,
  list (by blueprint), reorder (positions).
- NO instantiate / MarketplaceController.Instantiate changes.

## Steps
1. Add EF entity models + DbSet + ModelBuilder config (FKs, unique index on (blueprint_id, position),
   global soft-delete query filter on SecretBinding). Register public_id generation (`sb_`/`bsb_`).
2. `dotnet ef migrations add AddSecretBindings` — verify the generated Up/Down create + drop both tables.
3. DTOs + service + controller mirroring the playbook/blueprint code paths.
4. Build: `python3 .cicd/build-cloud-manager.py --target api`.

## Definition of done
API builds; migration up/down clean; CRUD + attach endpoints work for both example bindings.
Run `Test/0001-TEST.md`.
