# Task 0005 — Regression + docs (cloud-manager-api)

## Steps
1. Regression — raw-text save path: saving a playbook/role file with unchanged text, and with
   ordinary edited text, behaves byte-identically to pre-P3 (same revisions, same stored
   bytes, same events). The fidelity check and `ydoc` write must be additive-only on this
   path.
2. Regression — existing flows: playbook list/get/materialize/run and role-file CRUD
   unchanged; flag OFF (`ansible-studio` disabled) hides all new routes and the system
   behaves as before.
3. Run the full golden-file corpus once more against the final code (pytest, all green).
4. Deployment docs: document the sidecar in the api repo — compose service definition, how it
   starts/stops with the API stack, healthz, port (internal only), and the
   sidecar-unavailable behaviour (raw saves fall back per Task 0004; structured edits and
   roundtrip routes fail with a clear error).
5. Build: `python3 .cicd/build-cloud-manager.py --target api`

Done when: build passes; both regression sweeps green; corpus green; docs in place.
