# Task 0002 — `ydoc` entity + API proxy + write-time fidelity check (cloud-manager-api)

## Steps
1. RESEARCH (read, no changes): the revision-cutting paths from P2 (playbook create/update,
   role file create/update/restore); contracts §1 for the `ydoc` schema; contracts §3 for the
   round-trip route shapes; how existing flag-gated endpoints check `ansible-studio`.
2. Add `YamlDocument` entity (`ydoc`, schema `ansible` per contracts §1) + EF config +
   migration: one per `PlaybookRevision`/`RoleFileRevision` — canonical source text checksum +
   the sidecar's decomposition map (jsonb).
3. Write `ydoc` whenever a revision is cut, with the write-time fidelity proof:
   `recompose(decompose(text)) == text` via the sidecar; mismatch FAILS the save loudly (this

4. Add API proxy routes `api/v1/roundtrip/decompose` and `api/v1/roundtrip/recompose`

   Web/MCP never call the sidecar directly.
5. Conventions: camelCase JSON on the wire; public_ids only — no internal `Guid` leaks.
6. Build: `python3 .cicd/build-cloud-manager.py --target api`

Done when: build passes; cutting a revision writes a `ydoc` row with checksum + map; a forced
fidelity mismatch fails the save with a loud error; proxy routes work end-to-end through the
sidecar and respect the feature flag.
