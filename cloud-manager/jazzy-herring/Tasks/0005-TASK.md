# Task 0005 — Assignment tracking + log redaction parity

## Goal
Ensure porch (1) updates `vm_playbook_assignments` rows on terminal runs the same way the Python worker did, if vorch wasn't already doing it, and (2) cannot ever log a raw `hvs.*` Vault token to stdout/stderr/journal.

## Target repo(s)
`vorch-service`

## Steps

### 5a — Assignment tracking
1. Read `Retro/CHANGE-0002.md`'s assignment-tracking verification.
2. If vorch's `playbook_run_handler.go` already updates `vm_playbook_assignments` (`last_run_id`, `last_run_status`, `last_applied_at`) on terminal status: **skip 5a**, record "n/a — already implemented" in `Retro/CHANGE-0005.md`.
3. If not: implement the UPDATE in the same DB transaction that writes the run's terminal status (Succeeded=3, Failed=4, Cancelled=5). SQL:
   ```sql
   UPDATE vm.vm_playbook_assignments
   SET last_run_id = (SELECT id FROM vm.playbook_runs WHERE public_id = $1),
       last_run_status = $2,
       last_applied_at = now()
   WHERE id = $3;
   ```
   Only run the UPDATE when the run's `assignment_id` is non-null (some runs aren't tied to an assignment — e.g. ad-hoc triggers).

### 5b — Log redaction
1. Add a redacting wrapper around the stdlib `log` package. Either:
   - **Option A**: wrap `os.Stdout`/`os.Stderr` with an `io.Writer` that runs every line through `regexp.MustCompile(`hvs\.[A-Za-z0-9_\-]+`).ReplaceAllString(line, "hvs.<REDACTED>")` and call `log.SetOutput(...)` to it in `cmd/porch/main.go`.
   - **Option B**: a small `internal/log/` package with `Printf`/`Printf` wrappers that all internal code calls; ban direct `log.Printf` via a `go vet` rule or staticcheck config (overkill for this iteration — Option A is fine).
2. Audit the four ansible handlers + healthprobe for any place that might log a token value:
   - `playbook_run_exec.go` — env var injection points
   - `playbook_run_handler.go` — error paths
   - any place that prints `runDTO` or env vars
   Replace any direct token logging with the literal string `"using API-minted VAULT_TOKEN"` (no value).

## Definition of done
If assignment tracking was missing, the UPDATE is implemented and unit-traceable via SQL log. Log redaction wrapper is wired in `cmd/porch/main.go`. A grep for `runDTO.VaultRunToken` or `child_token` in `internal/ansible/` shows no direct log calls. `go build ./...` exits 0.
