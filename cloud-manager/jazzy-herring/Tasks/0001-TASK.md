# Task 0001 — Module rename

## Goal
Change `vorch-service/go.mod` from `module main` to a real Go module path so the upcoming `cmd/{vorch-service,porch}` split is buildable. **Behavior unchanged** — build still passes against the existing single-binary layout.

## Target repo(s)
`vorch-service`

## Steps
1. Confirm the target module path with the operator if uncertain (PLAN.md Open Question #1 suggests `github.com/talkersoft-com/vorch-service`). Default to that unless told otherwise.
2. Edit `vorch-service/go.mod`: replace `module main` with `module github.com/talkersoft-com/vorch-service`.
3. Find every Go file that imports `"main/..."`:
   ```bash
   cd vorch-service && grep -rln '"main/' --include='*.go'
   ```
4. Replace every occurrence of `"main/<pkg>"` with `"github.com/talkersoft-com/vorch-service/<pkg>"` (for now — the next phase moves these paths under `internal/`).
5. Run `go mod tidy` from `vorch-service/`.
6. Run `go build ./...` — must succeed.
7. Run `go vet ./...` — must succeed.
8. No code logic changes in this task. Only import-path edits.

## Definition of done
`go.mod` shows the new module path. `grep -rn '"main/' --include='*.go'` returns zero matches. `go build ./...` exits 0. `go vet ./...` exits 0.
