# Task 0002 — Package carve-out

## Goal
Reorganize `vorch-service` into the target shape from PLAN.md so two `cmd/` entry points can be added. Files move to `internal/` packages; existing `app.go` becomes `cmd/vorch-service/main.go`; a stub `cmd/porch/main.go` is added (wiring comes in Phase 3).

## Target repo(s)
`vorch-service`

## Steps
1. Create the new directories:
   ```
   cmd/vorch-service/
   cmd/porch/
   internal/vm/
   internal/ansible/
   internal/vmsub/
   internal/ansiblesub/
   internal/amqp/
   internal/pub/
   internal/models/
   internal/vault/
   ```
2. **Move (do not copy)** the following files:
   - `vorch-service/app.go` → `cmd/vorch-service/main.go` (rename `package main` stays; this IS the main package for the vorch binary)
   - `vorch-service/handlers/vm_handlers.go` → `internal/vm/`
   - `vorch-service/handlers/playbook_run_handler.go`, `playbook_run_exec.go`, `playbook_cancel_handler.go`, `cancel_registry.go`, `collection_install_handler.go`, `collection_remove_handler.go` → `internal/ansible/`
   - `vorch-service/messagesub/messagesub.go` (the VM `StartConsumer`) → `internal/vmsub/`
   - `vorch-service/messagesub/playbook_run_sub.go`, `playbook_cancel_sub.go`, `collection_install_sub.go`, `collection_remove_sub.go` → `internal/ansiblesub/`
   - `vorch-service/healthprobe/probe.go` → `internal/ansible/healthprobe/` (or `internal/ansible/` directly — implementer's call)
   - `vorch-service/messagepub/messagepub.go` → `internal/pub/`
   - `vorch-service/models/command_messages.go` → `internal/models/`
   - `vorch-service/vault/client.go` → `internal/vault/`
3. Factor out the shared AMQP connection-bootstrap code (currently inside `messagesub.go`) into `internal/amqp/`. Both `vmsub` and `ansiblesub` will call into it.
4. Update every import path in every moved file to the new `internal/...` path under the module.
5. Update package declarations: each moved file's `package` line should match its new directory name (e.g. files in `internal/ansible/` declare `package ansible`).
6. Create a stub `cmd/porch/main.go`:
   ```go
   package main

   func main() {
       // TODO Phase 0003 — wire consumers + healthprobe
   }
   ```
7. Run `go build ./...` — must succeed.
8. Run `go vet ./...` — must succeed.
9. **VERIFY** (per PLAN.md Phase 0002 directive): does the current `playbook_run_handler.go` write to `vm_playbook_assignments` (columns `last_run_id`, `last_run_status`, `last_applied_at`) on terminal status? Search:
   ```bash
   grep -nE "vm_playbook_assignments|last_run_id|last_applied_at" internal/ansible/*.go
   ```
   Record the finding (yes/no) in `Retro/CHANGE-0002.md`. This drives Phase 0005 scope.

## Definition of done
`vorch-service/handlers/`, `messagesub/`, `messagepub/`, `models/`, `vault/`, `healthprobe/` directories no longer exist (all files moved). `cmd/vorch-service/main.go` and `cmd/porch/main.go` exist. `go build ./...` exits 0. `go vet ./...` exits 0. The assignment-tracking verification is recorded in `Retro/CHANGE-0002.md`.
