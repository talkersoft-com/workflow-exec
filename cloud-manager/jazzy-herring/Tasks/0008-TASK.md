# Task 0008 — Retire the Python worker

## Goal
Zero Python `cloud-manager-worker` footprint on the host. Systemd unit gone, install directory gone, pipx venv gone, env file gone.

## Target repo(s)
(no code changes — service teardown only)

## Steps
1. Stop and disable (already done in Phase 0007 but re-confirm):
   ```bash
   sudo systemctl stop cloud-manager-worker 2>/dev/null || true
   sudo systemctl disable cloud-manager-worker 2>/dev/null || true
   ```
2. Remove the systemd unit:
   ```bash
   sudo rm -f /etc/systemd/system/cloud-manager-worker.service
   sudo systemctl daemon-reload
   ```
3. Remove the install directory and env file:
   ```bash
   sudo rm -rf /opt/cloud-manager-worker
   sudo rm -f /etc/cloud-manager/worker.env
   ```
4. Remove the pipx venv. The venv is owned by the `cloudmanager` user at `/home/cloudmanager/.local/pipx/venvs/cloud-manager-worker/`. Try pipx first; fall back to manual `rm -rf` if pipx state is corrupt:
   ```bash
   sudo -u cloudmanager pipx uninstall cloud-manager-worker 2>/dev/null || \
     sudo rm -rf /home/cloudmanager/.local/pipx/venvs/cloud-manager-worker \
                  /home/cloudmanager/.local/bin/cloud-manager-worker
   ```
5. Verify zero footprint (see Test 0008).

## Definition of done
- `systemctl status cloud-manager-worker` → "Unit could not be found".
- `ls /opt/cloud-manager-worker` → no such file or directory.
- `ls /etc/cloud-manager/worker.env` → no such file or directory.
- `sudo -u cloudmanager pipx list` does not mention `cloud-manager-worker`.
- `ls /home/cloudmanager/.local/pipx/venvs/cloud-manager-worker` → no such file or directory.
