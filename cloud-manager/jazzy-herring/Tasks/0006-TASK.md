# Task 0006 — Install scripts + systemd unit

## Goal
Build and publish the porch binary to `/kvm-automator/porch`. Create the systemd unit `porch.service` and its env file `/etc/cloud-manager/porch.env`. Add install/start/stop/uninstall scripts under `cloud-manager-api/scripts/porch/`.

## Target repo(s)
`cloud-manager-api` (scripts only — no API code changes here)

## Steps
1. Create `cloud-manager-api/scripts/porch/` with four scripts mirroring `cloud-manager-api/scripts/api/`:
   - `install-porch-service.py` — `go build ./cmd/porch -o /tmp/porch-build`; copies to `/kvm-automator/porch`; writes systemd unit + env file; `systemctl daemon-reload`.
   - `start-porch-service.py` — `sudo systemctl start porch`.
   - `stop-porch-service.py` — `sudo systemctl stop porch`.
   - `uninstall-porch-service.py` — stops, disables, removes unit + binary + env file.
2. Systemd unit content for `/etc/systemd/system/porch.service`:
   ```
   [Unit]
   Description=Porch — Ansible Orchestration Service
   After=network.target rabbitmq-server.service postgresql.service

   [Service]
   Type=simple
   User=root
   WorkingDirectory=/kvm-automator
   ExecStart=/kvm-automator/porch --config /kvm-automator/config-server.yaml
   EnvironmentFile=-/etc/cloud-manager/porch.env
   StandardOutput=journal
   StandardError=journal
   Restart=always
   RestartSec=5s
   ReadWritePaths=/kvm-automator /usr/share/ansible/collections /mnt/cloud-storage

   [Install]
   WantedBy=multi-user.target
   ```
3. Env file `/etc/cloud-manager/porch.env` populated from `/etc/cloud-manager/cloud-manager-config.env`:
   ```
   VAULT_ADDR=https://127.0.0.1:8200
   VAULT_TOKEN=<copied from api.env>
   VAULT_ORCHESTRATOR_TOKEN=<copied from worker.env — reuses the existing one>
   RABBITMQ_HOST=localhost
   RABBITMQ_USER=cloudmanager
   RABBITMQ_PASSWORD=<from common config>
   DBHOST=localhost
   DBUSER=cloudmanager
   DBPASSWD=<from common config>
   DBPORT=5432
   DATABASE=clouddb
   SSLMODE=Prefer
   TRUSTSERVERCERTIFICATE=true
   PLAYBOOK_WORKDIR_ROOT=/kvm-automator/playbook-runs
   PLAYBOOK_LOG_DIR=/kvm-automator/playbook-logs
   CLOUD_MANAGER_API_URL=http://localhost:5250
   ```
4. Run the install script to lay down the binary, unit, env file. DO NOT start the service yet — Phase 0007 owns cutover.

## Definition of done
`/kvm-automator/porch` exists and is executable. `/etc/systemd/system/porch.service` exists. `/etc/cloud-manager/porch.env` exists with required values. `systemctl status porch` shows `inactive (dead)` — installed but not started. All four scripts exist under `cloud-manager-api/scripts/porch/`.
