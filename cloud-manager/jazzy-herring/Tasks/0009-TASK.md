# Task 0009 — End-to-end verification

## Goal
Prove the new pipeline works on a real playbook run: API mints → AMQP → porch picks up → ansible writes secret → porch reports terminal → API revokes → assignments row updated. Confirm vorch is silent on ansible topics and porch's journal has zero `hvs.*` leaks.

## Target repo(s)
(no code changes — live verification only)

## Steps
1. Open the web app (https://ubuntu-server.talkersoft.com/) and navigate to `/ansible/run?playbookId=pb_EMEJCH08GG` (the `pg` playbook).
2. Select `postgres-test`. Click Run.
3. Watch the run progress on `/ansible/runs/{newRunId}`. Wait for terminal (Succeeded or Failed).
4. Verify the run row:
   ```sql
   SELECT public_id, status, vault_secrets_prefix, vault_run_token IS NULL AS token_cleared, error_message
   FROM vm.playbook_runs WHERE public_id = '<new run id>';
   ```
   Pass: `status = 3`, `vault_secrets_prefix = 'cloudmanager/data/playbook-secrets/<runId>'`, `token_cleared = true`.
5. Verify the assignment row updated:
   ```sql
   SELECT id, last_run_status, last_run_id, last_applied_at
   FROM vm.vm_playbook_assignments
   WHERE last_run_id = (SELECT id FROM vm.playbook_runs WHERE public_id = '<new run id>');
   ```
   Pass: `last_run_status = 3`, `last_applied_at` within the last few minutes.
6. Verify the secret landed (needs Vault root token — operator runs):
   ```bash
   VAULT_TOKEN=<root> vault kv get cloudmanager/playbook-secrets/<runId>/postgres-test/postgres
   ```
   Pass: returns fields `host`, `port`, `db`, `user`, `password`.
7. Verify porch journal has no token leaks:
   ```bash
   sudo journalctl -u porch --since "10 minutes ago" | grep -E 'hvs\.[A-Za-z0-9]' | head -5
   ```
   Pass: zero matches.
8. Verify vorch journal has zero ansible references:
   ```bash
   sudo journalctl -u vorch_service --since "10 minutes ago" | grep -iE 'playbook|ansible|healthprobe' | head -5
   ```
   Pass: zero matches.
9. Verify single consumer on `playbook-runs`:
   ```bash
   sudo rabbitmqctl list_consumers | grep playbook-runs
   ```
   Pass: exactly one row.
10. Verify health probe round-trip:
    ```sql
    SELECT key, healthy, last_probed_at
    FROM membership.feature_flags
    WHERE key = 'playbooks';
    ```
    Pass: `last_probed_at` within the last probe interval (≤5 min), `healthy = true`.

## Definition of done
All ten checks pass. If any fails, write `Retro/FIX-NNN.md` and re-run.
