# Task 0004 — Vorch loses ansible

## Goal
`cmd/vorch-service/main.go` no longer registers any ansible consumers and no longer starts the healthprobe. Its import graph contains zero `internal/ansible*` references. Symmetric check on porch.

## Target repo(s)
`vorch-service`

## Steps
1. Open `cmd/vorch-service/main.go`. Remove:
   - The `StartPlaybookRunConsumer` + `StartPlaybookCancelConsumer` calls.
   - The collection install/remove consumer registration calls.
   - The `healthprobe.New().Start(...)` call.
   - Any import of `internal/ansible*` or `internal/ansiblesub`.
2. Keep:
   - The AMQP connection setup (via `internal/amqp`).
   - `messagesub.StartConsumer(conn, serverCfg)` — moved last phase to `internal/vmsub.StartConsumer`.
   - Provisioning + libvirt helpers.
3. Run:
   ```bash
   go build ./cmd/vorch-service ./cmd/porch
   go list -deps ./cmd/vorch-service | grep -E 'internal/ansible|internal/ansiblesub' || echo OK
   go list -deps ./cmd/porch         | grep -E 'internal/vm($|sub)' || echo OK
   ```
   Both `grep` commands must print `OK` (i.e. find zero matches).

## Definition of done
`cmd/vorch-service` builds. `go list -deps` shows no ansible packages in vorch's graph and no vm packages in porch's graph.
