# Task 0007 — Cutover

## Goal
Switch the live consumer on `playbook-runs` from (vorch + Python worker) to (porch alone). Vorch keeps running for VM events.

## Target repo(s)
(no code changes — deployment + service-state changes only)

## Steps
Execute the 8-step cutover sequence from PLAN.md Design / Cutover safety:

1. Build + deploy the new vorch-service binary (built from Phase 4 sources):
   ```bash
   cd vorch-service && go build -o /tmp/vorch-new ./cmd/vorch-service
   ```
2. Stop `vorch_service.service`:
   ```bash
   sudo systemctl stop vorch_service
   ```
3. Stop + disable the Python worker:
   ```bash
   sudo systemctl stop cloud-manager-worker
   sudo systemctl disable cloud-manager-worker
   ```
4. Confirm zero consumers on `playbook-runs`:
   ```bash
   sudo rabbitmqctl list_consumers | grep playbook-runs || echo "no consumers (expected)"
   ```
5. Replace the vorch binary on disk with the new one (without ansible consumers):
   ```bash
   sudo cp /tmp/vorch-new /kvm-automator/vorch-service
   ```
6. Start `porch.service`. Verify single consumer on `playbook-runs`:
   ```bash
   sudo systemctl start porch
   sleep 2
   sudo rabbitmqctl list_consumers | grep playbook-runs
   ```
   Must show exactly one row with consumer-tag matching `porch`.
7. Start `vorch_service.service` (now ansible-free):
   ```bash
   sudo systemctl start vorch_service
   ```
8. Tail both journals briefly to confirm clean boot of both services:
   ```bash
   sudo journalctl -u porch -u vorch_service --since "30s ago"
   ```

## Definition of done
After step 8:
- `rabbitmqctl list_consumers | grep playbook-runs` → exactly 1 row, owned by porch.
- `rabbitmqctl list_consumers | grep playbook-run-cancel` → 1 row, owned by porch.
- `rabbitmqctl list_consumers | grep collection-` → rows owned by porch (no vorch).
- Both `porch` and `vorch_service` show `active (running)` in `systemctl status`.
- No exception traces in either journal in the last 30 seconds.
- Python worker is `inactive (dead)` and disabled — removal happens in Phase 0008.
