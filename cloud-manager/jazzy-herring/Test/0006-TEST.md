# Test 0006 — Install scripts + systemd unit

## TC-001 — Install scripts exist
```bash
ls cloud-manager-api/scripts/porch/install-porch-service.py \
   cloud-manager-api/scripts/porch/start-porch-service.py \
   cloud-manager-api/scripts/porch/stop-porch-service.py \
   cloud-manager-api/scripts/porch/uninstall-porch-service.py
```
Pass: all four exist (no "No such file" errors).

## TC-002 — Binary deployed
```bash
ls -la /kvm-automator/porch && file /kvm-automator/porch | grep -i 'executable'
```
Pass: exists, is an ELF executable.

## TC-003 — Systemd unit registered
```bash
sudo systemctl cat porch.service | head -3
```
Pass: prints `# /etc/systemd/system/porch.service` followed by the unit content.

## TC-004 — Env file exists
```bash
sudo test -f /etc/cloud-manager/porch.env && sudo grep -cE 'VAULT_ADDR|RABBITMQ_HOST|DBHOST|PLAYBOOK_WORKDIR_ROOT|CLOUD_MANAGER_API_URL' /etc/cloud-manager/porch.env
```
Pass: count >= 5 (all required vars present).

## TC-005 — Porch is installed but NOT started
```bash
systemctl is-active porch
```
Pass: prints `inactive`.
Fail: prints `active` (Phase 0007 owns starting it).
