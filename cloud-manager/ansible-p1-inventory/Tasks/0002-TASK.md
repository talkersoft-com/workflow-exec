# Task 0002 — Services + controllers (cloud-manager-api)

## Steps
1. DTOs + mappers: camelCase wire JSON, public ids only (`PublicIdByGuidAsync` /
   `GuidByPublicIdAsync`); internal Guids never leak.
2. Services + controllers for every route in PLAN.md, all `[RequireFeatureFlag("ansible-studio")]`:
   - `api/v1/inventory` — GET list, GET `/{id}`, POST, PATCH `/{id}`, DELETE `/{id}` (soft)
   - `api/v1/inventory/{id}/group` — GET, POST; PATCH/DELETE `/{gid}`
   - `api/v1/inventory/{id}/group/{gid}/var` — GET, PUT (upsert by name), DELETE `/{name}`
   - `api/v1/inventory/{id}/group/{gid}/member` — GET, POST (hostId), DELETE `/{invhId}`
   - `api/v1/inventory/{id}/host` — GET, POST (vmId or external hostName+ansibleHost);
     PATCH/DELETE `/{hid}`
   - `api/v1/inventory/{id}/host/{hid}/var` — GET, PUT, DELETE `/{name}`
3. Service-level acyclicity check on group ParentGroupId create/update → 400 on cycle.
4. External hosts: `invh` rows with null VirtualMachineId require explicit `ansibleHost`.
5. No `all` group rows in the DB — Ansible's implicit `all` is resolution-time behavior (P5).
6. Build: `python3 .cicd/build-cloud-manager.py --target api`

Done when: build passes; all routes respond per shape (C-INV wire example in PLAN.md); cycle
attempts return 400; routes 404/403 appropriately while `ansible-studio` is disabled.
