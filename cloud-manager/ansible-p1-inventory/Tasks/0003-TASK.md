# Task 0003 — Inventory events (cloud-manager-api)

## Steps
1. `IInventoryEventService` mirroring `IVmEventService`: `RecordEventAsync` + timeline getter
   only — never Update or Delete.
2. Record an `InventoryEvent` on every mutation from Task 0002: `created`, `updated`, `deleted`,
   `group_added`, `group_removed`, `host_attached`, `host_detached`, `var_set`, `var_deleted`
   (types per contracts §6). Best-effort try/catch — event write failure never blocks the
   primary write.
3. List endpoint `api/v1/inventory/{id}/event` (newest first), flag-gated like the rest.
4. Build: `python3 .cicd/build-cloud-manager.py --target api`

Done when: build passes; every mutation path records exactly one event; timeline endpoint
returns them newest first.
