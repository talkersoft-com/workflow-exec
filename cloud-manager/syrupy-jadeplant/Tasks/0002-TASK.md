# Task 0002 — Service playbooks: open each port to the trusted subnet

The exposing playbooks add a `ufw allow` task scoped to the trusted sources, mirroring how each one
already owns its `pg_hba` rules. The installing playbook owns its firewall rule.

## Steps
1. **Confirm the canonical playbook source (PLAN open question).** Determine whether the
   Postgres / RabbitMQ playbooks are **DB-only** (edit via the cloud-manager-mcp
   `cloud_playbook_get` / `cloud_playbook_update`, and `cloud_playbook_role_file_*` for role tasks)
   or **seeded from a repo**. Record the answer in `../deck.md`; if a repo backs them, add a
   `## Manual steps` git section there.
2. **Trusted sources var.** Reuse the Postgres playbook's existing `pg_hba_sources`, or define a
   parallel `ufw_sources` with the same values: `10.0.150.0/24` (VM subnet) + `10.0.144.140/32`
   (KVM host). Keep one source of truth per playbook.
3. **Postgres** (`pb_PG14JAMMY01` = pg14/jammy, `pb_EMEJCH08GG` = pg16) — add, looped over the
   sources:
   ```yaml
   - name: Allow Postgres from trusted sources
     community.general.ufw:
       rule: allow
       from_ip: "{{ item }}"
       to_port: "5432"
       proto: tcp
     loop: "{{ ufw_sources }}"
   ```
4. **RabbitMQ** (jammy/noble) — same shape; open `5672` (broker) always, and `15672` (mgmt UI) only
   if the operator wants it reachable on-subnet. **Confirm the exact port set during exec** and
   record it in `Results/RESULT.md`.
5. **`community.general` availability** — the `community.general.ufw` module must be present on the
   targets. Confirm via the playbook's collection requirements
   (`cloud_playbook_collection_requirement_list`); add `community.general` with
   `cloud_playbook_collection_requirement_create` (and install on the host via
   `cloud_host_collection_install`) if missing.
6. **Lint/check** each edited playbook (`cloud_playbook_lint` / `cloud_playbook_check`) green.

## Definition of done
Postgres (5432) and RabbitMQ (5672 [+15672 if chosen]) playbooks each add a `ufw allow from
{{ ufw_sources }} to <port>` task scoped to `10.0.150.0/24` + `10.0.144.140/32`; sources mirror
`pg_hba_sources` with a single source of truth; `community.general` requirement satisfied; canonical
playbook source recorded in deck.md; lint/check green.
