# Task 0001 — vorch-lib: cloud-init ufw baseline (every VM)

Repo: `vorch-lib`. The user-data builder is `buildUserData` in `provision/provision.go` (~line 464);
its base template ends the `runcmd` list with `rm /etc/netplan/50-cloud-init.yaml` → `netplan
generate` → `netplan apply` (~lines 488–492). Secret VALUES ride only in `write_files` (base64) and
are sealed by a separate runcmd line — the ufw block carries no secret and must not disturb that.

## Steps
1. **Append the lockout-safe ufw block to `runcmd`**, AFTER `netplan apply` (networking up first),
   in exactly this order so the live provisioning SSH session is never dropped:
   ```
    - ufw allow 22/tcp
    - ufw --force default deny incoming
    - ufw --force default allow outgoing
    - ufw --force enable
   ```
   `ufw` ships in the Ubuntu cloud image (no install); the commands are idempotent. Established
   connections are preserved and 22 is allowed, so the post-boot porch/ansible SSH runs are
   unaffected.
2. **Static template only** — no new create-vm field and no plumbing into `InitializeHost`'s
   signature; the block is the same for every VM. `vorch-service` stays unchanged.
3. **Keep the empty-secrets path byte-stable apart from the ufw block** — the no-bindings user-data
   must be identical to today plus exactly these four runcmd lines (no `cm-seed` write_files, no
   other runcmd churn). The secret-bearing path keeps the seal runcmd line after the ufw block.
4. **Test:** extend `provision/provision_secrets_test.go` — assert the generated user-data contains
   the four ufw lines in lockout-safe order immediately after `netplan apply`, and update/confirm
   the byte-identical golden (`TestBuildUserData_EmptyByteIdentical`) so the only delta vs. the old
   golden is the ufw block. `go build ./...` and `go test ./provision/...` green.

## Definition of done
`buildUserData` emits the four ufw runcmd lines in lockout-safe order after `netplan apply`; allow
22 precedes enable; no new create-vm field; empty-secrets user-data byte-stable except the ufw
block; vorch-lib build + provision tests green.
