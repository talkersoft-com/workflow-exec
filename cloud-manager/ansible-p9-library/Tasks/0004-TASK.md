## Steps
1. RESEARCH (read, no changes): contracts §8 (C-RES) — resolved vars, provenance, gap shapes
   and the `enforceRequired` flag; P8's context bar (inventory + host pickers, localStorage
   persistence) — reuse, do not fork; C-GRAPH usedBy response shape; the existing
   `ConfirmModal` danger pattern; where `plint` results surface for a playbook head revision.
2. Effective-config preview pane: on the composer and run pages, a per-target pane driven by
   P8's context bar listing resolved vars + provenance (winner / overridden chain) + gaps from
   C-RES; secret values render via `<SecretValue>` (masked, never cleartext); gaps render as
   chips, BLOCKING when `enforceRequired` is on, advisory otherwise.
3. Used-by guardrail: edit/delete of any shared entity (role, var, group var, playbook used by

   listing the consumers from C-GRAPH, destructive confirm in the existing `ConfirmModal`
   danger pattern. Zero consumers → no modal. No client-side reference guessing — usedBy is
   the only source.
4. Breaking-change warning: removing or renaming a `required` var of a role that has consumers
   → an explicit breakage list (which consumers break and why) before save can proceed.
5. Lint-gate toggle: a run-page setting that, when on, blocks triggering a run while the
   playbook's head revision has `plint` errors; surfaced inline with the lint results.
6. Unit tests: gap chips blocking vs advisory by `enforceRequired`, usedBy-gated modal
   (non-zero shows, zero skips), breaking-change detection on required-var rename/removal,
   lint-gate blocks and unblocks correctly.
7. Build: `python3 .cicd/build-cloud-manager.py --target web` must pass.

Done when: per-target preview with provenance and gaps works on composer + run pages, every
shared-entity mutation is used-by-guarded, breaking interface changes warn with the consumer
list, and the lint gate blocks dirty runs; web build clean.
