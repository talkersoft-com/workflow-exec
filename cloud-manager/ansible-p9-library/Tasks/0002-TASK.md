# Task 0002 — Part A: library browser + dependency graph view (cloud-manager-web repo)

## Steps
1. RESEARCH (read, no changes): contracts §5 (C-GRAPH) — `GET api/v1/entity/{id}/graph?depth=`
   and `GET api/v1/entity/{id}/usedBy` shapes; the existing `/ansible/*` list pages and their
   data hooks; house components (StatusBadge, SCSS tokens, list/table patterns); the plan's
   open-question defaults — d3-force for layout math only, custom React SVG rendering, no
   component library; inventories included in search results but no dedicated inventory pages.
2. Build `/ansible/library`: unified search across roles, playbooks, and collections (and
   inventories in results only) using existing list endpoints plus Task 0001's
   `?tag=&platform=&q=` filters; tag chips and platform badges as facets; per-entity used-by
   counts inline from C-GRAPH usedBy (batched/cached — no N+1 call storm on a page of results).
3. Build the dependency graph view: render `graph?depth=2` as an interactive SVG — d3-force
   for layout positions only, custom React rendering for nodes/edges; depth and edge-type
   filters; click a node → navigate to that entity; ~hundreds of nodes must stay responsive.
4. Wire navigation: library entries open existing entity pages; a "graph" affordance per
   entry opens the graph view centered on it.
5. Unit tests: search/filter composition, used-by count batching, graph data → node/edge
   mapping, edge-type filter behavior.
6. Build: `python3 .cicd/build-cloud-manager.py --target web` must pass.

Done when: `/ansible/library` searches and facets across the catalog with live used-by counts,
and the graph view renders C-GRAPH data interactively with click-through; web build clean.
Existing pages untouched.
