# Task 0005 — Finalize Results + Lessons; ship

## Goal
Finalize `Results/RESULT.md`, write `Retro/LESSONS.md`, then call `hv_ship` to open the PR(s).

## Target repo(s)
`cloud-manager-web` (the actual code change) + `workflow-exec` (workflow artifacts).

## Steps

1. **Finalize `Results/RESULT.md`** with:
   - Outcome: SHIPPED | PARTIAL | FAILED
   - Branch: the execution branch recorded in Task 0000
   - PRs: leave `TBD — filled from hv_ship output` (filled in after ship)
   - Phase summary: one short paragraph per phase
   - Pointers to the four screenshots and `verify.md` from Task 0004
   - The revision-count delta from Task 0004

2. **Write `Retro/LESSONS.md`** with:
   - What would have helped (be specific — missing MCP method, ambiguous task, false-positive test, missing doc)
   - List of FIX files written (if any)
   - Any deviation from the plan with a one-line justification

3. **Run `hv_status`** — verify only `cloud-manager-web` + `workflow-exec` are dirty (anything else is scope creep).

4. **Run `hv_ship`** with:
   ```
   hv_ship  deck: "cloud-manager"
            message: "feat(web): make Ansible playbook editor discoverable and dismissible"
            title:   "feat(web): make Ansible playbook editor discoverable and dismissible"
   ```

5. **Update `Results/RESULT.md`** with the PR URLs from the ship output. (`hv_ship` auto-advances the branch; if updating after-ship requires a commit on the next branch, leave it as is and rely on the ship output preserved in conversation history.)

6. **Stop.** Do not start anything else.

## Definition of done
- `Results/RESULT.md` populated.
- `Retro/LESSONS.md` populated.
- `hv_ship` returns success with merged or open PR URLs.
- All ORCH.md task checkboxes checked.
