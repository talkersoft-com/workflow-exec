# Task 0003 — Build + deploy

## Goal
Get the new web bundle running on `ubuntu-server.talkersoft.com` so Phase 4 can verify in a browser.

## Target repo(s)
`cloud-manager-web` (build) — deployed via `cloud-manager-api/scripts/web/install-web-app.py`.

## Steps

1. **Typecheck + build.**
   ```
   cd vm-infra/cloud-manager/cloud-manager-web
   npx tsc --noEmit
   sudo rm -rf dist
   npm run build
   ```
   - `tsc --noEmit` must report 0 errors.
   - `npm run build` must emit `dist/`.
   - The `sudo rm -rf dist` is needed because prior deploys leave `dist/assets` owned by `cloudmanager`; vite cannot overwrite as `todd`.

2. **Deploy.**
   ```
   cd vm-infra/cloud-manager/cloud-manager-api
   sudo python3 scripts/web/install-web-app.py
   ```
   Confirm the script finishes with `✅` or equivalent.

3. **Verify service is active.**
   ```
   sudo systemctl is-active cloud-manager-web
   ```
   Expected: `active`.

4. **Spot-check the deployed bundle has the new code.**
   - `grep -l "Discard unsaved changes" /opt/cloud-manager-web/dist/assets/*.js` — should print at least one filename.
   - This is a minimal sanity check that the new strings made it into the bundle.

## Definition of done
- `dist/` rebuilt clean.
- `install-web-app.py` finishes successfully.
- `cloud-manager-web` service `active`.
- New code string present in the deployed bundle.
