# Task 0001 — Editor close + referrer + dirty-guard + layout

## Goal
The `AnsiblePlaybookEditorPage` has a working Close button that returns the user to the page they came from (or falls back to detail on cold-load), warns on unsaved changes, and uses a flex layout instead of a magic-number height. `RoleFileEditor` exposes dirty state via a new optional prop.

## Target repo(s)
`cloud-manager-web`

## Steps

1. **Verify the existing `RoleFileEditor` shape.**
   - `grep -n "interface.*RoleFileEditor\|type.*RoleFileEditor\|export.*RoleFileEditor\|props" cloud-manager-web/src/components/RoleFileEditor/RoleFileEditor.tsx`
   - Confirm there is no existing `onDirtyChange` or equivalent dirty signal. If there IS one (e.g. `isDirty`, `onChange` exposes dirty), use it instead of adding a new prop. Record finding in `Retro/PROBE.md`.

2. **Check for a `ConfirmDialog` component.**
   - `grep -rn "ConfirmDialog\|<Confirm\b" cloud-manager-web/src`.
   - If a `ConfirmDialog` exists and is reusable here, use it. Otherwise `window.confirm` is acceptable.

3. **Extend `RoleFileEditor` with `onDirtyChange`.**
   - Add `onDirtyChange?: (dirty: boolean) => void` to its prop interface.
   - Inside the component, compute `dirty = currentContent !== initialContent` (the exact identifier names depend on the existing internals — re-use what's there; don't introduce a new state variable).
   - Wire a `useEffect` that calls `onDirtyChange?.(dirty)` whenever `dirty` changes.
   - No other internal changes.

4. **Update `AnsiblePlaybookEditorPage.tsx`.**
   - Replace the magic-number height (`calc(100vh - 120px)`) with a flex chain:
     ```tsx
     <div style={{ display: "flex", flexDirection: "column", height: "100%", minHeight: 0 }}>
       <div className="page__header" style={{ padding: 16, flexShrink: 0 }}>...</div>
       <div style={{ flex: 1, minHeight: 0 }}>
         <Suspense fallback={<Spinner />}>
           <LazyEditor ... onDirtyChange={setDirty} />
         </Suspense>
       </div>
     </div>
     ```
   - Add `const [dirty, setDirty] = useState(false);`
   - Add `const location = useLocation();` and `const navigate = useNavigate();` (import from `react-router-dom`).
   - Define the close handler:
     ```tsx
     const close = () => {
       if (dirty && !window.confirm("Discard unsaved changes?")) return;
       const from = (location.state as { from?: { pathname: string; search?: string } } | null)?.from;
       if (from?.pathname && from.pathname.startsWith("/ansible/")) {
         navigate(from.pathname + (from.search ?? ""));
       } else {
         navigate(`/ansible/playbooks/${pid}`);
       }
     };
     ```
   - Add a Close button in the header (right side, alongside the existing "Revisions" button). Label: `Close`. Use a secondary style consistent with other buttons on the page.
   - Add a `beforeunload` effect that warns when dirty:
     ```tsx
     useEffect(() => {
       if (!dirty) return;
       const handler = (e: BeforeUnloadEvent) => { e.preventDefault(); e.returnValue = ""; };
       window.addEventListener("beforeunload", handler);
       return () => window.removeEventListener("beforeunload", handler);
     }, [dirty]);
     ```

5. **Verify build is clean.**
   - `cd cloud-manager-web && npx tsc --noEmit` → 0 errors.
   - `npm run build` → completes successfully. Note: do NOT deploy in this task — Phase 3 owns deploy.

6. **Write `Retro/CHANGE-0001.md`** with one line per file edited:
   ```
   cloud-manager-web:src/components/RoleFileEditor/RoleFileEditor.tsx:<line range> — add onDirtyChange prop
   cloud-manager-web:src/pages/AnsiblePlaybookEditorPage.tsx:<line range> — Close button, referrer-aware navigate, flex layout, beforeunload guard
   ```
