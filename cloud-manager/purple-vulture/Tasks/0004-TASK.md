# Task 0004 — Browser verification (Playwright)

## Goal
Confirm all three Close paths and the entry points work in the deployed UI on `ubuntu-server.talkersoft.com`. Capture screenshots and route snapshots so Phase 5 has concrete before/after evidence.

## Target repo(s)
None (verification only).

## Steps

Use the Playwright MCP tools against `https://ubuntu-server.talkersoft.com`. Save screenshots and route notes under `Retro/` or `Results/`.

### Flow A — Open from list, save, close → land on list
1. Navigate to `/ansible/playbooks`.
2. Click the pencil icon on the `pg` row (or any visible playbook row).
3. Confirm URL is now `/ansible/playbooks/<pid>/edit` and the editor is rendered.
4. Take a tiny edit (e.g. add a comment line `# probe — purple-vulture`), click Save, wait for the success toast.
5. Click Close.
6. Confirm URL is back to `/ansible/playbooks`. Capture screenshot.

### Flow B — Open from detail, dirty close → confirm + land on detail
1. Navigate to `/ansible/playbooks/<pid>` (e.g. via clicking the row body, NOT the pencil).
2. Click the Edit button in the header.
3. Confirm URL is now `/ansible/playbooks/<pid>/edit`.
4. Make an edit (do NOT save).
5. Click Close.
6. A confirmation should appear (browser native `window.confirm` is fine). Cancel — confirm you stay on the editor.
7. Click Close again, this time accept the confirmation. Confirm URL is back to `/ansible/playbooks/<pid>`. Capture screenshot.

### Flow C — Cold-load, close → land on detail (fallback)
1. Navigate DIRECTLY to `/ansible/playbooks/<pid>/edit` (browser_navigate; no entry point).
2. Click Close (no dirty state needed).
3. Confirm URL is `/ansible/playbooks/<pid>` (fallback path). Capture screenshot.

### Sub-nav sanity
Across all three flows, the Ansible sub-nav should keep the "Playbooks" tab highlighted. Snapshot the editor view at least once to confirm the tab strip is visible above it.

### Save still creates a revision
- Note the revision count for the `pg` playbook before Flow A's save (`SELECT count(*) FROM vm.playbook_revisions WHERE playbook_id = (SELECT id FROM vm.playbooks WHERE public_id = 'pb_EMEJCH08GG');` via psql or db_execute_sql_query).
- After Flow A's save, re-query — count should be +1.

Save under `Results/`:
