# Task 0002 — Entry points: detail-page button + list-row pencil icon

## Goal
Operators can launch the editor from either `/ansible/playbooks` (icon button on each row) or `/ansible/playbooks/:pid` (primary button in header). Both surfaces pass `location.state.from` so the editor's Close button knows where to return.

## Target repo(s)
`cloud-manager-web`

## Steps

1. **`PlaybookDetailPage.tsx` — add Edit button to header.**
   - Import `useLocation, useNavigate` from `react-router-dom` if not already.
   - In the header (right-aligned, next to any existing actions): a primary button labeled `Edit` with a pencil-like icon (lucide / inline svg — match the icon system already used elsewhere in the page).
   - Click handler:
     ```tsx
     navigate(`/ansible/playbooks/${pid}/edit`, {
       state: { from: { pathname: location.pathname, search: location.search } },
     });
     ```

2. **`PlaybooksPage.tsx` — add pencil-icon edit affordance per row.**
   - Identify the row component or render block (likely a `<table>` row or a card). Add a trailing icon button using the same icon system.
   - Click handler:
     ```tsx
     (e) => {
       e.stopPropagation(); // don't trigger the row's existing click-to-open-detail
       navigate(`/ansible/playbooks/${playbook.publicId}/edit`, {
         state: { from: { pathname: location.pathname, search: location.search } },
       });
     }
     ```
   - The rest of the row continues to open the detail page on click (existing behavior).

3. **Verify build is clean.**
   - `cd cloud-manager-web && npx tsc --noEmit` → 0 errors.
   - `npm run build` → completes successfully. Do NOT deploy here — Phase 3 owns deploy.

4. **Append to `Retro/CHANGE-0002.md`** with one line per file edited.

## Constraints
- Don't move existing actions. Add the new affordance next to them.
- Don't add Edit affordances to Run, History, or role-detail surfaces.
- Don't change the row's existing click-to-detail behavior — the pencil icon uses `stopPropagation`.
- Use whatever icon system already exists in `PlaybookDetailPage` / `PlaybooksPage` (check imports). Don't introduce a new icon library.

## Definition of done
- `PlaybookDetailPage` has an Edit button in its header that navigates with `from` state.
- `PlaybooksPage` has a per-row pencil icon button that navigates with `from` state and doesn't propagate to the row's existing click handler.
- `npx tsc --noEmit` and `npm run build` both clean.
- `Retro/CHANGE-0002.md` exists.
- No other files in the repo are modified.
