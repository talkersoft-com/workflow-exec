# Task 0002 — MCP: cloud_secret_* tools

## Goal
Expose the Secret CRUD surface through `cloud-manager-mcp`, mirroring the API contract exactly —
the canonical client of the camelCase / public-id contract.

## Steps
- Add tools: `cloud_secret_list`, `cloud_secret_get`, `cloud_secret_create`, `cloud_secret_delete`.
- `cloud_secret_create` takes `name`, `description?`, `slug` (or `name`-derived), and the secret
  `value`(s) — and **never echoes the value back** in tool output (return the `Secret` metadata:
  public_id, name, vault_path, status).
- `cloud_secret_delete` surfaces the **409** guard message verbatim when a binding still references
  the secret.
- Mirror the existing `secret_binding` MCP tool implementations (request shaping, public_id passing,
  error surfacing). Register the tools in the server's tool list.

## Build
`python3 .cicd/build-cloud-manager.py --target mcp` must pass cleanly (npm ci lockfile-stable + build).

## Definition of done
Four tools build cleanly and, against the live API, create a manual secret, list/get it, and
surface the delete-guard 409. Test 0002 passes. (Operator must `/mcp` reload to use them live —
note this in RESULT.)
