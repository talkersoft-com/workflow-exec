# Task 0003 ‚Äî Web: Secret Manager page + binding‚Üímanaged-secret reference

## Goal
Add a **Secret Manager** page and let a Secret Binding reference an existing managed Secret ‚Äî
without a free-form path field anywhere a manual secret is created.

## Steps

### 1. Secret Manager page
- New route + nav entry (gate behind the `marketplace` feature flag, like Secret Bindings).
- **List**: name, `vault_path`, status badge (`Pending`/`Active`/`Failed`/`Deleting`/`Deleted`),
  delete action.
- **Create form**: the `cloudmanager/data/manual/` prefix is a **fixed, non-editable label**;
  the operator supplies only a **slug** (live-preview the composed path) + a **value** field
  (and `name`/`description`). No free-form path input. Submitting calls `create` and shows the
  resulting status.
- **Delete**: on the **409** guard, surface a clear message ("Can't delete ‚Äî referenced by binding
  N") rather than a generic error. On success, the row reflects `Deleted`.
- Reuse existing slice/page patterns (mirror `SecretBindingsPage` + its redux slice); add a
  `secrets` slice with `fetch/create/delete` thunks.

### 2. Binding ‚Üí managed Secret (static binding)
- On the Secret Bindings authoring modal, add a way to **reference an existing managed Secret**
  (select a Secret ‚Üí the binding stores `secret_id`; `path_template` becomes derived/read-only).
- **Keep templated authoring working** ‚Äî the existing free-form `{placeholder}` path flow for the
  `vm_public_id` case must remain (those bindings have `secret_id = null`).

### 3. UI discipline (per prior feedback)
Controls in a row must be **vertically aligned** (consistent 38px control height). After building,
**verify the create form and the binding modal visually** (the deploy makes them live) ‚Äî do not
claim the UI is correct without looking.

## Build
`python3 .cicd/build-cloud-manager.py --target web` must pass cleanly.

## Definition of done
Secret Manager page lists/creates (fixed-prefix) / deletes with the guard surfaced; a binding can
reference a managed Secret while templated authoring still works; web builds cleanly. Test 0003 passes.
