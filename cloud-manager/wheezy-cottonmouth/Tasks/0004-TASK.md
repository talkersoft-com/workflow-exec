# Task 0004 — Verify, build, deploy, ship

## Goal
Prove the feature end-to-end on the live system, then build, deploy, and integrate.

## Steps

### 1. End-to-end verification (live API + DB + Vault)
- **Create** a manual secret (e.g. slug `e2e-demo`) → confirm `vault_path =
  cloudmanager/data/manual/e2e-demo`, DB row `status=Active`, and the value is present in Vault.
- **Path-lock**: attempt (via SQL and via API) to create a secret with a path outside
  `cloudmanager/data/` → DB CHECK rejects + API rejects.
- **Gate L2** (Secret←Binding): reference the secret from a Secret Binding (`secret_id` set) →
  `DELETE` the secret → expect **409**; detach/delete the binding → secret delete now succeeds and
  the Vault key is gone (`status=Deleted`).
- **Gate L1** (Binding←VM): with a `vm_secret_bindings` row present (insert one by hand for the
  test, since population is workflow #2), attempt to delete that binding → expect RESTRICT/409;
  remove the usage row → delete succeeds. (This validates the FK only; the controller does not
  write usage rows in this workflow.)

### 2. Build
`python3 .cicd/build-cloud-manager.py --target all` — must pass cleanly.

### 3. Migration + deploy
- Apply the EF migration to the live DB **before** the api restart:
  ```
  cd .../cloud-manager-api/src/Models/CloudManager.Entities && \
    dotnet ef database update --connection "Host=localhost;User ID=cloudmanager;Password=P@ssw0rd!;Port=5432;Database=clouddb;SslMode=Prefer;Trust Server Certificate=true"
  ```
- `python3 .cicd/deploy-cloud-manager.py --target all`
- Confirm `cloud_health_check` green; web reachable on :3000.

### 4. Write outputs BEFORE hv_integrate
- `Results/RESULT.md` — outcome, branch, PR table (TBD until integrate), phase summary, deploy
  smoke results, and the **operator action**: *Restart cloud-manager-mcp via `/mcp` to pick up the
  new tools.*
- `Retro/LESSONS.md`.

### 5. Ship
`hv_integrate` per `../deck.md` Ship block (stage `exec`). Report PR links.

## Definition of done
End-to-end green (create + path-lock + both gate levels), all builds pass, migration applied,
services healthy, outputs written, PRs opened. Test 0004 passes.
