# Test 0001 — API: Secret entity, migration, state machine, path-lock

## TC-001 — Migration up/down clean
Apply the migration (`dotnet ef database update`), then roll it back one step, then re-apply.
Pass: all three succeed with no error; `marketplace.secrets` and `vm.vm_secret_bindings` exist
after up and are gone after down; `secret_bindings.secret_id` column + FKs appear/disappear.
Fail: any step errors, or Down leaves residue.

## TC-002 — vault_path CHECK constraint
Attempt `INSERT INTO marketplace.secrets (...) VALUES (..., 'secret/other/x', ...)`.
Pass: rejected by the CHECK. An insert with `cloudmanager/data/manual/x` succeeds.
Fail: the non-owned path is accepted.

## TC-003 — Create state machine (happy path)
`POST` a secret with slug `t-create` and a value.
Pass: response is camelCase with a `sec_…` public_id, `vaultPath = cloudmanager/data/manual/t-create`,
`status = Active`; the value is readable in Vault at that path; **no secret value in the response**.
Fail: status not Active, path wrong/free-form, value echoed, or Guid leaked.

## TC-004 — Create path-lock at API
`POST` attempting a path/slug that resolves outside `cloudmanager/data/`.
Pass: API rejects (400) before any Vault write.
Fail: accepted, or a Vault key written outside the namespace.

## TC-005 — Delete guard (L2: Secret←Binding)
Reference the secret from a Secret Binding (`secret_id` set), then `DELETE` the secret.
Pass: **409**, message names the blocking binding; secret remains `Active`; Vault key still present.
Fail: deletion proceeds while a binding references it.

## TC-006 — Leaf delete hard-deletes Vault
Remove the binding reference, then `DELETE` the secret.
Pass: `status=Deleted`, `deleted_at` set, and the Vault key is **gone**.
Fail: row not Deleted, or Vault key still present.

## TC-007 — FK is RESTRICT (L1: Binding←VM)
Insert a `vm.vm_secret_bindings` row pointing at a binding, then attempt to delete that binding.
Pass: blocked by RESTRICT (DB error / API 409); after removing the usage row, deletion succeeds.
Fail: binding deletes while a usage row references it.

## TC-008 — Build
`python3 .cicd/build-cloud-manager.py --target api`.
Pass: clean build. Fail: any compile error.
