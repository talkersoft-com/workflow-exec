# Task 0002 — Blueprint Migration

## Goal
Add an EF Core migration that registers both RabbitMQ marketplace blueprints and links
each to its playbook.

## File to create
New migration under
`cloud-manager-api/src/Models/CloudManager.Entities/Migrations/`
(model on `20260611040351_AddMarketplaceBlueprints.cs`).

## Up
- INSERT `marketplace.blueprints` × 2:
  - `rabbitmq-jammy` → image `Ubuntu 22.04 LTS (Jammy)`
  - `rabbitmq-noble` → image `Ubuntu 24.04 LTS (Noble)`
  - both: description per PLAN, `default_memory=4096`, `default_disk_size_gb=20`,
    `status='Published'`
- INSERT `marketplace.blueprint_playbooks` × 2: link each blueprint to the matching
  `vm.playbooks` row by name, `position=0`, `vars_override='{}'`.
- Use idempotent `SELECT ... WHERE name=` joins (same as the Postgres migration).
- Do NOT recreate the `marketplace` feature flag.

## Down
- Delete the `blueprint_playbooks` join rows, then the `blueprints` rows, for
  `name IN ('rabbitmq-jammy','rabbitmq-noble')`.

## Build
`python3 .cicd/build-cloud-manager.py --target api` must pass.

## Definition of done
Migration compiles; `Up`/`Down` present; build green.
