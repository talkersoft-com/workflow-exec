# Task 0001 — RabbitMQ Playbooks (jammy + noble)

## Goal
Author two seed playbooks + meta files under
`cloud-manager-api/seed/playbooks/` that install and configure RabbitMQ 4.x.

## Files to create
- `seed/playbooks/rabbitmq-jammy.yaml`  (model on `pg-14-jammy.yaml`)
- `seed/playbooks/rabbitmq-jammy.meta.json`
- `seed/playbooks/rabbitmq-noble.yaml`  (model on `pg-16-noble.yaml`)
- `seed/playbooks/rabbitmq-noble.meta.json`

## Playbook contents (both; differ only in release codename jammy/noble)
1. Add official Team RabbitMQ signing keys + apt sources (modern Erlang repo +
   rabbitmq-server repo) for the matching codename; `apt install` erlang deps +
   `rabbitmq-server`.
2. Enable + start `rabbitmq-server`.
3. `rabbitmq-plugins enable rabbitmq_management`.
4. Generate a strong password; create vhost `{{ rabbitmq_vhost }}`; create admin user
   `{{ rabbitmq_user }}` (tag `administrator`, full perms on the vhost); delete `guest`.
5. Write the generated password to Vault — reuse the Vault task block from
   `pg-14-jammy.yaml`, keyed for rabbitmq.
6. Open firewall ports 5672 (AMQP) + 15672 (management UI).

## meta.json
- `name`: `rabbitmq-jammy` / `rabbitmq-noble`
- `description`: `RabbitMQ 4.x on Ubuntu 22.04 LTS (Jammy)` /
  `RabbitMQ 4.x on Ubuntu 24.04 LTS (Noble)`
- `argumentSchema`: `rabbitmq_user` (default `admin`), `rabbitmq_vhost` (default `/`)
- `outputSchema`: `password` (Vault path), `management_url`

## Decisions carried from plan
- Install source: official Team RabbitMQ apt repos (not distro packages).
- Single-node only (no clustering/HA) for v1.

## Definition of done
Four files created; YAML parses; meta.json validates against the same schema shape as
the pg meta files.
