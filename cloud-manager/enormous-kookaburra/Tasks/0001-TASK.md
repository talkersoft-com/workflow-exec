# Task 0001 — ops/ Skeleton + Contract Files

## Goal

Create the `ops/` package with all input structs, a dispatch skeleton, and the committed contract files. No real logic yet — handlers return stubs. Build passes.

## Target repo

`hive-deck-pro` at `/Users/talker/workspace/hive-deck-pro/hive-deck-pro/`

## Steps

1. Create `ops/contract.go` — all input structs as defined in the plan (StatusInput, InitInput, ShipInput, NextInput, SyncInput, PruneInput, StashInput, UnstashInput, TeardownInput, WorkflowInput, DecksInput, ListInput, McpInput, BranchInput). Package name: `ops`.

2. Create `ops/dispatch.go` — `Dispatch(payload []byte) (string, error)` that unmarshals `op` field and switches on it. Each case calls a stub `Run*` function that returns `"not implemented: <op>"`. Import `encoding/json` and `fmt`.

3. Create one stub file per operation (e.g. `ops/status.go`) with the function signature matching the plan:
   ```go
   func RunStatus(in StatusInput) (string, error) { return "not implemented: status", nil }
   ```

4. Create `ops/contract/CONTRACT.md` — document every op with: op name, required fields, optional fields, example JSON payload, example output. Base content on the plan's input structs.

5. Create `ops/contract/schema.json` — JSON Schema draft-07 with `oneOf` discriminated on `"op"`. One entry per operation.

6. Run `CGO_ENABLED=0 go build ./...` from the repo root — must pass with no errors.

## Definition of done

`go build ./...` passes. `ops/` package exists with all files. `ops/contract/CONTRACT.md` and `schema.json` exist and are not in `.gitignore`.
