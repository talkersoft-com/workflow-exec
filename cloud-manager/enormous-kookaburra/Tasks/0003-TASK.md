# Task 0003 — Wire Cobra to ops + stdin JSON Dispatch

## Goal

Replace cobra RunE inline logic with direct `ops.Run*()` calls. Add stdin-piped JSON detection to `main()`. `hv <subcommand>` output is identical to before.

## Target repo

`hive-deck-pro` at `/Users/talker/workspace/hive-deck-pro/hive-deck-pro/`

## Steps

1. Update `cmd/hv/main.go` — add stdin JSON dispatch at the top of `main()`:
   ```go
   stat, _ := os.Stdin.Stat()
   if (stat.Mode() & os.ModeCharDevice) == 0 {
       payload, _ := io.ReadAll(os.Stdin)
       out, err := ops.Dispatch(payload)
       if err != nil { fmt.Fprintln(os.Stderr, "error:", err); os.Exit(1) }
       fmt.Print(out)
       return
   }
   ```
   Add imports: `io`, `os`, and the `ops` package.

2. For each cobra command's RunE body, replace the inline logic with a call to the corresponding `ops.Run*()` function, marshal args/flags into the typed input struct, and print the returned string:
   ```go
   RunE: func(cmd *cobra.Command, args []string) error {
       out, err := ops.RunStatus(ops.StatusInput{Op: "status", Deck: args[0]})
       if err != nil { return err }
       fmt.Print(out)
       return nil
   }
   ```

3. Run `CGO_ENABLED=0 go build ./...` — must pass.
4. Run `CGO_ENABLED=0 go test ./...` — must pass.
5. Install and smoke test:
   ```bash
   CGO_ENABLED=0 go install ./cmd/hv/
   hv status cloud-manager
   echo '{"op":"status","deck":"cloud-manager"}' | hv
   ```
   Both must produce identical output.

## Definition of done

`go build`, `go test` pass. `hv status cloud-manager` output unchanged. `echo '{"op":"status","deck":"cloud-manager"}' | hv` produces same output. cmd/hv/main.go RunE bodies contain no inline operation logic.
