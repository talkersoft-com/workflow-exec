# FIX-001 — cm-seed-secrets aborts with exit 141 (SIGPIPE) in gen_password

## Task / Test
Task 0001 — `cm-seed-secrets`. First `--dry-run` invocation (Test 0001 TC-001).

## Symptom
Script exited 141 right after the "starting" log line; no `.cred` files produced.

## Root cause
`gen_password()` used:
```sh
LC_ALL=C tr -dc 'A-Za-z0-9' < /dev/urandom | head -c 32
```
`head -c 32` closes the pipe after 32 bytes, so `tr` (still reading the infinite
`/dev/urandom`) receives SIGPIPE and exits 141. Under `set -o pipefail` the whole
pipeline reports 141, and `set -e` aborts the script. Classic `… | head` +
`pipefail` interaction.

## Fix
Bound the random input so the producer terminates on its own (no early pipe
close), then slice with bash parameter expansion:
```sh
gen_password() {
  local raw
  raw="$(LC_ALL=C tr -dc 'A-Za-z0-9' < <(head -c 1024 /dev/urandom))"
  printf '%s' "${raw:0:32}"
}
```
`head -c 1024 /dev/urandom` emits exactly 1024 bytes then EOF; `tr` consumes all
of it and exits 0; bash takes the first 32 alnum chars. No SIGPIPE, pipefail-safe.

## Retest
Re-run `--dry-run`: expect 3 `.cred` files at 0600 and a clean exit 0.
