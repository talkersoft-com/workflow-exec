# FIX-004 — cm-seed-secrets PG ALTER ROLE failed: psql `-c` does not interpolate `:'pw'`

## Task / Test
Task 0004 — live cutover, seeder step (also Task 0001 seeder behaviour).

## Symptom
During the cutover the seeder aborted at the Postgres apply:
```
ERROR:  syntax error at or near ":"
LINE 1: ALTER ROLE "cloudmanager" WITH PASSWORD :'pw';
```
`set -e` halted the cutover immediately — verified afterwards that nothing had
changed: all three services still `active`, PG still on the old password, no
drop-ins installed, base units unstripped, API `GET /Host/list` → 200. No damage,
fully safe to retry.

## Root cause
The seeder ran `psql -v pw="…" -c "ALTER ROLE … WITH PASSWORD :'pw';"`. psql does
**not** perform `:`-variable interpolation for the `-c` command string — the
literal `:'pw'` reached the server and was a syntax error. Variable interpolation
only happens for input read from stdin or a file.

## Fix
Feed the statement on **stdin** (where interpolation works) and use psql variable
quoting for both the role identifier and the password literal — injection-safe and
keeps the value out of the process argv:
```sh
( cd /tmp && sudo -u postgres psql -v ON_ERROR_STOP=1 -d "$PG_DB" \
    -v role="$PG_ROLE" -v pw="$DBPASSWD" -q >/dev/null ) <<'SQL'
ALTER ROLE :"role" WITH PASSWORD :'pw';
SQL
```
Verified the interpolation path against the live DB with a harmless SELECT before
retrying (`role=cloudmanager pwlen=9`).

## Retest
Re-run the cutover; expect the seeder to ALTER ROLE / change_password, store to
Vault, and render the three `.cred` files cleanly.
