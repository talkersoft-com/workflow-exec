# Task 0004 — Live cutover (reversible)

## Goal
Apply the change on the running controller, safely and reversibly.

## Steps (in order)
1. **Back up (MANDATORY, first):**
   - Vault snapshot: run `snapshot-vault.py` (cloud-manager-api/scripts/vault/scripts/).
   - `pg_dump clouddb` to a timestamped file off-box or in `/var/backups/cloud-manager/`.
   - Record both paths in `Results/RESULT.md`.
2. Deploy the API change: apply migration if any (none expected), then
   `python3 .cicd/deploy-cloud-manager.py --target api`.
3. Run the seeder: `sudo /…/cm-seed-secrets` -> writes the three `.cred` files, rotates PG/RabbitMQ.
4. Install the unit drop-ins + `sudo systemctl daemon-reload`.
5. `sudo systemctl restart cloud-manager-api vorch_service porch`.
6. Strip any remaining plaintext secret lines from units/config; daemon-reload again if needed.

## Rollback (if verify fails)
Restore `config-server.yaml`/units from `.pretoken-bak`/backups, restore Vault snapshot if
needed, restart services. Document under `Retro/FIX-NNN.md`.

## Definition of done
All three services `active`; secrets sourced from creds. Run `Test/0004-TEST.md`.
