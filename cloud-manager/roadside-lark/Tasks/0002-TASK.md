# Task 0002 — API reads secrets from systemd credentials (`AddKeyPerFile`)

## Goal
One change in `cloud-manager-api` so the .NET app sources `VAULT_TOKEN`, `DBPASSWD`,
`RABBITMQ_PASSWORD` from the decrypted credential files instead of plaintext env.

## Steps
1. In the API host/config setup (where `builder.Configuration` is composed), add, layered
   ABOVE environment variables:
   ```csharp
   var creds = Environment.GetEnvironmentVariable("CREDENTIALS_DIRECTORY");
   if (!string.IsNullOrEmpty(creds))
       builder.Configuration.AddKeyPerFile(directoryPath: creds, optional: true);
   ```
   Order so a present cred file overrides the env value (env -> cred).
2. Confirm the existing reads of `DBPASSWD` / `RABBITMQ_PASSWORD` / `VAULT_TOKEN`
   resolve through `Configuration[...]` (not raw `Environment.GetEnvironmentVariable`).
   Where they use the raw env API, route them through `IConfiguration` so the cred layer applies.
3. Do NOT change any other behaviour. Backward-compatible: env still works when no cred present.

## Build
`python3 .cicd/build-cloud-manager.py --target api`

## Definition of done
API builds clean; with a credentials dir present containing a `DBPASSWD` file, the API uses
that value. Run `Test/0002-TEST.md`.
