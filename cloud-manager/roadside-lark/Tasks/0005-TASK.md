# Task 0005 — Verify end-to-end, then ship

## Goal
Prove the system still works on creds, with no plaintext secrets, then write outputs and integrate.

## Steps
1. `systemctl is-active cloud-manager-api vorch_service porch` -> all `active`.
2. `sudo systemctl cat cloud-manager-api vorch_service porch | grep -iE 'P@ssw0rd|DBPASSWD=|RABBITMQ_PASSWORD=|hvs\.'` -> NO matches.
3. `cloud_health_check` (MCP) -> Healthy.
4. End-to-end provision: `cloud_vm_create` a Jammy VM -> within ~60s it gets `orchestrationStatus: 2`
   and an IP (vorch wrote the SSH key to Vault using the cred token). Tear it down after.
5. Write `Results/RESULT.md` (outcome, backup paths, PR table placeholder) and `Retro/LESSONS.md`.
6. `hv_integrate` (message/title per deck.md) -> PRs into the hive integration branch.

## Definition of done
All checks pass; Results + Retro written; `hv_integrate` opened the PRs. Run `Test/0005-TEST.md`.
