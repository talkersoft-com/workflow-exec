# Task 0001 — Token engine

## Goal
One path-expansion function, used everywhere paths are declared (PLAN.md §R7.5/R7.6).

## Steps (hive-deck-pro repo)
1. `internal/config`: `expandPath(declaringFile, value string, ws WorkspaceConfig) string` —
   `{{WorkspaceRoot}}` → workspace.root; `{{DecksRoot}}` → same (legacy alias); `~`/`~/x` → home;
   relative → joined to dir(declaringFile); absolute unchanged. Order: tokens, then ~, then
   relative check.
2. Wire into config path keys: `deck_config` (new key lands in 0002 — engine first),
   `plan_folder`, `exec_folder` (keep their existing `{{DeckRoot}}` handling working),
   `workspace.root` (~ already — converge on expandPath).
3. `internal/mcp/mcp.go`: `resolveArg`/`resolveEnvVal` accept `{{WorkspaceRoot}}` (same value as
   `{{DecksRoot}}`) and `~`; existing tokens and npm/flag passthrough behavior unchanged.
4. Unit tests: every form (canonical token, alias, ~, bare ~, relative w/ and w/o ./,
   absolute, `../` legacy) for config paths AND mcps args/env.
5. Build: `python3 .cicd/build-hive-deck.py --target go` (or make build) green. Do NOT install
   the binary yet.

## Definition of done
Build + unit tests green; mcps.yaml with {{WorkspaceRoot}} resolves identically to {{DecksRoot}}.
